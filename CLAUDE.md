## Game-start options

`GameStartOptions` (resource in `game_state.rs`) drives the spawn-select screen and is read once by `spawn_population` + `seed_starting_buildings_system`:
- `era: Era` â€” every spawned member starts with all techs through this era Aware. The *Learned* set is scoped by founder role via `PersonKnowledge::seeded_realistic_through_era`: chief gets Personal+Household+Subsistence+Specialist+Institutional, every ~8th member is Specialist (adds Specialist+MilitaryTransport), rest are Common (Personal+Household+Subsistence). `seeded_through_era` is retained for tests needing an everyone-knows-everything baseline. Structures and walls scale up accordingly.
- `player_population: u32` â€” group size for `group_idx == 0` (player faction). Other factions stay at hardcoded `GROUP_SIZE=20`.
- `economy: EconomyPreset` â€” `Subsistence` (empty policy = all-communist), `Mixed` (`mixed()` on non-staples; chief allocates food/wood/stone), `Market` (`capitalist()` on every resource). Applied per-faction in `spawn_population` via `policy::apply_preset`.
- `lifestyle: Lifestyle` â€” `Settled` (default) or `Nomadic`. Only the player faction reads it; AI factions stay Settled.
- `seed_buildings: bool` â€” sandbox sets false to skip pre-built seeding.

`spawn_population` sets `FactionData.chief_entity` and inserts `FactionChief` on the first spawned member.

## Architecture

CivGame is a Dwarf Fortress-style civilization simulation on **Bevy 0.15** (ECS). Six plugins:

| Plugin | Directory | Responsibility |
|--------|-----------|----------------|
| `WorldPlugin` | `src/world/` | Procedural terrain (Z-levels -16..+15), chunk streaming (32Ã—32 tiles), biomes, calendar, `SpatialIndex`, `ResourceCatalog` |
| `SimulationPlugin` | `src/simulation/` | Agent AI, needs, combat, reproduction, factions, technology, memory/gossip, plants, raids |
| `EconomyPlugin` | `src/economy/` | Markets, goods, prices, transactions |
| `PathfindingPlugin` | `src/pathfinding/` | Component-typed chunk graph, hotspot flow fields |

Per-directory `CLAUDE.md` files cover subsystem detail. Claude Code auto-loads them when reading/editing in those trees.

## Settlement construction

Settlement layout flows through `SettlementPlan` â†’ plot carving â†’ blueprint placement. See `src/simulation/CLAUDE.md` for system detail.

- **`Settlement.peak_population`** â€” monotonic max of owner-faction's `member_count`, maintained by `settlement_peak_population_system`. Drives civic milestones.
- **`StreetSpine`** â€” `None` (Paleo/Meso) / `Linear` / `Spokes` / `Grid`, generated per layout Ã— era. `settlement_planner_system` enqueues spine segments into `RoadCarveQueue`; `road_carve_system` lays Bresenham road tiles.
- **Plots** carry `frontage_edge` + `access_tile`; residential placement (Hut/Longhouse) prefers vacant frontage lots so doors face the road, falling back to zone-area scoring.
- **Door direction is sourced.** `Door` carries `dir: TileEdge` + `doormat_tile`. `plan_building` reads `door_dir` from the matched plot's `frontage_edge` (carried through `BuildCandidate.door_dir â†’ BuildIntent â†’ Blueprint.door_dir`); the entrance always lands on the centre of the road-facing wall, never a corner. Fallback when no frontage is `TileEdge::toward(centre, home)`. `entrance_cell_for_edge` computes the door cell from `(half_w, half_h, edge)`.
- **Doormat reservations** (`simulation/doormat.rs`): every door registers its 1-tile cardinal-outside neighbour in `DoormatReservations`. All footprint/zone/yard checks (`is_clear_footprint`, `plot_rect_vacant`, `find_palisade_site`, `find_clear_tile_in_zone`, `find_unfilled_civic_zone_tile`, `find_bed_tile_around_hearth`, `seed_farmstead_yard`) refuse reserved tiles. Door spawn writes the doormat to `Road` via `write_road_tile`; a `(doormat â†’ home)` Bresenham extension is queued onto `RoadCarveQueue` only when no existing Road sits within chebyshev 4 (`road_within`). The `Door` `on_remove` hook frees the reservation.
- **Door direction is verified, not assumed.** `pick_clear_door_cardinal` tries the preferred cardinal; if that doormat is Wall/Stone/Blueprint/Bed/impassable/reserved, it falls back to other cardinals ranked by doormatâ†’home chebyshev. Returns `None` if every cardinal is blocked â€” `plan_building` / `plan_composite_building` / `seed_walled_house_at` abort rather than place an unreachable door. The seed loop stamps failed anchors into `used` and continues.
- **Doormat reachability gate.** `doormat_reaches_home` runs a bounded BFS (1500-node cap) from each candidate doormat to the faction's home tile. Cardinals locally clear but sealed in a courtyard pocket fail this check.
- **Roads are protected from new construction.** `is_clear_footprint` / `is_clear_shape` / `seed_walled_house_at` preflight / `pick_seed_house_anchor` / `find_palisade_site` / `seed_perimeter` / `seed_perimeter_rect` all reject `TileKind::Road` candidates.
- **Wider palisade gateways.** All three palisade carvers leave a 3-tile gap on each cardinal axis so the spine has real flow capacity through the wall.
- **Civic milestones** (`civic_milestones.rs`): `(Era, peak_pop) â†’ bool` table gates Granary / Shrine / Market / Barracks / Monument growth. Seeded structures bypass the gate.
- **`building_template.rs`** ships `FootprintShape::{Rect, LShape, UShape}` + `Rotation` helpers. `BuildIntent::CompositeHouse { shape, rotation, wall_material }` is wired: `plan_composite_building` walks `shape_tiles` emitting perimeter Wall + one Door at the frontage cardinal + interior Bed blueprints. Chalcolithic+ residential rolls a 10% LShape farmstead (2Ã—2 + 2Ã—1) when `bed_deficit` is moderate. UShape exists in `shape_tiles` but isn't yet auto-emitted.
- **Organic layout randomness** (`settlement.rs::jitter_zones` + `generate_candidates`): `SettlementPlan.culture_hash` seeds `fastrand::Rng` for Â±1 tile zone jitter (skipping civic/defense). `generate_candidates` rolls Hut-vs-Longhouse (Longhouse forced when `bed_deficit â‰¥ 4`, otherwise 60/40). Different `faction_id` produces visibly different villages.
- **Game-start seeding** (`seed_starting_buildings_system`) is era-additive, house-per-founder: Paleo/Meso multi-hearth camp via `paleolithic_hearth_positions`; Neolithic+ walled Huts (3Ã—3 + door + bed) with 2Ã—2 fertility-bumped yard via `seed_farmstead_yard` (no Farmland tile â€” wheat grows on high-fertility Grass); Chalcolithic+ Hut/Longhouse alternation, Palisade walls (Bronze upgrades to Mudbrick); Bronze yard 3Ã—3. Bed count `= max(member_count, era_floor)`. Walls follow extent (used-tile bbox + 2, unioned with Defense zone).
- **Child farm plots** (`Plot.parent_plot`): residential plot acquisition also claims the nearest unowned same-village Agricultural plot within 12 tiles â€” same household, mirrored tenure (rent flows through the parent).

## Nomadic mode

`Lifestyle::{Settled, Nomadic}` on `FactionData`; nomadic factions skip Settlement spawning, run a camp pipeline, and migrate seasonally. Two coexisting flows: AI commits atomically (Surveying â†’ PendingCommit â†’ Walking phases) while player-controlled nomadic factions transition between `CampState::Pitched` and `CampState::Packed` via `PlayerCommand::PackCamp` / `PitchCamp`. Detailed system list in `src/simulation/CLAUDE.md` (search `Camp lifecycle`, `nomad`, `wild_herd`, `pack_deploy`, `sedentary_collapse`).

- **Shelters** (`pack_deploy.rs`): `Deployable { packed_form, refund_pct, refund_resource, refund_qty }` on every nomadic structure. Bedroll (1 skin + 2 wood) and Yurt (8 wood + 6 skin, gated on `PORTABLE_DWELLINGS`) are `fully_packable`; Tent (6 wood + 3 skin) is `refund_only(0.5, wood, 6)` â€” half drops as `GroundItem`s on teardown.
- **Camp seeding** (`seed_nomadic_camp`): hearth ring, bedrolls (radial 2..=5, one per founder), tents (outer 5..=7, ~1 per 4 founders), yurts (Neolithic+, inner 3..=5, capped 2).
- **Migration** (`nomad.rs`): `nomad_migration_system` (Economy, daily) scores candidates by food density, wild herd aggregate, water proximity, biome-season fit, predator danger, recency penalty (`recent_camps` deque cap 6); on trigger sets `pending_migration`. `nomad_migration_commit_system` (Sequential, exclusive World) redistributes essentials â†’ packs shelters into pack animals / inventories â†’ despawns old camp within `OLD_CAMP_RADIUS = 12` â†’ re-seeds at target â†’ stamps members with `MigrationTarget` + `Goal::MigrateToCamp`. `nomad_migration_dispatch_system` (ParallelB) routes `Task::WalkTo { why: Migration }`; `nomad_migration_arrival_system` (Sequential) clears markers on arrival within chebyshev 4 or 3-day timeout.
- **Pack animals**: `PackAnimalInventory { items: [(ResourceId, u32); 6], capacity_g }` on `Tamed` (Horse 60kg, Cow 80kg, Pig 30kg, Dog 15kg). `attach_pack_inventory_system` inserts on `Added<Tamed>`. `compute_faction_storage_system` folds tamed-animal inventories into nomad faction `storage.totals`. `combat::death_system` drops contents as `GroundItem`s on death.
- **Tamed-animal herding**: commit pass overwrites `AnimalAI.target_tile` for every owner-faction animal so the herd walks to the new camp.
- **Wild herds** (`wild_herd.rs`): `WildHerdRegistry` rows `(species, aggregate_count, leader_tile, range_center, bloomed, members, flee_until_tick, last_birthed_tick)`. Daily migration drifts leaders within `HERD_RANGE_HALF = 30` (Winter south, Spring north); predator flee dominates (wolves within `HERD_FLEE_RADIUS = 8` bias drift away Ã—3 for a day); otherwise water-seek + nomad-camp avoidance; non-Winter birth +12 capped at 200. Bloom/collapse at camera distance 32/48 spawns up to 60 individuals; predation shrinks the herd across cycles.
- **Slim chief directives** (`nomad_chief_directive_system`): nomadic factions retain a narrow chief role queueing Bedroll / Tent / Yurt blueprints (no postings) toward `nomad_shelter_targets(members)`.
- **Band redistribution** (`nomad_pool.rs`): `nomad_band_pool_balance_system` (every TICKS_PER_DAY/4) shrinks max-min holding spread of essentials (bedroll / packed_yurt / preserved_meat) to â‰¤1 unit across the band. Also runs inline before commit pass.
- **Preserved meat ration**: `2 meat + 1 wood â†’ 3 preserved_meat` (CraftRecipe 12, gated on `FOOD_SMOKING`). `eat_task_system` runs two passes â€” fresh first, preserved only when nothing else on hand.
- **Sedentarization** (`nomad_sedentarize_system`): nomadic faction with `â‰¥ 12` members + no pending migration + stable for one game-year emits `SettlementLifecycleEvent::SwitchArchetype` to the settled variant.
- **Reverse collapse** (`sedentary_collapse.rs`): settled faction sampling population crash + food deficit + shelter loss bumps `collapse_streak`; at one season triggers `SwitchArchetype` to the nomadic variant.

## Wage-aware labor market

End-to-end pipeline: chief-funded postings â†’ escrow â†’ worker payout â†’ faction wage signal â†’ EV-driven profession choice. See `plans/wage-aware-labor-market-v2.md` and `src/simulation/CLAUDE.md` for subsystem detail.

- **Chief postings pay from faction treasury (Mixed/Market only).** `chief_post_funding_system` (Economy, after `chief_job_posting_system`) scans new chief contracts with `reward == 0.0`. Unaffordable leaves `reward = 0`. Subsistence factions skip funding.
- **Worker payout closes the loop.** `job_payout_system` (Economy, exclusive World, after `job_claim_release_system`) drains `JobCompletedEvent`, looks up escrow via `JobEscrowIndex(AHashMap<JobId, Entity>)`, and on `completed` splits `escrow.amount` evenly across claimants by direct credit. Failed completions despawn the escrow intact; the `JobEscrow` `on_remove` hook refunds `beneficiary`. Workers get an `Earnings` ring (cap 16) + `ActivityEntryKind::WagePaid`. - **Skills decay toward peak-derived floors.** `Skills` is clamped at `SKILL_MAX = 255`; `SkillPeaks` + `SkillUseTicks` ratchet via `skill_peaks_tracker_system` (observer on `Changed<Skills>`). `skill_decay_system` (daily, `SKILL_DECAY_HALF_LIFE_DAYS = 90`): skills above `SKILL_MASTERY_LINE (80)` decay toward `SKILL_MASTERED_FLOOR (30)`; below mastery toward `max(SKILL_FLOOR_BASE (5), peak Ã— 0.30)`. - **Capital recognition lifts EV for well-equipped agents.** `capital_factor(...)` averages three axes in `[1.0, 2.0]`: `tool_capital_factor` (+0.5 when an inventory/hand item maps to `prof` via `tool_profession`: weaponâ†’Hunter, toolsâ†’Crafter); `workshop_capital_factor` (+1.0 household-owned affine workshop within `WORKSHOP_AFFINITY_RADIUS = 12`, else +0.5 village-owned; `WorkshopKind::affine_to` keys Marketâ†’Bureaucrat, Workbench|Loomâ†’Crafter); `land_capital_factor` (Farmer: +0.5 for household-held Agricultural plot under non-StateOwned tenure). `OwnedBy { faction_id, kind, tile }` at workshop finalize maintains `WorkshopOwnership` via add/remove hooks.

- **Profession choice is EV-ranked.** `expected_wage = aggregate_wage_per_day Ã— skill_competence(primary_skill) Ã— capital_factor` where `skill_competence(s) = 0.2 + (s/SKILL_MAX) Ã— 0.8`. ## Spatial / tile / rendering conventions

- World tiles: `(i32, i32)`; convert with `tile_to_world()`.
- Chunks: `ChunkCoord::from_world()` (uses `div_euclid()`).
- Z-levels: `i8`, `Z_MIN=-16`, `Z_MAX=15`.
- `TileKind::River` is the freshwater sibling of `TileKind::Water`. Both are impassable; `kind.is_water_like()` accepts either, `kind.is_freshwater()` is River-only. Lakes/oceans stay `Water` until `LakeBasin` learns a fresh/salt flag. `chunk_map.river_distance_at(tx, ty)` returns chebyshev tiles to nearest river (`u8::MAX` = far / unloaded), populated at chunk-gen and used by riparian biome shift, fertility boost, settlement-spawn scoring, and herd/nomad fresh-water preference.
- **Tile palette:** `TileKind` carries 23 variants. **Surfaces:** `Grass`, `Forest`, `Sand` (hot/dry; reuses Farmland's u8 slot), `Snow` (tundra), `Marsh` (wetland, slow path), `Scrub` (steppe / arid), `Water`, `River`, `Road`. **Stone lithologies** (`is_stone_like()`): `Stone` (legacy/fallback), `Granite`, `Limestone` (yields 3 vs. 2 per swing), `Sandstone`, `Basalt`, plus underground `Wall` and `Ore`. **Soil variants** (`is_soil_like()`): `Dirt` (legacy), `Loam` (1.5Ã— fertility), `Silt` (1.4Ã—, riparian), `Clay`, `SandySoil` (0.6Ã—). Helpers: `is_stone_like` / `is_soil_like` / `stone_yield_count` / `soil_fertility_mult`. **Farmland is removed** â€” wheat now spawns on high-fertility `Grass` and the `seed_farmstead_yard` path bumps fertility on natural soil rather than writing a synthetic tile. Pathing speeds: `Sand 0.75`, `Snow 0.6`, `Marsh 0.4`, `Scrub 0.9`, soils 0.85â€“0.9, stone variants 1.0.
- Fixed update: **20 Hz** (`main.rs`). Game speed and pause live on `Time<Virtual>` via the `GameSpeed` resource (`src/simulation/speed.rs`): `Paused / Normal / Fast (2Ã—) / VeryFast (5Ã—)`. HUD speed buttons and the keyboard handler (Space toggles pause; 1/2/3 pick presets, egui-focus-gated) write `GameSpeed`, and `sync_game_speed_to_virtual_time` (PreUpdate) mirrors it onto `Time<Virtual>::set_relative_speed` / `pause`. Higher presets fire FixedUpdate more often per real second â€” agents, needs, cooldowns, calendar, plant lifecycle, and every tick-cadence Economy system scale together. `SimClock.scale_factor()` now carries bucket compensation only (population / bucket_size); the speed multiplier is gone. `SimTimingDiagnostics` (debug panel "Sim Timing") tracks ticks/frame, EMA tick CPU, and worst-tick window; reddens when the rolling average exceeds the preset's CPU budget (1Ã— = 50 ms, 2Ã— = 25 ms, 5Ã— = 10 ms).
- After mutating a tile, emit `TileChangedEvent { tile }`; `refresh_changed_tiles_system` (PostUpdate) rebuilds sprites.
- `CameraViewZ` defaults to `i32::MAX` (surface); lower it to peer underground.
- `TileMaterials`/`FogTileMaterials` keyed by `(TileKind, OreKind, z_bucket)`. Ore tiles fan out via `RENDERABLE_ORES`; colors in `color_map.rs::ore_tile_color`.
- **`sprite_library.rs`:** procedural pixel art from a 32-color palette via `ascii_to_image`. Reuse the palette/helpers â€” don't introduce new color systems.
- **PNG textures** in `assets/textures/` toggled by `entity_sprites::toggle_art_mode`.

## Constraints

- **ECS:** logic in Systems; Components hold data only. No OO inheritance.
- **UI:** `bevy_egui` for panels (avoid `bevy_ui` except specific overlays).
- **Hashing/randomness:** `ahash::AHashMap` (not `std::HashMap`). `fastrand` in hot paths, `rand` for init.
- **No new crates** without explicit permission.
- **Error handling:** avoid `unwrap()` in core systems; use `match`/`if let`.
- **Mutable aliasing:** be careful with Bevy query aliasing; test empirically.
- **Doc updates:** when behaviour changes, update the matching `CLAUDE.md`. Subsystem-local changes in `src/<dir>/CLAUDE.md`; cross-cutting in this file.
