# UI (`src/ui/`)

All UI uses `bevy_egui` (not `bevy_ui`, except specific overlays).

- **Right-click context menu (`orders.rs`):** Three sections — tile actions (Move/Mine/Gather/Dig Down/Deconstruct/Build), entities on tile (Attack / Pick up corpse), ground item stacks (Pick up: Nx Good). UI is **event-only**: `right_click_context_menu_system` translates the chosen action into a `PlayerCommand` and emits `PlayerCommandEvent` (`simulation/player_command.rs`). Sim drains the event, attaches `Commanded`, and routes via `dispatch_player_command_system` — UI never mutates `PersonAI` / `ActionQueue` directly. Adding a new order type: (1) new `PlayerCommand` variant, (2) match arm in `dispatch_one`, (3) match arm in `player_command_lifecycle_system` (terminal-state detector), (4) UI button emits the event. No HTN filter updates required.
- **Inspector (`inspector.rs`):** Drop / unequip / equip queue actions via `PendingInspectorAction`; `inspector_action_system` executes after panel render. `valid_equip_slots(good)` (`../simulation/items.rs`) maps goods to slots. Knowledge section lists Learned/Aware sets and capacity. Each Learned row has Lecture/Encode buttons (`HoldLecture`/`EncodeTablet`); Aware-only rows with a matching tablet/book in inventory show Read (`ReadItem`). "In progress" sub-section lists `study_progress` as `tech: progress/threshold ticks`. **Task & State** reads `MethodHistory` + `SimClock`: `Method:` falls back to `(direct dispatch — no HTN method)` when `aq.current != Idle` but `active_method` is None (bureaucrats / traders / player orders), or `(none — see HTN history)` when fully idle. When `task_id == UNEMPLOYED && aq.current == Idle`, an orange `Idle:` line shows `ticks_idle`, `last_goal_eval_tick`, and the most recent in-TTL `MethodHistory` entry. A collapsible **HTN history** sub-section colors entries by outcome.
- **Tech panel (`tech_panel.rs`):** Tri-state per tech — ✓ green (chief Learned), ◉ teal (chief Aware only), ◎ yellow (discoverable), ○ gray (locked). Hover row: "Complexity: N pt(s) · Members: N/M learned, N aware".
- **Activity log (`activity_log.rs`):** Bottom-right egui panel; `ActivityLogEvent { tick, actor, faction_id, kind }`. Kinds: `Constructed`, `Crafted`, `TechDiscovered`, `RegionSettled`, `Taught`, `Read`. Filtered to player faction; cap 16 entries.
- **Spawn-select (`spawn_select.rs`):** Full-screen biome map (rendered at `WORLD_MAP_OVERSAMPLE=2` × climate-cell density via per-pixel `biome::classify_at_tile`, with elevation-driven hillshade brightness `0.55 + 0.95*elev_f` so mountains read bright and deep ocean reads dark). Two grid overlays: faint climate-cell sub-grid (every 4 cells) + bold mega-chunk grid. Hover tooltip surfaces dominant biome, elevation band (below sea level / lowland / highland / mountain), raw u8 elevation, and temperature. Click a habitable mega-chunk sets `PendingSpawn`. Ocean/Mountain non-habitable. Side panel hosts era / pop / economy / lifestyle controls plus a **World** section with editable `Seed:` field and Apply/Reroll buttons; both fire `RegenerateWorldRequest` (handled by `world::regenerate_world_system`) which rebuilds `Globe` + `WorldGen` and clears the `SpawnSelectTexture` / `WorldMapTexture` caches.
- **World-map switcher (`world_map.rs`):** Settled mega-chunks outlined (yellow=player, red=other). Renders at `WORLD_MAP_OVERSAMPLE=2` for the same sub-cell biome detail. Click bookmarks current camera onto its region, then jumps to target's `camera_bookmark`. Tab toggles map.
- **Muster button (`hud.rs`):** Emits `PlayerCommandEvent { actors: <player Hunters>, command: Muster }`. `dispatch_player_command_system` inserts `Drafted` on each actor, clears plan/reservations, removes `Carrying`. Player issues rally point separately via right-click. `apply_draft_toggle_system` (HUD button + `R` key) emits `Muster` or `Disband` depending on whether any selected unit is undrafted. Orthogonal to chief muster (uses `HuntOrder::Hunt::mustered`, not `Drafted`).
- **Pack/Pitch Camp (`hud.rs` + `orders.rs`):** Mobile (nomadic) player factions show a `Pack Camp` button on the HUD when `camp_state == Pitched` — emits `PackCamp` on the chief. When `camp_state == Packed`, the HUD reads "Mobile (right-click → Pitch)" and the right-click context menu adds a `Pitch Camp Here` action on any passable tile ≥ `MIN_PITCH_DISTANCE` from the current packed centroid; emits `PitchCamp { tile, z }`. Between Pack and Pitch the band is mobile — players issue `PlayerCommand::Move` on selected members to caravan, and members can forage / sleep / fight via the goal-gating safety net.
