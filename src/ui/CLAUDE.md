# UI (`src/ui/`)

All UI uses `bevy_egui` (not `bevy_ui`, except specific overlays).

- **Right-click context menu (`orders.rs`):** Three sections — tile actions (Move/Mine/Gather/Dig Down/Deconstruct/Build), entities on tile (Attack / Pick up corpse), ground item stacks (Pick up: Nx Good). UI is **event-only**: `right_click_context_menu_system` translates the chosen action into a `PlayerCommand` and emits `PlayerCommandEvent` (`simulation/player_command.rs`); sim attaches `Commanded` and dispatches. **Multi-select broadcast:** the menu renders against `SelectedEntity` (focus) but dispatch broadcasts to every non-drafted player-faction worker in `SelectedEntities` — drafted units stay under `military_right_click_system`. Adding an order: new `PlayerCommand` variant, arm in `dispatch_one`, arm in `player_command_lifecycle_system` (terminal-state detector), UI button emits the event.
- **Inspector (`inspector.rs`):** Drop / unequip / equip queue actions via `PendingInspectorAction`; `inspector_action_system` executes after panel render. `valid_equip_slots(good)` (`../simulation/items.rs`) maps goods to slots. Knowledge section lists Learned/Aware sets with Lecture/Encode (Learned) and Read (Aware with matching tablet) buttons; "In progress" sub-section lists `study_progress`. **Task & State** reads `MethodHistory` + `SimClock`: `Method:` falls back to `(direct dispatch — no HTN method)` for bureaucrats / traders / player orders, or `(none — see HTN history)` when fully idle; idle agents show ticks_idle + `last_goal_eval_tick`. **Profession line** inlines apprenticeship: `Profession: Apprentice  apprentice 4/30 d (13%)` plus mentor entity index; masters show `mentoring #N`. **Skills section** prints `cur / peak P (floor F)` via `skills::skill_floor(peak)` (shared with `skill_decay_system`); mastered (peak ≥ `SKILL_MASTERY_LINE`) render green. **Wage & Labor section** (between Stats and Knowledge): 24-hour Earnings rollup + ring total + last entry; per-profession EV table (`Farmer / Hunter / Crafter / Bureaucrat / Trader` — current highlighted yellow, zero-EV greyed) using `capital::capital_factor`; own-faction `wage_signal` top-6 by EMA/day; cross-faction `PerceivedFactionWages` top-6 with observation-age. Params bundled into `WageInspectorParams` SystemParam (peaks / earnings / perceived / apprenticeship / household + `WorkshopOwnership` + `PlotIndex` + `Plot` query) to stay under Bevy's per-tuple ceiling.
- **Tech panel (`tech_panel.rs`):** Per-tech icon mirrors community `AdoptionStage` — ◆ Institutionalized / ✓ Adopted / ● Practiced / ◐ Demonstrated / ◉ Rumored / ◎ Discoverable / ○ Locked. Hover shows adoption stage, chief's Learned/Aware (separately) and per-tech member tally.
- **Activity log (`activity_log.rs`):** Bottom-right egui panel; `ActivityLogEvent { tick, actor, faction_id, kind }`. Kinds: `Constructed`, `Crafted`, `TechInsight`, `TechDiscovered`, `TechAdopted` / `TechInstitutionalized` (from `derive_tech_adoption_system`), `RegionSettled`, `Taught`, `Read`, `CampMoved`, `Sedentarized`, `WagePaid`, plus labor-market entries: `ProfessionChanged { from, to }` emitted centrally by `profession_choice::profession_change_log_system` (Economy, observes `Changed<Profession>` via `LastSeenProfession` shadow — runs after profession-mutation systems so same-tick chains collapse to one entry); `ApprenticeshipStarted { mentor }` from `chief_craft_assignment_system`; `ApprenticeshipGraduated` from `apprentice_progress_system`. Centralised emitter suppresses `None↔Apprentice` / `Apprentice→Crafter` to avoid double-logging. Filtered to player faction; cap 16.
- **Spawn-select (`spawn_select.rs`):** Full-screen biome map rendered at `WORLD_MAP_OVERSAMPLE=2` × climate-cell density via per-pixel `biome::classify_at_tile`, with elevation-driven hillshade (`0.55 + 0.95*elev_f`). Two grid overlays: faint climate sub-grid (every 4 cells) + bold mega-chunk grid. Hover tooltip: dominant biome, elevation band, raw u8 elevation, temperature. Click a habitable mega-chunk → `PendingSpawn`. Ocean/Mountain non-habitable. Side panel: era / pop / economy / lifestyle controls + **World** section with editable `Seed:` + Apply/Reroll buttons firing `RegenerateWorldRequest` (handled by `world::regenerate_world_system`), which rebuilds `Globe` + `WorldGen` and clears `SpawnSelectTexture` / `WorldMapTexture` caches.
- **World-map switcher (`world_map.rs`):** Settled mega-chunks outlined (yellow=player, red=other). Renders at `WORLD_MAP_OVERSAMPLE=2` for sub-cell biome detail. Click bookmarks current camera, then jumps to target's `camera_bookmark`. Tab toggles map.
- **Speed controls (`hud.rs` + `simulation/speed.rs`):** HUD speed buttons (`⏸ / 1× / 2× / 5×`) write `GameSpeed.current`, which a PreUpdate sync system mirrors onto `Time<Virtual>::set_relative_speed / pause`. Keyboard shortcuts (`Space` toggles pause; `1/2/3` pick `Normal/Fast/VeryFast`) are gated on `EguiContexts::wants_keyboard_input` so typing into a panel doesn't hijack them. The debug panel's "Sim Timing" section shows ticks/frame + EMA tick CPU + worst-tick window (60-sample), reddens when `avg > preset.budget_ms_per_tick()`.
- **Muster button (`hud.rs`):** Emits `PlayerCommandEvent { actors: <player Hunters>, command: Muster }`; dispatch inserts `Drafted`, clears plan/reservations, removes `Carrying`. Rally point set separately via right-click. `apply_draft_toggle_system` (HUD button + `R`) emits `Muster` or `Disband` based on whether any selected unit is undrafted. Orthogonal to chief muster (uses `HuntOrder::Hunt::mustered`, not `Drafted`).
- **Pack/Pitch Camp (`hud.rs` + `orders.rs`):** Mobile (nomadic) player factions show `Pack Camp` when `camp_state == Pitched` (emits `PackCamp` on chief). When `Packed`, HUD reads "Mobile (right-click → Pitch)" and right-click adds `Pitch Camp Here` on passable tiles ≥ `MIN_PITCH_DISTANCE` from packed centroid (emits `PitchCamp { tile, z }`). Between Pack and Pitch, members default to "Awaiting Orders" (strict `PackedMigrationAutonomy::Hold`); HUD's `Autonomy: Hold/Forage` toggle (mirrored in Migration panel) flips to `Forage` for autonomous food/sleep/social/scout/play. `PackCamp` resets to `Hold`.
