# UI (`src/ui/`)

All UI uses `bevy_egui` (not `bevy_ui`, except specific overlays).

- **Right-click context menu (`orders.rs`):** Three sections — tile actions (Move/Mine/Gather/Dig Down/Deconstruct/Build), entities on tile (Attack / Pick up corpse), ground item stacks. **Event-only**: `right_click_context_menu_system` emits `PlayerCommandEvent` (`simulation/player_command.rs`); sim attaches `Commanded` and dispatches. **Multi-select broadcast**: menu renders against `SelectedEntity` (focus) but dispatch broadcasts to every non-drafted player-faction worker in `SelectedEntities` (drafted units stay under `military_right_click_system`). Adding an order: new `PlayerCommand` variant, arm in `dispatch_one`, arm in `player_command_lifecycle_system`, UI button emits.
  - **Vehicle Yard ▸** submenu — right-click a player `VehicleYard` lists stock designs as `Assemble <name>` → faction-level `QueueVehicle`.
  - **Vehicle ▸** submenu — right-click a player `Vehicle` (or select one + click destination) lists Move/Load/Unload/Assign Crew/Hitch/Unhitch/Salvage (overturned hulk: only Right/Salvage). `build_vehicle_ops` builds the list; each entry emits `PlayerCommand::VehicleOrder { vehicle, kind }`.
- **Vehicle designer (`vehicle_designer.rs`):** `egui::Window` toggled by HUD "Vehicles" (`VehicleDesignerState`). Three columns over `10×8×6` grid. Left: `EditMode::{Cell, Module}` toggle, part palette + variant/material ComboBoxes (Cell), weapon-module list + rotation (Module). Centre: Z-tabs + `10×8` slice (left-click place, right-click clear). Right: live `derive_stats` / `validate_grid` / `design_bill`, capability summary, tech-gate line, inset preview canvas via `entity_sprites::vehicle_sprite_plan(design, heading)` with N/E/S/W rotation buttons. Queue button disabled when invalid or player lacks tech; emits `PlayerCommandEvent { command: QueueCustomVehicle { name, grid, purpose, required_animals } }`. "Saved & proposed designs" `CollapsingHeader` lists player-faction designs + AI proposals + on-disk `user_*.ron` (`from_user_file`) with Load buttons. **Save to disk** via `vehicle::save_custom_design` → `assets/data/vehicles/user_<slug>.ron`. **Debug-only Test Drive** (`cfg!(debug_assertions)`) emits `PlayerCommand::DebugSpawnTestVehicle`.
- **Manual drive (`manual_drive.rs`):** debug Test-Drive loop. `ManualDriveState { active, last_status }` (in `simulation::vehicle`). `manual_drive_input_system` (Update) reads `just_pressed`: W/Up = step, Q/E = forward-diagonal, A/D/Left/Right = turn in place, S/Down = clear in-flight `VehiclePathFollow`, Esc = release. Egui keyboard focus suppresses all keys; movement keys drop while path in flight (tap-to-step). Paths via `vehicle::plan_manual_step` (same `cell_ok` rules as `plan_vehicle_route`). `rendering::camera::camera_input_system` short-circuits WASD pan while active. **`PlayerPiloted` marker** (set in Test-Drive arm of `drain_player_command_events_system`, removed on Esc) excludes vehicle from `htn_vehicle_haul_dispatch_system` and right-click `VehicleOrderKind::MoveTo` so autonomous writers don't overwrite the player's `VehiclePathFollow`.
- **Selection (`selection.rs`):** `selection_input_system` picks `Person` + `Vehicle`. Single-click = globally-nearest view-space hit (Person within `SINGLE_CLICK_RADIUS_PX = 12`, Vehicle × 1.5). Drag-rect collects player Persons + Vehicles into `SelectedEntities.ids`; inspector renders per-entity-kind panel.
- **Inspector — Vehicle section (`inspector.rs`):** when selection is `Vehicle`, renders a single "Vehicle" `CollapsingHeader` (design name, state, footprint, mass, speeds, stability, turn radius, cargo, driver, hitched count, route progress, overturned warning) and returns early. Backed by `VehicleInspectorParams` (nested in `TaskDisplayParams` for param ceiling).
- **Inspector (`inspector.rs`):** Drop/unequip/equip via `PendingInspectorAction`; `inspector_action_system` executes after panel render. `valid_equip_slots(good)` (`../simulation/items.rs`) maps goods to slots. Knowledge section lists Learned/Aware with Lecture/Encode/Read buttons; sub-section lists `study_progress`. **Phase I**: Beliefs subsection at the bottom of Knowledge reads `PersonKnowledge.belief: AHashMap<BeliefGroupId, BeliefState>` and renders one row per group (Cosmology / Disease / Omens) — accepted belief name in truth-status colour, confidence as a 0–100% progress bar, plus an italic "rejected: ..." line listing prior beliefs the agent has cycled through. **Task & State** reads `MethodHistory` + `SimClock`: `Method:` falls back to `(direct dispatch — no HTN method)` for bureaucrats/traders/player orders, or `(none — see HTN history)` when idle; idle agents show `ticks_idle` + `last_goal_eval_tick`. **Profession line** inlines apprenticeship + mentor; masters show `mentoring #N`. **Skills section** prints `cur / peak P (floor F)` via `skills::skill_floor(peak)`; mastered (peak ≥ `SKILL_MASTERY_LINE`) render green. **Wage & Labor section** (between Stats and Knowledge): 24-hour earnings rollup, per-profession EV table via `capital::capital_factor`, own-faction `wage_signal` top-6, cross-faction `PerceivedFactionWages` top-6 with observation-age. Bundled in `WageInspectorParams`.
- **Knowledge panel (`tech_panel.rs`, `KnowledgePanelOpen`):** HUD "Knowledge" button toggles. Per-tech icon mirrors `AdoptionStage` — ◆ Institutionalized / ✓ Adopted / ● Practiced / ◐ Demonstrated / ◉ Rumored / ◎ Discoverable / ○ Locked. Hover shows stage + chief Learned/Aware + member tally. **Phase I**: top-level Beliefs section reads chief's accepted Cosmology / Disease / Omens belief with truth-status colour (green=True, yellow=FalseUseful, red=FalseHarmful, purple=Contested). Per-era tech list filters `KnowledgeKind::Belief` entries out — beliefs surface in the Beliefs section only.
- **Activity log (`activity_log.rs`):** Bottom-right panel; `ActivityLogEvent { tick, actor, faction_id, kind }`. Kinds: `Constructed`, `Crafted`, `TechInsight`, `TechDiscovered`, `KnowledgeAdopted { id }` / `KnowledgeInstitutionalized { id }` (from `derive_tech_adoption_system`; renderer reads `knowledge_def(id).kind()` for kind-aware verb — skill "adopted"/"institutionalized", technique "rolled out"/"standardized", lore "recorded"/"preserved", belief "now believes"/"now holds sacred"), `RegionSettled`, `Taught`, `Read`, `CampMoved`, `Sedentarized`, `WagePaid`, `ProfessionChanged { from, to }` (emitted by `profession_choice::profession_change_log_system`, Economy — observes `Changed<Profession>` via `LastSeenProfession` shadow), `ApprenticeshipStarted { mentor }`, `ApprenticeshipGraduated`, plus diplomacy: `TreatyFormed { with_faction, treaty }`, `TreatyBroken { with_faction, treaty }`, `TrespassWarning { intruder_faction, tile }`, `DiplomacyProposalReceived { from_faction, proposal }`. Emitter suppresses `None↔Apprentice` / `Apprentice→Crafter`. Player-only (filter by `faction_id == player_faction`); cap 16.
- **Spawn-select (`spawn_select.rs`):** Full-screen biome map at `WORLD_MAP_OVERSAMPLE=4` × climate-cell density via per-pixel `biome::classify_surface_at_tile`, hillshade `0.55 + 0.95*elev_f`. Uploaded LINEAR for smooth feathered borders. Two grid overlays (climate every 4 cells, mega-chunk). Hover tooltip: mega-chunk coords, centre tile, dominant biome (8×8 surface-classify sample), centre + min/mean/max elevation (4×4 via `sample_elevation_stats`), avg fertility (8×8 via `region::average_fertility_in_megachunk`), temperature. Click habitable mega-chunk → `PendingSpawn`; home is constrained by `region::pick_player_home_in_megachunk`. Side panel: era/pop/economy/lifestyle + **World** section with `Seed:` + Apply/Reroll firing `RegenerateWorldRequest` (handled by `world::regenerate_world_system`).
- **World-map switcher (`world_map.rs`):** Settled mega-chunks outlined (yellow=player, red=other). `WORLD_MAP_OVERSAMPLE=4` per-pixel `biome::classify_surface_at_tile`, LINEAR. Click bookmarks current camera + jumps to target's `camera_bookmark`. Tab toggles. **Fertility overlay** (toggle): `WorldMapView.show_fertility` tints by `region::average_fertility_in_megachunk` (brown→tan→green, 65% alpha) before the river pass; cache keys on the toggle. **Territory overlay** (toggle): `WorldMapView.show_territory` paints each mega-chunk by its dominant `TerritoryMap` owner using `territory_tint_for_faction` (deterministic per-id RGB lift, ~50% alpha). Sparse — only mega-chunks containing claimed tiles render; abstract-faction extents continue to surface via the settled-region outlines. Hover tooltip shows mega-chunk coords, centre biome, avg fertility.
- **Diplomacy panel (`diplomacy_panel.rs`):** HUD "Diplomacy" button toggles `DiplomacyPanelOpen`. Centre-anchored two-column window. Left: scrollable known-foreign-factions list with one-word `attitude_label` + treaty badges (e.g. `Faction 4 — Friendly [Trade+NAP]`). Right: selected faction's reputation tracks, last-8 incident log, pending-proposal cards (Accept / Reject), action buttons (Offer Trade Pact / Offer Alliance / Offer Non-Aggression / Declare War, or Offer Peace when at war). Buttons greyed when treaty already in effect; Break buttons appear for each active non-War treaty. All actions route through `CommandSender` as `PlayerCommand::{SendDiplomacyProposal | RespondDiplomacyProposal | DeclareWar | BreakTreaty}` (faction-level, validated against `ControlledFactions` at the loopback boundary).
- **Speed controls (`hud.rs` + `simulation/speed.rs`):** `⏸ / 1× / 2× / 5×` write `GameSpeed.current`; PreUpdate sync mirrors onto `Time<Virtual>`. Keys: `Space` pause toggle; `1/2/3` → Normal/Fast/VeryFast (gated on `!EguiContexts::wants_keyboard_input`). Debug panel's "Sim Timing" shows ticks/frame + EMA tick CPU + worst-tick window; reddens when avg > `preset.budget_ms_per_tick()`.
- **Muster button (`hud.rs`):** Emits `PlayerCommandEvent { actors: <player Hunters>, command: Muster }`; dispatch inserts `Drafted`, clears plan/reservations, removes `Carrying`. `apply_draft_toggle_system` (HUD button + `R`) emits Muster/Disband based on selection state. Orthogonal to chief muster (`HuntOrder::Hunt::mustered`).
- **Pack/Pitch Camp (`hud.rs` + `orders.rs`):** Nomadic player factions show `Pack Camp` when `camp_state == Pitched` (emits `PackCamp` on chief). When `Packed`: HUD reads "Mobile (right-click → Pitch)", right-click on passable tile ≥ `MIN_PITCH_DISTANCE` from packed centroid emits `PitchCamp { tile, z }`. Between Pack/Pitch, members default to `PackedMigrationAutonomy::Hold` ("Awaiting Orders"); HUD's `Autonomy: Hold/Forage` toggle (mirrored in Migration panel) flips to Forage. `PackCamp` resets to Hold.
