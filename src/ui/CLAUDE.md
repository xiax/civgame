# UI (`src/ui/`)

All UI uses `bevy_egui` (not `bevy_ui`, except specific overlays).

- **Right-click context menu (`orders.rs`):** Three sections — tile actions (Move/Mine/Gather/Dig Down/Deconstruct/Build), entities on tile (Attack / Pick up corpse), ground item stacks (Pick up: Nx Good). `PlayerOrderKind::{PickUpItem, AttackEntity, PickUpCorpse}` route to Scavenge/MilitaryAttack/PickUpCorpse tasks. `TileDisplayQueries` and `RoutingResources` are `SystemParam` bundles (Bevy 16-param limit). **Move orders dispatch via `tasks::apply_move_order_system` (Sequential)** — UI only inserts the `PlayerOrder { Move, … }` marker; the simulation system fires on `Changed<PlayerOrder>` and calls `assign_task_with_routing`. One-tick latency; programmatic insertions get the same dispatch.
- **Inspector (`inspector.rs`):** Drop / unequip / equip queue actions via `PendingInspectorAction`; `inspector_action_system` executes after panel render. `valid_equip_slots(good)` (`../simulation/items.rs`) maps goods to slots. Knowledge section lists Learned/Aware sets and capacity. Each Learned row has Lecture/Encode buttons (`HoldLecture`/`EncodeTablet`); Aware-only rows with a matching tablet/book in inventory show Read (`ReadItem`). "In progress" sub-section lists `study_progress` as `tech: progress/threshold ticks`. **Task & State** reads `MethodHistory` + `SimClock`: `Method:` falls back to `(direct dispatch — no HTN method)` when `aq.current != Idle` but `active_method` is None (bureaucrats / traders / player orders), or `(none — see HTN history)` when fully idle. When `task_id == UNEMPLOYED && aq.current == Idle`, an orange `Idle:` line shows `ticks_idle`, `last_goal_eval_tick`, and the most recent in-TTL `MethodHistory` entry. A collapsible **HTN history** sub-section colors entries by outcome.
- **Tech panel (`tech_panel.rs`):** Tri-state per tech — ✓ green (chief Learned), ◉ teal (chief Aware only), ◎ yellow (discoverable), ○ gray (locked). Hover row: "Complexity: N pt(s) · Members: N/M learned, N aware".
- **Activity log (`activity_log.rs`):** Bottom-right egui panel; `ActivityLogEvent { tick, actor, faction_id, kind }`. Kinds: `Constructed`, `Crafted`, `TechDiscovered`, `RegionSettled`, `Taught`, `Read`. Filtered to player faction; cap 16 entries.
- **Spawn-select (`spawn_select.rs`):** Full-screen biome map (rendered at `WORLD_MAP_OVERSAMPLE=2` × climate-cell density via per-pixel `biome::classify_at_tile`, so map biomes match in-game terrain instead of blocky cell colours). Two grid overlays: faint climate-cell sub-grid (every 4 cells) + bold mega-chunk grid. Click a habitable mega-chunk sets `PendingSpawn`. Ocean/Mountain non-habitable. Side panel hosts era / pop / economy / lifestyle controls plus a **World** section with editable `Seed:` field and Apply/Reroll buttons; both fire `RegenerateWorldRequest` (handled by `world::regenerate_world_system`) which rebuilds `Globe` + `WorldGen` and clears the `SpawnSelectTexture` / `WorldMapTexture` caches.
- **World-map switcher (`world_map.rs`):** Settled mega-chunks outlined (yellow=player, red=other). Renders at `WORLD_MAP_OVERSAMPLE=2` for the same sub-cell biome detail. Click bookmarks current camera onto its region, then jumps to target's `camera_bookmark`. Tab toggles map.
- **Muster button (`../simulation/military.rs`):** Sets `MusterHuntersRequest.pending`; `apply_muster_hunters_system` (Economy) inserts `Drafted` on every player-faction Hunter, clearing plan/reservations and removing `Carrying`. Player issues rally point separately via right-click. Orthogonal to chief muster (uses `HuntOrder::Hunt::mustered`, not `Drafted`).
