# Economy (`src/economy/`)

Resource catalog, goods, items, carrying, markets, recipes, currency primitives.

## Resource catalog

- **`ResourceCatalog` (`resource_catalog.rs`):** data-driven registry, loaded from `assets/data/resources/*.ron` at `WorldPlugin::build`. Each `ResourceDef` carries `key`, `display_name`, `class`, `bulk`, `weight_g`, optional `edible_calories`/`plantable_as`, `entertainment_value`, `storage_class`, `trade_base_value`, free-form `tags`, optional `sprite_key` (consumed by `entity_sprites::spawn_ground_item_sprites`). Indexed by `ResourceId(u16)` (deterministic across runs via alphabetical sort of `key`).
- **`core_ids` (`core_ids.rs`):** pre-resolved `OnceLock<ResourceId>` per legacy `Good` variant. Hot paths read `core_ids::WOOD.get().copied().unwrap()`. The catalog also lives as a process-global `OnceLock` (`core_ids::catalog()`) for `Good::*` methods that can't take system params; `core_ids::install_catalog(catalog)` runs in `WorldPlugin::build`.
- **`Good` enum is gone.** Storage, hauling, crafting, gathering, deposit slots, faction storage rollup, market prices, agent inventory accessors, item construction, equipment validity, build recipes, planting seeds, corpse yields, scavenging — all `ResourceId`-canonical. Adding a new resource = one RON entry in `core.ron` + optional sprite; no Rust changes for non-AI code.
- **`ResourceId` accessors:** `bulk()`, `unit_weight_g()`, `nutrition()`, `entertainment_value()`, `is_edible()`, `is_seed()`, `class()` — all catalog-backed, lazy. `core_ids::wood()`, `stone()`, etc. are snake-case accessor fns generated by the `core_ids!` macro. Constructor sites use the lazy fns; hot loops cache locally.
- **All keyed by `ResourceId`:** `EconomicAgent`, `Carrier`, `Item`, `Equipment`, `Bulk`, `GoodNeed`, `JobProgress`, `CraftRecipe`, `BuildRecipe`, `Market`, `FactionStorage`. `Item::new_commodity` / `new_manufactured` take `impl Into<ResourceId>`. `compute_combat_stats` keys on `core_ids::Weapon / Shield / Armor`. `valid_equip_slots(id)` reads catalog `class`. `BuildRecipe` is a lazy `OnceLock<Vec<...>>` mirroring `CraftRecipe`.
- **Recipes (`ResourceId`-keyed):** `CraftRecipe.inputs: Vec<(ResourceId, u32)>`, `output_resource: ResourceId`. Exposed via `craft_recipes() -> &'static [CraftRecipe]`.
- **`FactionStorage`** keys `totals`, `resource_supply`, `resource_demand`, `material_targets`, `material_deficit_ema` on `ResourceId`. Helpers: `stock_of`, `supply_of`, `demand_of`, etc.
- **`Market` (`ResourceId`-keyed):** `prices`, `bids_cleared`, `bids_stockout`, `bids_unaffordable`, `price_floor`, `market_stock` are `AHashMap<ResourceId, f32>` (sparse). **Bid-driven price discovery** — each tick, `signal = (stockout − unaffordable) / (cleared + stockout + unaffordable)` nudges price by up to ±5%; quiet markets (no buyer activity) leave price unchanged. No upper ceiling — natural cap is buyer affordability, since runaway price floods `unaffordable` and flips the signal. Sales never push price (sellers don't crash their own market — autumn glut alone doesn't drop prices below cost). `Market::default` seeds 22 legacy goods with `(base_price, floor=0.1)`; `SettlementMarket::default` is empty (implicit price `1.0` until trade). Test/admin shortcut: `set_price(id, p)` forces a posted price. Fields private — call sites use helpers (`price_of`, `stock_of`, `set_stock`, `set_price`, `add_bid_cleared`, `add_bid_stockout`, `add_bid_unaffordable`, `sell_item`, `try_buy_item`).
- **Adding a resource:** add a `ResourceDef(...)` to `assets/data/resources/core.ron` (or new `*.ron` in same dir).

## Carrying & item routing

- **Carrier (`../simulation/carry.rs`):** Two hand slots, separate from `EconomicAgent.inventory`. `Bulk` class — `TwoHand` (Wood, Stone, Iron) needs both hands empty; `OneHand`/`Small` need one. `enforce_hand_state_system` (before gather/dig) consults `gather_target_yield_bulk` to upgrade requirements when a yield is `TwoHand`. `Carrier::pickup_capacity(item)` mirrors `try_pick_up` for resolver pre-sizing.
- **Withdraw routes through hands first** (`production.rs::withdraw_material_task_system`): Stone weighs 5000g (= entire inventory cap), so storage withdraws offer to `Carrier::try_pick_up` first, then `EconomicAgent::add_good` for leftovers. Resolver sums `pickup_capacity + (inv_room/unit_w)`; if 0 it picks a different tile.
- **Eat from hands too** (`eat_task_system`): foraged Fruit/Meat/Grain are `Bulk::Small` and usually live in hands. `total_edible(agent, carrier)` covers both stores.
- **Equipment (`item.rs`, `../simulation/items.rs`, `../simulation/combat.rs`):** Combat stats on `Item` (`weapon_stats`, `armor_stats`), built by `Item::new_manufactured(good, material, quality)`. `Equipment::items: HashMap<EquipmentSlot, Item>` (value-typed, no entity layer). `ArmorStats::covered_parts` is a 4-bit bitmask over `armor_coverage::{HEAD, TORSO, ARMS, LEGS}` so `Item` stays `Copy`. `TaskKind::Equip` drives `equip_task_system` (Sequential, after `item_pickup_system`): pulls the highest-multiplier matching `Item` from inventory or `Carrier`, swaps slot; displaced occupant goes to inventory or ground via `spawn_or_merge_ground_item_full` (merges by full `Item` equality).

## Pluralist Economy primitives

Cross-cutting wiring lives in `simulation/`; the economy crate owns currency transfer, market state, and policy types. See `../simulation/CLAUDE.md` for consumer-side detail.

- **`pay(world, from, to, amount) -> bool`** (`transactions.rs`): atomic agent-to-agent currency transfer; the only sanctioned way. Refuses non-positive / insufficient / missing components.
- **`split_market_earnings_with_household(world, agent, earned)`** (`transactions.rs`): redirects `HOUSEHOLD_INCOME_SKIM` (10%) of trade earnings into the household sub-faction's treasury when the agent has `HouseholdMember`. Currency invariant preserved.
- **`trader_buy_at_settlement` / `trader_sell_at_settlement`** (`transactions.rs`): atomic trader primitives — debit/credit currency, mutate settlement treasury, mutate market stock + flow. Returns per-unit price paid.
- **`SettlementMarket`** (`market.rs`): same shape as the global `Market` minus the `Resource` impl. Lives on the `Settlement` component. Helpers: `calculate_price`, `sell_item`, `try_buy_item`, `set_stock` (trader fast path bypassing currency double-mutation), `clear_flow`.
- **`market_sell_system` / `market_buy_system`** (`transactions.rs`): route through the agent's faction's first settlement market when one exists; SOLO/unsettled fall back to the global `Market`. Disjoint borrows enable parallel scheduling.
- **`settlement_price_update_system`** (`market.rs`): Economy schedule, alongside `price_update_system`. Walks every Settlement, ticks `update_prices`, clears bid counters. No synthetic baseline demand — prices respond only to realized buyer-attempt outcomes. **Both run every `PRICE_UPDATE_INTERVAL=5` ticks** (4 Hz) — bid signals accumulate over the window, trajectory matches per-tick at 1/5 the cost.
- **`ResourceControlPolicy`** (`policy.rs`): composable per-resource flag bag — `chief_allocates_labor`, `private_actors_allowed`, `state_sells_at_market`, `prices_fixed_by_state`, `fixed_price`. `Default` = all-communist; `capitalist()` preset. Communism / Capitalism / Wartime / Mixed are *combinations*, not enum variants.
- **Global `Market` resource is retained** as the SOLO/unsettled fallback. `ui/economy_panel.rs` falls back to it when the player's faction has no settlement.
