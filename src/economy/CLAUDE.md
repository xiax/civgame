# Economy (`src/economy/`)

Resource catalog, goods, items, carrying, markets, and craft recipes.

## Resource catalog

- **`ResourceCatalog` (`resource_catalog.rs`):** Data-driven registry of every resource. Loaded from `assets/data/resources/*.ron` at `WorldPlugin::build`. Each `ResourceDef` carries `key` (stable snake_case id), `display_name`, `class`, `bulk`, `weight_g`, optional `edible_calories`/`plantable_as`, `entertainment_value`, `storage_class`, `trade_base_value`, free-form `tags`, and an optional `sprite_key` (consumed by `entity_sprites::spawn_ground_item_sprites` for ground-pile rendering — pinned for every legacy good by `every_legacy_good_has_a_sprite_key`). Indexed by stable `ResourceId(u16)` (alphabetical sort of `key` ⇒ deterministic IDs across runs).
- **`core_ids` (`core_ids.rs`):** Pre-resolved `OnceLock<ResourceId>` per legacy `Good` variant, populated by `init_core_ids(catalog)` at startup. Hot paths read `core_ids::WOOD.get().copied().unwrap()` rather than re-querying the catalog. The catalog also lives as a process-global `OnceLock` (`core_ids::catalog()`) for `Good::*` methods that can't take system params; `core_ids::install_catalog(catalog)` is called from `WorldPlugin::build`. Module disappears once the `Good` enum does.
- **`Good` enum is GONE (Phase 2 residual #7 complete, 2026-05-06):** Storage, hauling, crafting, gathering, deposit slots, faction storage rollup, market prices, agent inventory accessors, item construction, equipment validity, build recipes, planting seeds, corpse yields, scavenging — every code path is `ResourceId`-canonical. No more `Good::X.into()` boundary hops, no `good_to_resource_id` / `resource_id_to_good` reverse-resolves, no per-`Good` accessor methods. `goods.rs` shrank from 67 `Good::` references to zero; only the `Bulk` enum + its catalog-keyed `Bulk::for_resource(id, catalog)` accessor remain. Adding a new resource = one RON entry in `core.ron` plus an optional sprite; no Rust changes for any non-AI code path.
- **`ResourceId` accessors:** `ResourceId` carries `bulk()`, `unit_weight_g()`, `nutrition()`, `entertainment_value()`, `is_edible()`, `is_seed()`, and `class()` — all catalog-backed, lazy-loaded on first call. `core_ids::wood()`, `core_ids::stone()`, etc. are the snake-case accessor functions for founding resources, generated by the `core_ids!` macro alongside the per-resource `OnceLock` static. Constructor-syntax sites (plan registry, market default seed, build recipes) use the lazy fns; hot loops cache the `*Wood.get().unwrap()` deref locally.
- **`EconomicAgent`, `Carrier`, `Item`, `Equipment`, `Bulk`, `GoodNeed`, `JobProgress`, `CraftRecipe`, `BuildRecipe`, `Market`, `FactionStorage`** are all `ResourceId`-keyed. `Item::new_commodity` / `new_manufactured` take `impl Into<ResourceId>`. `compute_combat_stats` keys on `core_ids::Weapon / Shield / Armor`. `valid_equip_slots(id)` reads catalog `class` instead of switching on `Good`. `faction.rs`'s crafted-drop classifier and `recompute_inventory_capacity_system`'s torso bonus key on `ResourceClass` matches. `BuildRecipe` is a lazy `OnceLock<Vec<...>>` table mirroring `CraftRecipe`'s pattern.
- **Recipes are `ResourceId`-keyed:** `CraftRecipe.inputs: Vec<(ResourceId, u32)>` and `CraftRecipe.output_resource: ResourceId`. The recipe table is exposed via `craft_recipes() -> &'static [CraftRecipe]` backed by a lazy `OnceLock<Vec<...>>`. `craft_order_system` constructs the output via `Item::new_commodity(recipe.output_resource)` / `Item::new_manufactured(recipe.output_resource, mat, quality)` directly — both constructors take `impl Into<ResourceId>` since Phase 2 #7 sub-PR (b). `CraftRecipe::output_good_legacy()` survives only as a test-pin helper that asserts every recipe's output reverse-resolves to a legacy `Good`; production code no longer goes through it.
- **Faction storage is `ResourceId`-keyed:** `FactionStorage.totals`, `FactionData.resource_supply`, `FactionData.resource_demand`, `FactionData.material_targets`, and `FactionData.material_deficit_ema` are all `AHashMap<ResourceId, _>`. Lookup helpers `FactionStorage::stock_of(id)` and `FactionData::{supply_of, demand_of, material_target_of, material_deficit_ema_of}(id)` take `ResourceId` directly since sub-PR (d-9); legacy `Good`-typed callers convert at the boundary via `good.into()`. New code should index these maps with `ResourceId` directly.
- **Market is `ResourceId`-keyed:** `Market.{prices,supply,demand,price_floor,price_ceiling,market_stock}` are `AHashMap<ResourceId, f32>` (sparse — entries materialise on first trade). `Default` seeds the legacy 22 goods with their original price/ceiling values; a fresh resource added to the catalog gets `price=1.0`, `floor=0.1`, `ceiling=∞`, `stock=0` until traded. Helpers `price_of(good)`, `stock_of(good)`, `add_supply(good, qty)`, `add_demand(good, qty)` accept legacy `Good` and reverse-resolve via `core_ids`. `update_prices` walks the union of price/supply/demand keys so untraded resources don't churn. The fields are private — call sites use the helpers (or `calculate_price(item)` for manufactured items).
- **Adding a new resource:** add a `ResourceDef(...)` block to `assets/data/resources/core.ron` (or a new `*.ron` file in the same dir). Until the `Good` enum is fully retired, also add a `Good` variant + per-method arms to keep `Good`-typed code paths working — but `Item` itself no longer needs touching for new resources.

## Carrying & item routing

- **Carrier (`../simulation/carry.rs`):** Two hand slots, separate from `EconomicAgent.inventory`. `Bulk` class — `TwoHand` (Wood, Stone, Iron) needs both hands empty; `OneHand`/`Small` need one. `enforce_hand_state_system` runs before gather/dig and consults `gather_target_yield_bulk` to upgrade requirements when a harvest yield is `TwoHand`. `Carrier::pickup_capacity(item)` mirrors `try_pick_up` for resolver pre-sizing.
- **Withdraw routes through hands first** (`../simulation/production.rs::withdraw_material_task_system`): Stone weighs 5000g — same as the entire inventory cap — so storage withdraws offer to `Carrier::try_pick_up` first, then `EconomicAgent::add_good` for leftovers. Resolver sums `pickup_capacity + (inv_room/unit_w)`; if 0 it picks a different tile.
- **Eat from hands too** (`../simulation/production.rs::eat_task_system`): Foraged Fruit/Meat/Grain are `Bulk::Small` and usually live in hands. `total_edible(agent, carrier)` covers both stores; `requires_any_edible` precondition uses the same.
- **Equipment unification (`item.rs`, `../simulation/items.rs`, `../simulation/combat.rs`):** Combat stats live on `Item` (`weapon_stats`, `armor_stats`), built by `Item::new_manufactured(good, material, quality)`. `Equipment::items: HashMap<EquipmentSlot, Item>` (value-typed, no entity layer). `ArmorStats::covered_parts` is a 4-bit bitmask over `armor_coverage::{HEAD, TORSO, ARMS, LEGS}` so `Item` stays `Copy`. `TaskKind::Equip` (StepDef 56) drives `equip_task_system` (Sequential, after `item_pickup_system`): pulls highest-multiplier matching `Item` from inventory or `Carrier`, swaps slot, displaced occupant goes to inventory or ground via `spawn_or_merge_ground_item_full` (merges by full `Item` equality so manufactured stats survive storage round-trip).
