# Economy (`src/economy/`)

Resource catalog, goods, items, carrying, markets, recipes, currency primitives.

## Resource catalog

- **`ResourceCatalog` (`resource_catalog.rs`):** data-driven registry, loaded from `assets/data/resources/*.ron` at `WorldPlugin::build`. Each `ResourceDef`: `key`, `display_name`, `class`, `bulk`, `weight_g`, optional `edible_calories` / `plantable_as` / `entertainment_value` / `storage_class`, `trade_base_value`, free-form `tags`, optional `sprite_key`. Indexed by `ResourceId(u16)` (deterministic alphabetical sort).
- **`core_ids` (`core_ids.rs`):** pre-resolved `OnceLock<ResourceId>` per legacy `Good` variant. Hot paths read `core_ids::WOOD.get().copied().unwrap()`. Process-global catalog `OnceLock` via `core_ids::catalog()`; installed by `install_catalog(catalog)` in `WorldPlugin::build`.
- **`Good` enum is gone.** Storage, hauling, crafting, gathering, deposits, faction storage rollup, market prices, inventory, items, equipment, recipes, planting seeds, corpse yields, scavenging — all `ResourceId`-canonical. Adding a resource = one RON entry in `core.ron` (+ optional sprite); no Rust changes for non-AI code.
- **Accessors:** `bulk()`, `unit_weight_g()`, `nutrition()`, `entertainment_value()`, `is_edible()`, `is_seed()`, `class()` on `ResourceId` (catalog-backed, lazy). `core_ids::wood()`/`stone()`/etc generated by `core_ids!` macro.
- **Catalog-derived sets:** `core_ids::edibles() -> &'static [ResourceId]` (every resource with `edible_calories`); `core_ids::min_edible_calories() -> u32` (lowest edible calorie value, conservative deficit-target unit). Both extend automatically when a new edible lands.
- **All keyed by `ResourceId`:** `EconomicAgent`, `Carrier`, `Item`, `Equipment`, `Bulk`, `GoodNeed`, `JobProgress`, `CraftRecipe`, `BuildRecipe`, `Market`, `FactionStorage`. `Item::new_commodity` / `new_manufactured` take `impl Into<ResourceId>`. `compute_combat_stats` keys on `core_ids::Weapon/Shield/Armor`. `valid_equip_slots(id)` reads catalog `class`. `BuildRecipe` is a lazy `OnceLock<Vec<...>>` mirroring `CraftRecipe`.
- **Recipes:** `CraftRecipe.inputs: Vec<(ResourceId, u32)>`, `output_resource: ResourceId`. `craft_recipes() -> &'static [CraftRecipe]`.
- **`FactionStorage`** keys `totals`, `resource_supply`, `resource_demand`, `material_targets`, `material_deficit_ema` on `ResourceId`. Helpers: `stock_of`, `supply_of`, `demand_of`.
- **`Market` (`market.rs`):** `prices`, `bids_cleared`, `bids_stockout`, `bids_unaffordable`, `price_floor`, `market_stock` are sparse `AHashMap<ResourceId, f32>`. **Bid-driven price discovery** — each tick `signal = (stockout − unaffordable) / (cleared + stockout + unaffordable)` nudges price ±5%; quiet markets unchanged. No upper ceiling — runaway floods `unaffordable` and flips signal. Sales never push price. `Market::default` seeds 22 legacy goods at `(base_price, floor=0.1)`; `SettlementMarket::default` empty (implicit 1.0). Fields private — use helpers (`price_of`, `stock_of`, `set_stock`, `set_price`, `add_bid_*`, `sell_item`, `try_buy_item`). `listings: Vec<(Item, u32)>` pruned in two places: `try_buy_inner` `swap_remove`s zero-qty rows; `price_update_system` / `settlement_price_update_system` / `camp_price_update_system` run `retain(|(_, q)| *q > 0)` every `PRICE_UPDATE_INTERVAL = 5` ticks. `sell_item` early-returns on `qty == 0`.

## Carrying & item routing

- **Volume + weight, per-bucket caps.** Every `ResourceDef` carries `volume_ml: u32` alongside `weight_g`. `ResourceId::unit_volume_ml` / `Item::stack_volume_ml(qty)` mirror the weight helpers; volume is per-resource only (no material multiplier). Carrying capacity is gated by **both** weight AND a per-`Bulk`-bucket volume. `Bulk` is now purely slot-occupancy classification — quantities are physics-driven.
- **Carrier (`../simulation/carry.rs`):** Two hand slots, separate from `EconomicAgent.inventory`. `Bulk` class — `TwoHand` (Wood/Stone/Iron) needs both; `OneHand`/`Small` needs one. Per-hand caps: `HUMAN_HAND_CAP_G=25_000`, `HAND_SMALL_VOLUME_ML=2_000` (Bulk::Small pouches), `HAND_OVERSIZED_VOLUME_ML=100_000` (OneHand/TwoHand clutched loads). `HAND_QTY_CAP` is **removed** — qty is bounded by weight + volume only. `try_pick_up` / `pickup_capacity` stay in lock-step (regression-tested). `should_return_to_deposit(item)` ≡ `pickup_capacity(item) == 0` (item-keyed); `should_return_to_deposit_held()` covers gather catch-alls where the just-harvested item isn't in scope. `enforce_hand_state_system` (before gather/dig) upgrades via `gather_target_yield_bulk`.
- **`Carrier::batch_capacity_for(item, agent)`** sums hand pickup + the agent's inventory bucket matching `item.bulk()` — HTN dispatchers use it to size `Task::WithdrawMaterial.qty` by `min(stock, capacity)` (replacing the old hardcoded `qty: 1`).
- **`EconomicAgent` volume buckets:** `base_small_vol_ml=8_000` (small-bulk satchel) + `base_bulky_vol_ml=25_000` (lashed pack). `add_item` and `is_inventory_full` gate on weight AND the matching bucket. `free_vol_ml_for(item)` returns the per-item-bulk free volume.
- **Withdraw routes through hands first** (`production.rs::withdraw_material_task_system`): Stone weighs 5000g (= entire inventory cap), so withdraws offer `Carrier::try_pick_up` first, then `EconomicAgent::add_good` for leftovers. Resolver sums `pickup_capacity + (inv_room/unit_w)`; if 0, picks a different tile.
- **Eat from hands too** (`eat_task_system`): foraged Fruit/Meat/Grain are `Bulk::Small`, usually in hands. `total_edible(agent, carrier)` covers both stores.
- **Equipment (`item.rs`, `../simulation/items.rs`, `../simulation/combat.rs`):** Combat stats on `Item` (`weapon_stats`, `armor_stats`), built by `Item::new_manufactured(good, material, quality)`. `Equipment::items: HashMap<EquipmentSlot, Item>` (value-typed). `ArmorStats::covered_parts` is a 4-bit bitmask over `armor_coverage::{HEAD, TORSO, ARMS, LEGS}` so `Item` stays `Copy`. `TaskKind::Equip` drives `equip_task_system` (Sequential, after `item_pickup_system`): pulls highest-multiplier match from inventory/`Carrier`, swaps slot; displaced occupant → inventory or ground via `spawn_or_merge_ground_item_full`.

## Pluralist Economy primitives

Cross-cutting wiring lives in `simulation/`; this crate owns currency transfer, market state, policy types. See `../simulation/CLAUDE.md` for consumer-side detail.

- **`pay(world, from, to, amount) -> bool`** (`transactions.rs`): atomic agent-to-agent transfer; only sanctioned way. Refuses non-positive / insufficient / missing components.
- **`split_market_earnings_with_household(world, agent, earned)`** (`transactions.rs`): redirects `HOUSEHOLD_INCOME_SKIM` (0.10) of trade earnings into household sub-faction treasury when agent has `HouseholdMember`. Currency invariant preserved.
- **`trader_buy_at_settlement` / `trader_sell_at_settlement`** (`transactions.rs`): atomic trader primitives — debit/credit currency, mutate settlement treasury + market stock + flow. Returns per-unit price paid.
- **`SettlementMarket`** (`market.rs`): same shape as global `Market` minus `Resource` impl. Lives on `Settlement`. Helpers: `calculate_price`, `sell_item`, `try_buy_item`, `set_stock` (trader fast path), `clear_flow`.
- **`market_sell_system` / `market_buy_system`** (`transactions.rs`): route through agent's faction's first settlement market when one exists; SOLO/unsettled fall back to global `Market`. Disjoint borrows enable parallel scheduling.
- **`settlement_price_update_system`** (`market.rs`): Economy schedule alongside `price_update_system`. Walks every Settlement, ticks `update_prices`, clears bid counters. No synthetic baseline demand — prices respond only to realized buyer outcomes. Both run every `PRICE_UPDATE_INTERVAL = 5` ticks (4 Hz).
- **`ResourceControlPolicy`** (`policy.rs`): composable per-resource flag bag — `chief_allocates_labor`, `private_actors_allowed`, `state_sells_at_market`, `prices_fixed_by_state`, `fixed_price`. `Default` = all-communist; `capitalist()` preset. Communism / Capitalism / Wartime / Mixed are *combinations*, not enum variants.
- **`LandPolicy`** (`policy.rs`): faction-level land tenure flags — `state_sells_land`, `state_rents_land`, `state_sharecrops`, `private_freehold_allowed`, `default_lease_period_days`, `rent_yield_pct`, `default_share_to_landlord`. Default = all-false (Subsistence/communal — plots stay `StateOwned`). `land_policy_for(EconomyPreset)` flips Mixed → rent + sharecrop, Market → adds sale + freehold. Stored on `FactionData.land_policy`; populated in `spawn_population` alongside `apply_preset`. Distinct from `ResourceControlPolicy` because land is positional — see `simulation/land.rs` for `Plot` consumers.
- **Global `Market` resource is retained** as SOLO/unsettled fallback. `ui/economy_panel.rs` falls back to it when player faction has no settlement.
