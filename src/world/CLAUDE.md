# World (`src/world/`)

Procedural terrain, globe generation, climate, geology, spatial index. See root `CLAUDE.md` for coords/Z conventions and `SpatialIndex` discipline.

## Tile palette (`tile.rs`)

`TileKind` has 26 variants:
- **Surfaces:** `Grass`, `Forest`, `Sand`, `Snow`, `Marsh`, `Scrub`, `Water`, `River`, `Road`.
- **Stone lithologies** (`is_stone_like`): `Stone` (legacy), `Granite`, `Limestone` (yields 3 vs. 2), `Sandstone`, `Basalt`, plus underground `Wall` and `Ore`.
- **Soils** (`is_soil_like`): `Dirt`, `Loam` (1.5×), `Silt` (1.4×, riparian), `Clay`, `SandySoil` (0.6×), `Cropland` (1.3×).
- **`Bridge`** — passable, road-speed, reports `is_freshwater()` (water flows under decking).
- **`Dam`** — constructed barrier; passable + road-speed (crest carries a road) but **not** water-like/freshwater/drinkable. Durable truth is the `Dam` entity in `DamMap`; tile kind is its cache projection, restamped on chunk reload. Tech-gated on `DAM_BUILDING`.
- **`Cropland`** — tilled farm soil, only inside an Agricultural plot. Worked into existence by `simulation::farm::prepare_field_task_system`; `is_soil_like`, speed 0.9, never paved by road carving.

Helpers: `stone_yield_count`, `soil_fertility_mult`. No `Farmland` variant — Grain grows on `Cropland`. World-gen `TileData.fertility` is an immutable per-tile recovery ceiling; `simulation::farm::FieldTileIndex.by_tile[tile].nutrients` is the live nutrient pool. Pathing speeds: Sand 0.75, Snow 0.6, Marsh 0.4, Scrub 0.9, soils 0.85–0.9, stone 1.0.

## Water system

Real water columns + worldgen hydrology truth + persistent runtime water + dams + background fluid sim. Phase log + v2 deferrals in `plans/water-simulation.md`; this is the end state.

**Hydrology truth (`globe.rs`/`hydrology.rs`).** `Globe.hydrology: HydrologyMap { cells: Vec<HydroCell>, reservoirs: Vec<Reservoir> }`. `HydroCell { raw_height, filled_height, flow_to, discharge, reservoir_id, aquifer_level }`; `Reservoir { id, kind, spill_level, outlet_cell, salinity }`; `ReservoirKind { Ocean(0)|Lake|Wetland|Endorheic|Spring|Dam }` (Spring/Dam runtime). Pure fns: `weighted_discharge`, `strahler_order`, `classify_reservoirs` (closed → Endorheic/salinity 0.6, shallow → Wetland, else Lake; Ocean = id 0, salinity 1.0), `aquifer_table`, `build_hydrology`. Built in `generate_globe` step 7. Shared accessors `Globe::water_level_at/reservoir_at/salinity_at/hydro_cell_at` feed both chunk stamping and world-map overlay.

**Chunk water columns.** `Chunk` carries `surface_ground_z` (i8 bed), `surface_water_depth` (f32 Z-units, 0=dry), `surface_reservoir_id` (u32, MAX=none). `surface_z_at` returns rendered top (water surface for wet tiles); `ground_z_at` is the additive solid-bed accessor. Dry invariant `ground==surface, depth==0, rid==MAX`, re-asserted by `set_delta`. Accessors `ground_z_at`/`water_depth_at`/`water_level_at`/`reservoir_id_at`/`water_column_at`/`apply_water_column`. `generate_chunk_from_globe` stamps rivers `bed = water_surf - ceil(depth)` and reservoirs by basin (Lake/Endorheic→Water, Wetland→Marsh, at `spill_level·GLOBE_H_TO_Z`; Ocean→biome). Elevation comparisons use `ground_z_at`; exception: `land.rs::plot_value_factor` stays on `surface_z_at` (its `z` feeds `tile_at_3d` for kind/fertility).

**Persistent runtime water (`water_runtime.rs`).** Chunks regenerate from `Globe+seed` on stream-in (deltas not re-applied), so dam pools / sim writes would die on unload. `RuntimeWater` Resource — tile-keyed `RuntimeWaterCell { ground_z, depth, reservoir_id, salinity, source_rate }` + `dam_crests` + `runtime_reservoirs` — holds it off-chunk as a derived cache (only `world.bin`/Globe serialises). `update_chunk_retention_system` pins chunks under every dam + every cell with `depth>0 || source_rate>0`. `RuntimeWater::set` drops `depth<=0`. `restamp_runtime_water_on_chunk_load` (FixedUpdate, `.after(chunk_streaming_system)`, before PostUpdate `refresh_changed_tiles_system`) re-applies `RuntimeWater` columns and re-stamps `Bridge`/`Dam` tiles from `BridgeMap`/`DamMap` on each `ChunkLoadedEvent`. `simulation::well::restamp_wells_on_chunk_load` chains before it (same batch): well shaft/helix geometry is also a chunk delta and re-carves first, then the water restamp lays the column. Restamp uses `set_tile`.

**Dam (`construction.rs` + `tile.rs`).** `TileKind::Dam=25` (26 variants) — passable + road-speed (crest carries road) but not water-like/fresh/drinkable (contrast `Bridge`). `Dam` entity in `DamMap` is durable truth; tile kind is its cache projection. `BuildSiteKind::Dam` mirrors Bridge: 6 stone+4 wood / 180 work / `DAM_BUILDING`-gated (Bronze; prereqs `BRIDGE_BUILDING`+`MONUMENTAL_BUILDING`); `is_water_anchored()` covers `Bridge|Dam`; finalize stamps tile + entity + `RuntimeWater::register_dam(tile, crest_z)`; deconstruct (shared `water_anchored_refund_tile`) restores prior tile + `clear_dam` + refund. `FurnitureMaps` bundles `dam_map`+`runtime_water`. Right-click Build Dam on River/Water + AI-planned by `organic_settlement::dam_intent_emitter_system` (`DAM_BUILDING` + `CivicKind::Dam` Bronze+30, `DAM_MIN_SPACING=24`, one dam/settlement/cadence). `BuildRecipeIdx::Dam` appended last (stable indices).

**Background fluid sim.** `water.rs` is a pure, Bevy-free, deterministic virtual-pipe shallow-water core: `WaterGrid` of `Free`/`Pinned` cells + `dam_crests`. `step()` builds a sorted transfer list (cardinal pipe edges ∝ surface-Δ + dam-weir over-crest spill), per-giver volume-clamps, applies. Conserves volume exactly; bit-for-bit deterministic (unit-tested with basin-fill / dam pool overtop/drain). `water_runtime.rs` wraps it: `WaterSim` runs one `AsyncComputeTaskPool` task on a 20-tick cadence. `spawn` (PostUpdate) builds the active region — `WATER_SIM_RADIUS=28` box around every dam + `WATER_CELL_RADIUS=3` box around every persisted runtime cell — and self-terminates when both sets are empty. Takes a flat `RawTile` snapshot (kind/ground_z/water_depth/surface_z) and hands classify-and-simulate to the task pool, against the snapshot + cached `Arc<Globe>` (cloned once; Globe is fixed during Playing). All hydrology lookups, `dam_impoundment_set`, `edge_crossings_in_bbox` run off-thread. If the region exceeds `WATER_SIM_MAX_REGION_TILES≈30k`, a rotating cursor simulates a bounded window per cadence.

In the classify: ocean/lake/unloaded → Pinned at hydrology Z; per-cell flow routing via `RiverNetwork::edge_crossings_in_bbox` places inlets (inject `discharge`) / outlets (Pinned) at the true channel crossings. A free-flowing `TileKind::River` cell is Pinned at chunk-gen column surface; it free-evolves and its discharge inlet fires **only inside a dam impoundment** (`dam_impoundment_set` BFS-floods from each dam over connected cells with bed below crest). Well shafts project as `Water`, never `River` (fishing sees the `Water` projection but `simulation::fishing::tile_supports_fishery` excludes it by component-size — a 1-cell shaft can't be a fishing spot). A Free cell whose bed sits below `HydroCell.aquifer_level` seeps upward (springs + dug pits), capped at the table (`bed+depth<aquifer_z` only — snapshot-time gate, zero `water.rs` change). Dam footing → weir crest = footing + `DAM_RISE_Z=3`. Inflow follows snowmelt seasonal hydrograph `Calendar::discharge_multiplier` (Spring 1.5 / Summer 0.7 / Autumn 1.0 / Winter 0.25), damped for aquifer seep.

Truth gate is the per-tile natural table: `terrain::surface_height(tx,ty,...)` minus `(filled_height − aquifer_level) · GLOBE_H_TO_Z` (per-cell aquifer depth, ~0.16 Z wet to ~1.6 Z dry). Applied to every interior Free cell: bed below table ⇒ source; pool ≥ table ⇒ cap zeroes source. Natural depressions and dug pits treated identically. **Single shared formula (chunk-gen + runtime + wells):** `cell_surface_z = Z_MIN + (sample_climate(tx,ty).0/255) · CHUNK_HEIGHT`; `aquifer_depth_z = (filled_height − aquifer_level) · GLOBE_H_TO_Z`; below iff `bed_z < cell_surface_z − aquifer_depth_z`. `aquifer_table` calibrated at 1.5 m tile scale: `depth_raw = 0.0625 + 1.4375 · (1 − rainfall_norm)` → ~0.5 Z saturated lowland to ~12 Z true arid.

Chunk-gen Pass 4.5 (after reservoir basin stamp) applies the gate to every remaining dry tile: bed below ⇒ flip to `Marsh`, bed = original surface_z, surface = clamped `table_z`, depth ≥ `MIN_RIVER_DEPTH_Z`, `reservoir_id = u32::MAX`. `aquifer_seep_emitter_system` (PostUpdate, before `spawn`) is region-bootstrap: on `TileCarvedEvent` (from `dig_system`, not the broad `TileChangedEvent`) that clears the per-tile table, insert a depth-0 runtime cell so the sim's active region covers the dig. `poll` (PreUpdate) writes `RuntimeWater` (incl. `source_rate`) + live `ChunkMap`, restores natural kind on drain, emits `TileChangedEvent` only on wet/dry passability flip. Main tick never blocks. **Only disturbed cells (`WaterTileOut.disturbed`) are written back** — a cell already in `RuntimeWater` or inside a dam impoundment. Natural marshes that just sit in the active region are hydraulic context and discarded (Pass 4.5 owns their column). Dry-but-source-fed branch updates existing cells in place but never creates them — bounds `RuntimeWater.cells` and the active region.

**Salinity + wells (`biome.rs`/`drink.rs`).** `WaterKind::{Fresh,Brackish,Salt}`; `water_kind_at` reads `Globe::salinity_at` via pure `classify_salinity` (Endorheic brackish, ocean salt, rivers/open lakes fresh; River/Marsh always Fresh). `WaterKind::is_drinkable()` (Fresh-only) is the single salt/brackish rejection rule. Excavated wells (`simulation::well`): no virtual reach gate. A well is a dug 5×5 stepwell whose central shaft is carved one Z below the per-cell table (`well::aquifer_z_at`, same frame as Pass 4.5 + the fluid sim seep gate); `well_spec_at` rejects sites whose table exceeds `MAX_HAND_DUG_WELL_DEPTH_Z=8`. The shaft holds a physical `RuntimeWater` column — charged on completion, recharged by `AQUIFER_SEEP_RATE`, drawn per sip; `well_has_water` reads the column. Drained shaft → `DrinkOutcome::WellDry`, skipped by `nearest_well_tile`, refills from seep. Settlement/nomad/herd scoring rides `TileKind::is_freshwater()` / `river_distance_at` (unchanged; `Dam`=false, `Bridge`=true). Pathfinding rides `TileChangedEvent`.

## Locality (`locality.rs`)

Read-only site-context queries used by the knowledge-system overhaul (Phase D, `plans/knowledge-system-overhaul.md`). Pure of `Globe` + the canonical biome bands so chunk-gen, world-map preview, and locality stay in lock-step.

- `stone_kind_at(globe, tx, ty) -> Option<TileKind>` returns the biome's top stone slot when `ReliefClass::permits_stone_outcrop()`, else `None`.
- `forest_density_around` / `clay_proximity` / `wetland_proximity` aggregate a `(2r+1)²` window keyed on canonical `classify_at_tile` + `biome_bands` + `topsoil_kind`; outputs are `Proximity` (`u8`).
- `river_silt_proximity` saturates within `RIVER_PROXIMITY_SCALE=16` tiles of the nearest river; `reeds_available_within` is a wetland-adjacency boolean.
- `LocalSiteContext { biome, stone_kind, forest_density, clay, wetland, river_silt }` bundles all of the above. Cached on `Settlement.locality: Option<LocalSiteContext>` at spawn time by `auto_found_default_settlements_system`; hover tooltip (`ui/hover.rs`) recomputes per-cursor-tile for verification. No selection change yet — Phase E reads this to pick `BuildingTechnique`.

## Water currents (`water_current.rs`)

`WaterCurrentField` (Resource) — derived, non-persistent per-tile current map (Phase 3 of `plans/swimming.md`). `CurrentVector { dir, speed, source }`; `CurrentSource::{RiverChannel, StillWater}`. `River` tiles flow along local channel tangent — PCA of nearby `River` tiles, oriented downstream by coarse D8 `HydroCell.flow_to`; speed saturates from `HydroCell.discharge`. Every other wet tile reads `StillWater`. `water_current_field_system` (FixedUpdate, `.after(restamp_runtime_water_on_chunk_load)`) rebuilds a chunk's cells on `ChunkLoadedEvent` and drops them on `ChunkUnloadedEvent`, plus a first-run bootstrap for spawn-area chunks that pre-load without events. Deterministic. Consumed by `simulation::swimming` (current resist/assist) and `rendering::water_current_render`. v1: dam-pool surface gradient (`RuntimeFlow`) not distinguished; `TileChangedEvent` doesn't trigger rebuild.

## World generation

- **Terrain noise (`terrain.rs::surface_v`):** Globe-anchored. Macro = bilinear `Globe::sample_climate(tx,ty)` elevation; per-tile detail = 3-octave Perlin at **relief-conditional** amplitude `relief_detail_amp(ReliefSample)` driven by `Globe::sample_relief`. Plains/coastal/floodplain 0.015 (±0.5 Z); RollingHills 0.035; UplandPlateau 0.025; Foothills 0.070; Badlands 0.080; MountainSlope 0.130 (±4.2 Z, dramatic); MountainRidge 0.200 (±6.4 Z, deliberately saturates Z ceiling — ridges read as impassable cliff faces). `BasinWetland`/`OceanShelf` force `detail=0` (basin floors flat). MountainSlope/Ridge use ridged-multifractal noise on the same `gen.surface` instance (no new noise field). Continuous `local_relief` lift softens class boundaries.
- **Geomorphology layer (`geomorph.rs`, `Globe.relief`):** Per-climate-cell relief diagnostics computed once in `generate_globe` after `build_hydrology`. `ReliefCell { slope_norm, local_relief, topographic_position, coast_distance, mountain_distance, aquifer_depth_norm, relief: ReliefClass }`. `ReliefClass`: `OceanShelf / CoastalPlain / Floodplain / BasinWetland / LowlandPlain / RollingHills / UplandPlateau / Badlands / Foothills / MountainSlope / MountainRidge`. Classification reads `filled_height` (post pit-fill, post erosion), `aquifer_level`, reservoir membership, per-cell Strahler. `Globe::sample_relief(tx,ty)` bilinearly interpolates numerics + re-derives class so the per-tile class is continuous across cell boundaries (no 64-tile patches). Wet classes (Floodplain/BasinWetland) inherited from the closest corner cell when numerics still support that class. Drives: per-tile detail amp (`surface_v`), bare-rock outcrop gate (`ReliefClass::permits_stone_outcrop`, see Pass 1), per-tile fertility multiplier (`ReliefClass::fertility_mult`), settlement scoring (`pick_player_home_in_megachunk` + `score_home_candidate`), world-map/spawn-select tooltip (`dominant_relief_in_megachunk`).
- **Globe pipeline (`globe.rs::generate_globe`):** 512×256 climate-cell grid. (1) Lloyd-relaxed Voronoi over `NUM_PLATES=8` with domain-warped assignment (`plates::assign_nearest` offsets by two-octave Perlin warp, `WARP_FREQ=0.04`, `WARP_AMP=8`). Uplift/subsidence stamped along boundaries; post-smooth uplift gets ±12% Perlin jitter (`JITTER_FREQ=0.05`). Bump `GLOBE_FILE_VERSION` when retuning. (2) Heightmap: macro-dominated multi-octave Perlin (52% on two low-freq) + 30%×1.4 plate uplift; frequencies scaled by `nscale = 256 / GLOBE_WIDTH`. (3) `erosion.rs` thermal(20) → hydraulic(40). (3.5) Sea-level alignment: subtract 30th-percentile so `h<=0` means ocean. (4) `hydrology.rs` pit_fill → flow_dirs (D8) → flow_accum → `extract_rivers(min_accum=80)` + `LakeMap` (lakes gated on `height>0`; sub-sea basins become ocean). (5) Three-segment elevation remap anchored at `h_min→0.0`, `h=0→0.22` (ocean line), `h_peak (90th-pct)→0.82` (mountain line), `h_max→1.0` — guarantees ~30% ocean / ~10% mountain. `climate.rs` temp from latitude+elev; rainfall = base Perlin + orographic. (6) Per-cell biome via `biome::classify`. (7) `hydrology::build_hydrology` truth layer. (8) `geomorph::build_relief` — per-cell Sobel slope + 5×5 local relief + chebyshev coast/mountain distance + `ReliefClass`.
- **Grid scales:** `GLOBE_WIDTH=512, GLOBE_HEIGHT=256, GLOBE_CELL_CHUNKS=2` → climate cell = 2×2 chunks = 64×64 tiles. `MEGACHUNK_SIZE_CHUNKS=16` → mega-chunk = 16×16 chunks = 8×8 climate cells. Total world: 1024×512 chunks ≈ 32K×16K tiles.
- **Continuous climate:** `Globe::sample_climate(tile_x, tile_y)` bilinearly interpolates the four nearest cells (X-wrap, Y-clamp). `biome::classify_at_tile` runs per-tile — no biome stripes at cell boundaries.
- **Surface-biome layer (`biome.rs::classify_surface_at_tile` / `surface_biome_sample_at_tile`):** Separate visual/terrain biome decision so borders feather organically without touching canonical climate/hydrology. Land branch (`classify_land`, split from `classify`) runs on temp/rain sampled at domain-warped offset (128-tile wavelength × ±24-tile amplitude, hash value-noise from `globe.seed`; deterministic, `&Globe`-only). Ocean (`elev_f < OCEAN_ELEV_GATE=0.22`) and Mountain (`elev_f > MOUNTAIN_ELEV_GATE=0.82`) gates stay on true elevation. `SurfaceBiomeSample { base, accent, accent_weight }` adds O(1) ecotone: secondary warp's biome is `accent`, weight = high-freq-dithered `MAX_ACCENT_WEIGHT=0.35` when base ≠ accent. Chunk-gen Pass 1 picks surface kind from `accent` palette when `surface_band_dither < accent_weight`, else `base`. `biome_cache` / `local_detail_amp` / `topsoil_kind` / Pass-3 riparian read `base`. Callers: `terrain::generate_chunk_from_globe` Pass 1, `terrain::surface_v`, `terrain::tile_at_3d`, `terrain::climate_fertility_estimate_at`, world-map / spawn-select previews. Canonical keepers (on `classify_at_tile`/`classify`): stored `cell.biome` at worldgen, nomad camp scoring, `water_kind_at` ocean check. Does not bump `GLOBE_FILE_VERSION`.
- **Bridges:** `TileKind::Bridge` overlays former `River`. `tile_speed_multiplier = 1.4` (Road-speed); `is_water_like` / `is_freshwater` / `is_drinkable_candidate` return `true` (water flows under). `is_passable` / `is_floor` true; `is_stone_like` / `is_soil_like` false. Spawned only via `BuildSiteKind::Bridge`; world-gen never produces. `chunk_map.river_distance_at` not invalidated.
- **Fertility model (`terrain::surface_fertility_of`):** `kind_fertility_factor(kind) × elevation_fertility_curve(v) × ReliefClass::fertility_mult()` clamped to `u8`. Productive kinds: `Grass` 1.0×, `Marsh` 0.9×, `Forest` 0.7×, `Scrub` 0.3×; everything else 0. Elevation curve `(1 − |v − 0.45| · 2.0).max(0) · 255` (peaks v=0.45, support ~`[-0.05, 0.95]`). Relief multiplier: Floodplain 1.30, Lowland/Coastal/Basin 1.10, RollingHills 1.00, UplandPlateau 0.85, Foothills 0.60, Badlands 0.40, Mountain 0.15, Ocean 0. Pass 3 of chunk-gen multiplies by `river_fertility_mult` (1.6× / 1.3× in 2..=3 / 4..=5 riparian band) on top. Consumers: wild plant spawn (`chunk_streaming.rs`), plot scoring (`land.rs`), settlement scoring (`organic_settlement.rs`), hover UI. **`TileKind::Cropland`** is runtime-stamped (never world-gen) over Grass/soil by `land::carve_plots_system` / seed-farm paths for Agricultural plots; `is_soil_like` (`soil_fertility_mult` 1.3), pathing speed 0.9, road-carve-protected (`simulation::land::tile_is_farm_protected`).
- **Climate-only fertility estimate (`terrain::climate_fertility_estimate_at`):** Pure-climate counterpart. `Globe::sample_climate` + `biome::classify_surface_at_tile` + relief-gated `biome_bands(...).pick(v)` / `pick_grounded(v)` → kind; `surface_fertility_of(kind, v, relief_class)` × `river_fertility_mult` via `Globe::nearest_river_chebyshev`. Uses surface-biome `base` (not ecotone-dithered) so expected fertility averages over zero-mean accent dither. Used by `region::average_fertility_in_megachunk` (8×8 grid) for world-map overlay + spawn-select / world-map hover.
- **Material palette decoupled from elevation noise (`biome_bands` + `BiomeBands::pick_grounded`).** Land biome palettes no longer paint `TileKind::Water` at their low band — surface water arrives only from reservoir basin stamp (Pass 4) and aquifer-marsh (Pass 4.5). Stone-outcrop bands gated on `ReliefClass::permits_stone_outcrop()` via `pick_grounded` (collapses top stone slots to the middle vegetated kind): plains/floodplains stay grass even when noise crosses the high band. Only UplandPlateau/Badlands/Foothills/MountainSlope/MountainRidge cells expose bare rock.
- **Rivers:** `extract_rivers` produces `RiverEdge` between climate-cell centers; `chaikin_river_path` turns each into a deterministic curving tile polyline (3 perpendicular-jittered control points + two Chaikin passes). Polylines stored on `RiverNetwork.edge_polylines`. `terrain.rs::diamond_stamp` rasterises with Manhattan-clamped diamond + width tapering, writes `TileKind::River`, sets the water column, populates `Chunk.surface_river_distance` for the feather ring (radius `RIVER_FEATHER_DIST=16`). Riparian band shifts `BiomeBands.thresholds` toward greener slots (`greenness_rank`: Forest>Marsh>Grass>Scrub) and multiplies fertility ×1.6 / ×1.3 at distance 2-3 / 4-5; biome/fertility/topsoil effects hard-gate on `river_d ≤ 5`. Topsoil within `river_d ≤ 5` overrides to `Silt`. Settlement spawn (`score_home_candidate`, `score_tile`) uses `chunk_map.river_distance_at` and rewards `river_d ∈ 13..=16` so the initial `base_r ≈ 12` footprint fits on one bank before Chalcolithic bridges. `score_water` (nomad) and `nearest_water` (wild herd) prefer fresh. `biome::water_kind_at → WaterKind::{Fresh,Brackish,Salt}` from reservoir salinity drives the thirst pipeline without persisting a flag. World-map renders `edge_polylines` per-pixel. Lakes flood-fill via reservoir basin membership (`TileKind::Water`).
- **Cached globe (`world.bin`):** `GlobeFile { version, globe }`; bump `GLOBE_FILE_VERSION` (currently `11`) on layout changes — auto-regenerates on mismatch or on `globe.seed != WorldSeed.0`. Per-component seeded RNGs.
- **World re-rolling:** `WorldSeed` resource (default 42) drives `globe::generate_globe` and `terrain::WorldGen::with_seed`. Spawn-select Apply/Reroll fires `RegenerateWorldRequest`; `regenerate_world_system` (Update, SpawnSelect-only) reinserts both. `world.bin` persists only on `OnExit(SpawnSelect)`.
- **Worldgen RNG stays separate from sim RNG.** Globe/terrain generation uses its own `WorldSeed`-keyed RNGs (`plates.rs` `StdRng::seed_from_u64`, Perlin `set_seed`) — untouched by the runtime simulation RNG, which lives in `simulation::sim_rng::SimRng` (see `src/simulation/CLAUDE.md` → Deterministic simulation RNG). The two determinism domains are deliberately distinct.
- **Resource patch clustering (`chunk_streaming.rs`):** loose rocks, wild trees in fertile grass, wild berry bushes use two-tier deterministic hash — coarse `patch_hash` (`PATCH_CELL_SIZE=6`, separate seeds: `ROCK_PATCH_SEED`, `TREE_PATCH_SEED`, `BERRY_PATCH_SEED`) gates whether cell is a patch; per-tile hash gates density inside. Forest biome and Farmland keep uniform rates.
- **Loose rocks (`chunk_streaming.rs::spawn_chunk_loose_rocks`):** Stone GroundItems (qty 1–3); ~30% of stone-surface patch cells are rocky, ~70% of tiles in a rocky patch get a rock (~21% overall). Per-tile seed `ROCK_HASH_SEED=0xDEAD_C0DE`.

## Geology & mining

- **Surface vs subsurface:** `biome_bands(biome)` returns per-biome 4-threshold + 5-`TileKind` palette; `BiomeBands::pick(v)` chooses surface tile. Tundra → `Snow`/`Scrub`/`Granite`; Desert → `Sand`/`Scrub`/`Sandstone`; Wetland → `Marsh`/`Grass`/`Forest`; Tropical → `Marsh`/`Grass`/`Forest`/`Basalt`; Steppe → `Scrub`/`Grass`/`Sandstone`; Badlands → `Sand`/`Scrub`/`Sandstone`/`Granite`; Temperate / Grassland → `Grass`/`Forest`/`Limestone`; Mountain core → `Granite`/`Basalt`. Below: `proc_tile` runs cave-noise → `topsoil_depth(biome)` of soil → ore vein lookup → `Wall` bedrock. `topsoil_kind(biome, river_d)`: river band (`river_d ≤ 5`) → `Silt`; Wetland/Tropical → `Clay`; Temperate/Grassland/Steppe → `Loam`; Desert/Badlands → `SandySoil`; Tundra/Taiga/Mountain/Ocean → `Dirt`. `topsoil_depth`: Mountain 1; Desert/Tundra/Badlands 2; Grassland/Steppe 3; Taiga/Tropical/Temperate/Wetland 4; Ocean 0.
- **Biome classification (`biome::classify`):** Whittaker matrix extended with `Wetland` (elev_f<0.30, rain>0.75, temp>0.30), `Steppe` (rain 0.30..0.50, temp 0.40..0.70), `Badlands` (rain<0.25, elev_f 0.45..0.80). `Biome` is 11 variants. Land branch factored into `classify_land(elev,temp,rain)` so the surface-biome layer can call it with warped temp/rain; Ocean (`<OCEAN_ELEV_GATE=0.22`) / Mountain (`>MOUNTAIN_ELEV_GATE=0.82`) gates stay on true elevation.
- **Ore veins (`ORE_BANDS`):** 6 3D Perlin fields, seeded `WORLD_SEED+2..=7`. Shallow→deep: Coal (1..6, 0.45), Copper (2..8, 0.50), Tin (5..12, 0.55), Iron (6..14, 0.52), Silver (10..18, 0.60), Gold (14..32, 0.65). Encoded as `TileKind::Ore` + `TileData.ore: u8` (`OreKind`). Surface stone never drops random ore.
- **Mining yields (`carve.rs::carve_tile`):** generic stone-like tiles route via `is_stone_like()`; per-variant yield from `TileKind::stone_yield_count()` (Limestone 3, Granite/Sandstone/Basalt/Stone/Wall 2). `Ore → (ore_yield_good(ore), 2)`. `gather.rs`/`dig.rs` route via `route_yield`/`Carrier::try_pick_up`. `ActivityKind` count: 14 — keep `ACTIVITY_KINDS`/`ACTIVITY_NAMES` (debug_panel.rs) and `activity_name` (tech_panel.rs) in sync. When floor is already passable, `carve_tile` still writes a delta with the existing kind so `Chunk::set_delta` refreshes `surface_kind` off `Air`. `carve_tile` itself is now a one-shot wrapper around `finalize_carved_tile` (head + floor rewrites, no yield) — used by wells / wall destruction / terraform. Worker-driven incremental mining goes through `simulation::excavation::advance` instead (see simulation CLAUDE.md).
- **`TileData.flags` layout** (`world/tile.rs`): bit 0 building, bit 1 road, bit 2 explored, bit 3 visible, **bits 4-6 `excavation_level` (0..=7)** mirrored from `simulation::ExcavationMap`, bit 7 reserved. Helpers `TileData::excavation_level()` / `set_excavation_level(lvl)` / `is_partially_excavated()`. Pathfinding + movement + sprite refresh read these bits via `tile_speed_multiplier_from_data` / `tile_step_cost_from_data` / `step_cost_for_data` so partial-excavation tiles slow traversal.
- **Two tile-read paths:** `chunk.rs::tile_at_local` (cache-only) returns `Wall` for any uncarved subsurface tile — passability/LOS-correct but material-wrong. For ore-accurate reads call `terrain::tile_at_3d`.

## Streaming schedule

- **`chunk_streaming_system` runs on `FixedUpdate` (20 Hz)**, not `Update`. `update_chunk_retention_system` and `update_simulation_focus_system` move with it; `fog::fog_update_system` stays on `Update` (it owns `fog_map.dirty_tiles`, consumed by `apply_fog_to_tiles_system` PostUpdate — moving it to FixedUpdate would let a double-tick wipe the diff). Gizmos and `update_tile_z_view_system` also stay on `Update`.
- **Streaming is queued and budgeted.** `PendingChunkStreams` stores data loads, sprite loads, and unloads. Each FixedUpdate computes desired data/sprite sets, enqueues missing work, drains data first (`PerfWorkBudget.chunk_data_loads_per_tick`, default 24), camera sprites second (default 8), unload despawns last (default 32). Region-focus chunks load data only; camera-focus chunks enqueue sprite work once data exists.
- **Tile refreshes are deduped and capped.** `refresh_changed_tiles_system` collects `TileChangedEvent`s into `PendingTileRefreshes` and refreshes at most `PerfWorkBudget.tile_refreshes_per_tick` unique tiles (default 512).
- **Wall entities are durable across streaming.** Not registered in `TileSpriteIndex.by_chunk` (chunk unload despawns that list, but `WallMap` is kept) — `WallMap` ↔ wall-entity lifetimes stay in sync. Index despawns route through `despawn_entity_if_present` (a `Commands::get_entity` guard). The `Wall` tile delta is lost on chunk regen like Bridge/Dam/well geometry; `construction::restamp_walls_on_chunk_load` (FixedUpdate, chained between `restamp_wells_on_chunk_load` and `restamp_runtime_water_on_chunk_load`) re-stamps it from `WallMap` at `surface_z + 1` on each `ChunkLoadedEvent` (skips tiles already `Wall`).
- **`update_tile_z_view_system`** carries a `Local<Option<i32>>` last-applied-z guard — Bevy's `is_changed()` fires false-positives, and the system iterates the full loaded-sprite set on every fire.

## Floristic regions & native-plant spawn (`flora_regions.rs`, `chunk_streaming.rs`)

`Globe.flora_regions: FloraRegionMap` built once in `generate_globe` (after `build_relief`) via flood-fill of land cells partitioned by latitude band + dominant biome. Each `FloraRegion { realm: FloraRealmKindWire, flavor_name, … }`; `floristic_region_at_tile` is the per-tile accessor. Deterministic across runs of the same seed — the build walks AHashMap-grouped cells through a `(lat, biome)`-sorted promotion pass so per-process hasher state can't leak into region ids (`Biome` derives `Hash`). Bumps `GLOBE_FILE_VERSION`.

`PlantCatalog.native_pools` precomputes per-`(FloraRealmKind, Biome)` species lists in `from_defs`; `native_pool_for(realm, biome)` is the O(1) accessor (catalog lives in `simulation/plant_catalog.rs`). `chunk_streaming::spawn_chunk_plants` walks this slice (typically 0–12 species) instead of all ~50. **Phase 1 ecology seeder (community/stratum):** the per-`(realm,biome)` native pool *is* the tile's plant community, structured into vertical strata (`PlantForm::stratum` → Canopy/Understory/GroundCover). `biome_stratum_targets(biome)` sets per-stratum density targets (plants per `COMMUNITY_CELL_AREA` = `PATCH_CELL_SIZE²` tiles) — the ecological knob (forest biomes carry a canopy; grassland is canopy-free with dense ground cover; desert sparse). Per tile, strata are walked Canopy→Understory→GroundCover; each rolls an independent deterministic acceptance (`tile_roll_hash(tx,ty,salt)` × `stratum_suitability(relief)` from slope/TPI/aquifer-moisture), and the first stratum that wins *and* has a gate-passing candidate claims the tile (one `Plant`/tile). All legacy per-tile gates still apply (surface_tiles, fertility, coastal, **river_distance via `chunk_map.river_distance_at`**, patch_noise, base_weight weighted pick). Each tile is pure in `(tx,ty,seed)` → independent of chunk load order. `legacy_pick_species` (the old per-biome `EMPTY_BIAS` lottery) is retained behind `USE_COMMUNITY_SEEDER`. (Catalog/lifecycle/harvest detail in `../simulation/CLAUDE.md` → Plant lifecycle.)

## Thin housing edge walls (`edge.rs`, `plans/thin-edge-walls.md`)

Permanent dwellings (huts/longhouses) put walls/doors on the *boundary edge between two tiles* instead of consuming whole tiles, so the full footprint stays passable floor. `EdgeKey{axis,x,y}` (canonical lower/left owner tile) + `EdgeState{Open/Wall/ClosedDoor/OpenDoor}`. A cheap per-`Chunk` cache `edge_bits: Option<Box<ChunkEdgeBits>>` (N/E edge per tile, 2 bits each) is read by `ChunkMap::{edge_blocks_move,edge_blocks_los_opaque}` *inside* `passable_step_3d`/`passable_step_for` (cardinal; diagonals gated by the corner legs) and the LOS `walk_ray` transition test — zero new params ripple out.

Durable truth = `EdgeStructureMap` (`EdgeWall{material,owner_faction}` / `EdgeDoorRef{open,faction,dir}`), re-stamped on chunk load by `restamp_edge_structures_on_chunk_load` (mirrors `WallMap`↔`TileKind::Wall`). Construction: `BuildSiteKind::{EdgeWall(WallMaterial),EdgeDoor}` reuse the `Wall`/`Door` recipes; `walled_house_tile_plan` emits **one `EdgeWall` blueprint per perimeter tile** carrying an `edge_sides: u8` bitmask (`edge_side::{N,E,S,W}`; corners=2 bits) so blueprints stay 1:1 with tiles and `BlueprintMap`/haul/claim are untouched; finalize stamps `EdgeStructureMap` + cache + `WallConstructed`. Reachability sim is edge-aware (`simulate_house_reachable_edges`, `ReachOpts.blocked_edge`). Full-tile `Wall`/`Door` (natural rock, palisades, manual, composite L/U houses) are unchanged.

**Floor keep-out:** `DwellingEnvelopeMap` (tile→`DwellingId` + per-dwelling `DwellingInfo{tiles,bbox}`) is registered with the full footprint rect at `plan_building` and `seed_walled_house_at`. Consulted by `organic_settlement::single_tile_clear` (the actual placement gate — `rect_clear_for_parcel` stays unaware since the parcel is only a zoning hint), `road_carve_system`'s `try_write_road` + connector planner, and `well_footprint_clear`. `populate_seed_reservation_system` folds the envelope into `SeedReservation` (seed-time plant/obstacle/stranded-member keep-out). Kitchen gardens: `SurveyStructureSnapshot.dwelling_rects` carries envelope footprints to the async survey so `append_dwelling_gardens` finds thin-wall huts. Rendering + net replication live in `../rendering/CLAUDE.md` / `../net/CLAUDE.md`.
