# World (`src/world/`)

Procedural terrain, globe generation, climate, geology, spatial index. See root `CLAUDE.md` for tile/chunk/Z conventions and `SpatialIndex` discipline.

## World generation

- **Terrain noise (`terrain.rs::surface_v`):** Globe-anchored. Macro signal = bilinearly-sampled `Globe::sample_climate(tx,ty)` elevation; per-tile detail = 3-octave Perlin perturbation at ¬±`LOCAL_DETAIL_AMP=0.20`. Result: in-game tile z follows the world map's elevation field ‚Äî Ocean biomes have low z, Mountain biomes have tall peaks, Grassland sits near sea level ‚Äî instead of the old purely-local noise that decoupled per-tile z from globe biome. Drop `surface_v` reshape (was `sign(c)*|c|^0.65`); the macro‚Üídetail split now provides terrain shape.
- **Globe pipeline (`globe.rs::generate_globe`):** 512√ó256 climate-cell grid. (1) Lloyd-relaxed Voronoi over `NUM_PLATES=8` with **domain-warped assignment** (`plates::assign_nearest` offsets the lookup point by a two-octave Perlin warp at `WARP_FREQ=0.04`, `WARP_AMP=8` cells) so plate boundaries are wavy/fingered rather than chord-straight. Uplift/subsidence stamped along boundaries; post-smooth uplift gets ¬±12% Perlin jitter (`JITTER_FREQ=0.05`) so ridge height varies along its length instead of reading as a uniform wall. Bump `GLOBE_FILE_VERSION` (now 4) when retuning warp/jitter constants. (2) Heightmap composition: macro-dominated multi-octave Perlin (52% on two low-freq octaves so continents read as cohesive landmasses) + 30%√ó1.4 plate uplift. Noise frequencies scaled by `nscale = 256 / GLOBE_WIDTH` so doubling the grid keeps continents the same world-size. (3) `erosion.rs` thermal(20) ‚Üí hydraulic(40). (3.5) **Sea-level alignment shift**: subtract the 30th-percentile from every cell so `h<=0` means ocean for both hydrology and biome classification. Without this, hydrology drained to the absolute h=0 contour, which usually fell well inland of the visible coastline ‚Üí rivers terminated mid-continent instead of reaching the sea. (4) `hydrology.rs` pit_fill ‚Üí flow_dirs (D8) ‚Üí flow_accum ‚Üí `extract_rivers(min_accum=80)` + `LakeMap` (lakes gated on `height > 0`; sub-sea basins become ocean). (5) **Three-segment elevation remap** anchored at `h_min ‚Üí 0.0`, `h=0 ‚Üí 0.22` (ocean line), `h_peak (90th-pct) ‚Üí 0.82` (mountain line), `h_max ‚Üí 1.0`. Guarantees ~30% ocean / ~10% mountain regardless of distribution shape (verified by `ocean_fraction_within_band` test). `climate.rs` temp from latitude+elev; rainfall = base Perlin + orographic. (6) Per-cell biome via `biome::classify`.
- **Grid scales:** `GLOBE_WIDTH=512, GLOBE_HEIGHT=256, GLOBE_CELL_CHUNKS=2` ‚Üí climate cell = 2√ó2 chunks = 64√ó64 tiles. `MEGACHUNK_SIZE_CHUNKS=16` ‚Üí mega-chunk = 16√ó16 chunks = 8√ó8 climate cells. Total world: 1024√ó512 chunks ‚âà 32K√ó16K tiles (unchanged from the pre-doubling layout ‚Äî only the climate sampling density doubled).
- **Continuous climate:** `Globe::sample_climate(tile_x, tile_y)` bilinearly interpolates the four nearest cells (X-wrap, Y-clamp). `biome::classify_at_tile` runs *per-tile* ‚Äî no biome stripes at cell boundaries.
- **Rivers**: `extract_rivers` produces `RiverEdge { from, to, from_width, to_width }` between climate-cell centers; `chaikin_river_path` then turns each into a deterministic curving tile polyline (3 perpendicular-jittered control points + two Chaikin corner-cut passes). Polylines stored on `RiverNetwork.edge_polylines` (parallel to `edges`). `terrain.rs::diamond_stamp` rasterises with a Manhattan-clamped diamond + width tapering between `from_width`‚Üí`to_width`, writes `TileKind::River` (sibling of `Water` ‚Äî see root `CLAUDE.md`), depresses `surface_z` by 1, and populates `Chunk.surface_river_distance` for the feather ring (radius `RIVER_FEATHER_DIST=5` outside the channel). The riparian band shifts `BiomeBands.thresholds` toward greener slots (Desert/Tundra ‚Üí Grass/Forest strips, ranked by `greenness_rank` so Forest>Marsh>Grass>Scrub) and multiplies fertility by 1.6√ó / 1.3√ó at distance 2-3 / 4-5. Topsoil within `river_d ‚â§ 5` overrides to `Silt`. Settlement spawn (`person.rs`) uses `chunk_map.river_distance_at` in a best-of-N picker ‚Äî riverside camps preferred over arid interior. `score_water` (nomad) and `nearest_water` (wild herd) prefer fresh (`is_freshwater()`) over salt. **`biome::water_kind_at(globe, kind, tx, ty) -> WaterKind`** classifies ocean `Water` as `Salt` (via `Biome::Ocean` lookup) and inland `Water`/`Marsh`/`River` as `Fresh` ‚Äî drives the thirst pipeline's drinkable-tile filter without persisting a flag on `TileData`. World-map render walks `edge_polylines` for a per-pixel polyline overlay; cell-level `is_river` tint dropped (was overpainting adjacent land). **Lakes** still flood-fill discs at `lake.level_z` (TileKind::Water).
- **Cached globe (`world.bin`):** `GlobeFile { version, globe }`; bump `GLOBE_FILE_VERSION` (currently `6`) on layout changes ‚Äî auto-regenerates on mismatch *or* on `globe.seed != WorldSeed.0`. Determinism via per-component seeded RNGs.
- **World re-rolling:** `WorldSeed` resource (default 42) drives both `globe::generate_globe` and `terrain::WorldGen::with_seed`. The spawn-select UI exposes Apply/Reroll buttons that fire `RegenerateWorldRequest`; `regenerate_world_system` (Update, SpawnSelect-only) reinserts both resources. `world.bin` is only persisted on `OnExit(SpawnSelect)` so rapid re-rolls don't thrash disk.
- **Resource patch clustering (`chunk_streaming.rs`):** loose rocks, wild trees in fertile grass, and wild berry bushes are placed via a two-tier deterministic hash ‚Äî a coarse `patch_hash` (`PATCH_CELL_SIZE=6` tiles, separate seeds per kind: `ROCK_PATCH_SEED`, `TREE_PATCH_SEED`, `BERRY_PATCH_SEED`) gates whether a cell is a patch; a per-tile hash gates density inside the patch. Result: discrete groves / berry patches / rock fields rather than a uniform carpet. Forest biome and Farmland keep their original uniform spawn rates (already curated zones).
- **Loose rocks (`chunk_streaming.rs::spawn_chunk_loose_rocks`):** Stone GroundItems (qty 1‚Äì3); ~30% of stone-surface patch cells are rocky, ~70% of tiles inside a rocky patch get a rock (~21% overall, was 35% uniform). Per-tile hash seed `ROCK_HASH_SEED=0xDEAD_C0DE`.

## Geology & mining

- **Surface vs subsurface:** `biome_bands(biome)` returns the per-biome 4-threshold + 5-`TileKind` palette; `BiomeBands::pick(v)` chooses the surface tile (replaces the old `surface_kind_fn` with its 4-float tuple). Each biome has its own flavour: Tundra ‚Üí `Snow`/`Scrub`/`Granite`; Desert ‚Üí `Sand`/`Scrub`/`Sandstone`; Wetland ‚Üí `Marsh`/`Grass`/`Forest`; Tropical ‚Üí `Marsh`/`Grass`/`Forest`/`Basalt`; Steppe ‚Üí `Scrub`/`Grass`/`Sandstone`; Badlands ‚Üí `Sand`/`Scrub`/`Sandstone`/`Granite`; Temperate / Grassland ‚Üí `Grass`/`Forest`/`Limestone`; Mountain core ‚Üí `Granite`/`Basalt`. Below: `proc_tile` runs cave-noise ‚Üí `topsoil_depth(biome)` of soil ‚Üí ore vein lookup ‚Üí `Wall` bedrock. Topsoil variant via `topsoil_kind(biome, river_d)`: river band (`river_d ‚â§ 5`) ‚Üí `Silt`; Wetland/Tropical ‚Üí `Clay`; Temperate/Grassland/Steppe ‚Üí `Loam`; Desert/Badlands ‚Üí `SandySoil`; Tundra/Taiga/Mountain/Ocean ‚Üí `Dirt`. `topsoil_depth`: Mountain 1; Desert/Tundra/Badlands 2; Grassland/Steppe 3; Taiga/Tropical/Temperate/Wetland 4; Ocean 0.
- **Biome classification (`biome::classify`):** Whittaker matrix extended with three new biomes: `Wetland` (elev_f<0.30, rain>0.75, temp>0.30), `Steppe` (rain 0.30..0.50, temp 0.40..0.70), `Badlands` (rain<0.25, elev_f 0.45..0.80). `Biome` is now 11 variants; `GLOBE_FILE_VERSION=7`.
- **Ore veins (`ORE_BANDS`):** 6 3D Perlin fields, seeded `WORLD_SEED+2..=7`. Shallow‚Üídeep: Coal (1..6, 0.45), Copper (2..8, 0.50), Tin (5..12, 0.55), Iron (6..14, 0.52), Silver (10..18, 0.60), Gold (14..32, 0.65). Encoded as `TileKind::Ore` + `TileData.ore: u8` (`OreKind`). Surface stone never drops random ore ‚Äî players must dig down.
- **Mining yields (`carve.rs::carve_tile`):** generic stone-like tiles route via `is_stone_like()`; per-variant yield from `TileKind::stone_yield_count()` (Limestone 3, Granite/Sandstone/Basalt/Stone/Wall 2). `Ore ‚Üí (ore_yield_good(ore), 2)`. `gather.rs`/`dig.rs` route via `route_yield`/`Carrier::try_pick_up`. `ActivityKind` count: 13 ‚Äî keep `ACTIVITY_KINDS`/`ACTIVITY_NAMES` (debug_panel.rs) and `activity_name` (tech_panel.rs) in sync when adding. When the floor is already passable (procedural topsoil), `carve_tile` still writes a delta with the existing kind so `Chunk::set_delta` refreshes `surface_kind` off `Air`.
- **‚öÝÔ∏è Two tile-read paths:** `chunk.rs::tile_at_local` (cache-only) returns `Wall` for any uncarved subsurface tile ‚Äî passability/LOS-correct but **material-wrong**. For ore-accurate reads (mining yields, inspector display) call `terrain::tile_at_3d`.

## Streaming schedule

- **`chunk_streaming_system` runs on `FixedUpdate` (20 Hz)**, not `Update` (60+ Hz). The per-frame variant burned cycles on the full-map unload filter even when the camera was stationary. One fixed-tick (‚â§50 ms) load lag is imperceptible. `update_chunk_retention_system`, `update_simulation_focus_system`, and `fog::fog_update_system` move with it; gizmos and `update_tile_z_view_system` stay on `Update`.
- **`update_tile_z_view_system`** carries a `Local<Option<i32>>` last-applied-z guard. Bevy's `is_changed()` fires false-positives (first-tick re-fire, same-value `set_changed()` writes) and the system iterates the full loaded-sprite set on every fire.
