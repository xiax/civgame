# Simulation (`src/simulation/`)

Agent AI, factions, knowledge, hunting, typed-task pipeline, pluralist economy. See root `CLAUDE.md` for cross-cutting `SimulationSet` ordering.

## Game speed and pause (`speed.rs`)

`GameSpeed { current, last_unpaused }` (Resource) is the player-facing state; `SpeedPreset::{Paused, Normal, Fast (2×), VeryFast (5×)}` is the enum. `sync_game_speed_to_virtual_time` (PreUpdate, change-detected) mirrors it onto `Time<Virtual>::set_relative_speed` / `pause`. Higher presets cause Bevy to run FixedUpdate more often per real second — every per-tick decay, cooldown, calendar advance, and `tick % CADENCE == 0` Economy system scales naturally. **`SimClock` no longer carries `speed` / `accum`** — `scale_factor()` returns bucket compensation (`population / bucket_size`) only. Per-tick callers (`needs::tick_needs_system`, `tasks::work_progress_system`, `combat::combat_system`, `movement::movement_system`'s smooth-step `effective_speed * dt`, `animals::animal_movement_system`'s `sim_dt`) read `clock.scale_factor()` unchanged; the speed multiplier rides on FixedUpdate firing rate, not an inline factor. `set_max_delta(50ms)` in `main.rs` keeps catch-up from spiraling after an OS stall. The keyboard handler (`handle_speed_keybinds_system`, registered by `UiPlugin` so headless tests skip it) maps `Space` → toggle pause and `1/2/3` → `Normal/Fast/VeryFast`, gated on `EguiContexts::wants_keyboard_input`. HUD speed buttons in `ui/hud.rs` write `GameSpeed.current`. **Intentional behavior change:** the calendar (`world/seasons.rs::advance_calendar_system`), plant lifecycle, day-night overlay, and every daily Economy cadence now scale with `GameSpeed` — previously they were independent of `SimClock.speed`. `SimTimingDiagnostics` (resource, ticks/frame + tick-CPU EMA + worst-60 window) surfaces in the debug panel's "Sim Timing" section and reddens when avg tick CPU exceeds the preset's budget (50/25/10 ms at 1×/2×/5×). Per-preset CPU budgets live on `SpeedPreset::budget_ms_per_tick()`; adding `10×` later is one enum variant + one HUD button + one budget entry. Path drain (`pathfinding::worker::drain_path_requests_system`) now lives on FixedUpdate before `SimulationSet::Sequential` with `PATH_BUDGET_PER_TICK = 192` (up from 64), so the per-real-second budget at 1× matches the previous PreUpdate-at-60fps budget and scales linearly with game speed.

## Agent AI (Goals → HTN → Tasks)

AI dispatch runs end-to-end through HTN.

- **Goals (`goals.rs`):** high-level objectives (`Survive`, `GatherWood`, `GatherStone`, `GatherFood`, `Haul`, `Defend`, `Farm`, `Build`, `Socialize`, `Sleep`, `MigrateToCamp`, …) driven by Needs + Faction state.
- **HTN registry (`htn.rs`):** per-goal decomposition. Methods carry `precondition` / `utility` / `expand(ctx) -> Vec<Task>`; the dispatcher argmaxes over applicable methods and routes the head into the typed-task channel, with the tail prefetched on `ActionQueue`'s queue ring.
- **Tasks (`tasks.rs`):** the agent's *current* action — `TaskKind` enum (Gather, Construct, WithdrawMaterial, Hunt, Butcher, Equip, Migrate, …). Transient; `Idle` between tasks.
- **GoalScorer + Disposition infrastructure (`goal_scorers.rs`):** `Disposition { entrepreneurial, gregariousness, curiosity, martial }` Component (`u8` on `[0, 255]`, scattered at spawn) drives per-agent goal preferences. `GoalClass` enum (`Survival > Subsistence > Safety > Belonging > Esteem > Enterprise > Discretionary`, `Ord`-derived) tiers qualitatively-different goals so registry argmax picks `Survival → Sleep` over `Enterprise → Craft` regardless of raw score. `GoalScore` carries `AgentGoal`, class, score, reason, `GoalCommitment`, `InterruptPolicy`; `AgentDecisionState` records the last selection for debug UI / hysteresis / interrupts. `GoalScorer` trait + `GoalScorerRegistry` Resource exposes `best(ctx)` / `best_with_incumbent(ctx, current_goal)`; `register_default_scorers` pushes the default set at `SimulationPlugin::build`.
- **Behavioural-richness scorer set (`goal_scorers.rs` + `utility_curves.rs`).** Pure-function continuous utility curves (`hunger_utility / sleep_utility / social_utility / play_utility / material_deficit_utility / disposition_lift`) anchored on legacy `HUNGER_*` / `SLEEP_*` cliffs. Nine scorers cover the legacy need-cascade: `SurvivalHungerScorer` (Survival; emits `Survive` only at the four legacy cliffs, returns `None` below `hunger 150` since every food/eat HTN method gates on `EAT_TRIGGER_HUNGER = 180`), `SleepScorer` (Survival, sleep_utility × `time_of_day_bonus`), `ReturnSurplusScorer` (Subsistence), `SocialScorer` (Belonging), `PlayScorer` (Discretionary), `TameHorseScorer` (Subsistence, curiosity lift), `PersonalBuildScorer` (Esteem), `CraftDemandScorer` (Subsistence; entrepreneurial lift + Crafter/Apprentice bonus), `StockpileScorer` (bifurcated: Subsistence 0.85 when `prioritize_food`, else Discretionary 0.20). Per-scorer Disposition lifts (`SOCIAL_GREG_LIFT = 1.5`, `PLAY_GREG_LIFT = 1.2`, etc.) live in one tunables block at the top of `goal_scorers.rs`.
- **Scorer pipeline is the sole goal-selection path (`goal_update_system`).** Builds a `GoalScoringContext` from precomputed gates (`is_starving / faction_has_food / can_return_camp / prioritize_food / fallback_gather* / has_horse_taming / has_personal_build_site / should_craft`) plus `time_of_day_bonus` from `Calendar::time_phase()` (Day 0.0 / Dawn 0.2 / Dusk 0.6 / Night 1.0), then calls `GoalScorerRegistry::best_with_incumbent(...)`. The legacy imperative cascade survives only as `fallback_pick` for SOLO agents / fixtures with no faction.
- **Goal-selection hysteresis:** `GOAL_CHALLENGER_MARGIN = 0.10` dampens single-tick flips. When best != current goal, the current goal is re-scored across all scorers at ≥ best's class; if `best.score - cur_score < 0.10`, keep current. `interrupt_policy_allows(policy, current_class, challenger_class)` gates both normal replacement and opportunistic interrupts.
- **`ScorerInputs` SystemParam (`goals.rs`):** bundles `GoalScorerRegistry`, `JobBoard`, `OpportunityIndex`, `DecisionMetrics`, and per-agent `Disposition / Skills / Profession / AgentDecisionState / Injury` queries to stay under Bevy's 16-param ceiling. Missing-component fallbacks (`Disposition::default()`, etc.) keep test fixtures robust.
- **Opportunity index (`opportunity.rs`):** rebuilt in Economy every 20 ticks after `compute_faction_storage_system`, with TTL expiry between rebuilds. Producers emit `PaidJob` / `MaterialDeficit` / `FoodSource` / `CareNeed` into kind, faction+kind, region+kind buckets. `EarnIncomeScorer` and `ProvideCareScorer` read the index first, with direct fallback.
- **Opportunistic en-route interrupts (`opportunistic.rs`).** `GoalScorer::opportunistic()` trait method (default false); `SocialScorer` / `PlayScorer` opt in. `opportunistic_interrupt_system` (ParallelA, `.after(goal_update_system).before(record_abandoned_method_system)`, 20-tick cadence) eligibility: not Dormant, not `JobClaim`-locked, `aq.current == Task::WalkTo`, `ai.active_method` not `MF_UNINTERRUPTIBLE`. Picks highest `(class, score)` of opportunistic scorers above `OPPORTUNISTIC_INTERRUPT_THRESHOLD (0.50)`, not on the agent's `GoalCooldown` ring. On interrupt: stamps prior goal into the cooldown ring for `OPPORTUNISTIC_COOLDOWN_TICKS (150)`, flips `*goal`, `aq.cancel()`, resets `ai.task_id = UNEMPLOYED`, increments `OpportunisticInterruptStats`.
- **HTN Disposition lift (`htn.rs`).** `Method::disposition_lift(&self, Disposition) -> f32` (default 1.0). `score_method_with_history_and_disposition(...)` multiplies utility × disposition_lift before the failure penalty. Overridden by `SocializeWithPartnerMethod` (gregariousness), `HuntPreyMethod` (martial), `PlayWithPartnerMethod` (gregariousness) — each capped at 1.3 so the range `[1.0, 1.3]` doesn't cross `UTIL_VISIBLE_GROUND` / `UTIL_CLAIMED_HAUL` tier breakpoints. `dispatch_for_goal(...)` is the shared argmax helper used by Socialize / Play / Scout / ReturnSurplus / TameWildHorse / HarvestPlant.
- **Heal pipeline — injury data layer (`medicine.rs`).** `Injury { severity: u8, applied_tick, last_damage_tick }` derived reactively from `Health.current < Health.max` by `injury_tracking_system` (Sequential, after `combat::combat_system`) via observer on `Changed<Health>`. Severity = `max - current`; clears when fully restored. Two `AgentGoal` variants: `SeekCare = 22` (patient; in `allowed_while_packed` + `is_policy_uninterruptible`) and `ProvideCare = 23` (provider).
- **Heal pipeline — scorers.** `HealNeedScorer` is **Survival** class (Safety wouldn't preempt a routine craft posting); score = `severity / 255`. `ProvideCareScorer` (Subsistence, 0.65) fires for `Healer` / `Apprentice` when any same-faction agent has `Injury` — beats `CraftDemand` (0.50) and idle `StockpileScorer` (0.20) but loses to prioritize-food `StockpileScorer` (0.85). `GoalScoringContext` carries `injury: Option<Injury>` + `faction_has_injured: bool`; the latter is precomputed via `injured_faction_q: Query<&FactionMember, With<Injury>>`.
- **Heal pipeline — provider task chain.** `htn_provide_care_dispatch_system` (ParallelB, after `htn_combat_faction_dispatch_system`) scans within `HEAL_SCAN_RADIUS = 12` chebyshev for nearest same-faction patient → `Task::Heal { patient }`. `TaskKind::Heal` is adjacency-mode. `heal_task_system` (Sequential, after combat) requires chebyshev ≤ `HEAL_ADJACENCY_RADIUS (1)`, decrements `Injury.severity` by `HEAL_SEVERITY_PER_TICK (1)`, grants `HEAL_MEDICINE_XP_PER_TICK (1)` Medicine XP, removes `Injury` at zero + `aq.advance()`.
- **Heal pipeline — patient routing.** `TaskKind::SeekCare` + `htn_seek_care_dispatch_system` (ParallelB, after `htn_provide_care_dispatch_system`); gates on `AgentGoal::SeekCare + Idle + UNEMPLOYED + Without<Drafted> + With<Injury>`. Resolves nearest faction-owned `Shrine` via `WorkshopOwnership::workshops_for(fid)` with `home_tile` fallback. Short-circuits when within `SEEK_CARE_AT_SITE_RADIUS = 6` of the chosen site. `WalkReason::SeekCare`; `goal_dispatch_system` preserve arm `(SeekCare, TaskKind::SeekCare)`.
- **Heal pipeline — chief Healer assignment (`medicine::chief_healer_assignment_system`).** Economy, every `HEALER_ASSIGNMENT_CADENCE = TICKS_PER_DAY/4`. Mirrors `chief_craft_assignment_system` (EV-ranked promote/demote; `HEALER_DEMOTE_BUFFER = 1`; survival-override on `FARMER_SURVIVAL_FLOOR`; sub-`APPRENTICE_THRESHOLD` Medicine routes through `ApprenticeProgress { target_profession: Healer }` bound to a master Healer; shared `demote_profession_state`). Target is injury-driven: `target = if injured == 0 || per_head < survival_floor { 0 } else { min(max(1, injured.div_ceil(HEALER_PER_INJURY_DIVISOR=4)), member_count / HEALER_MAX_DIVISOR=3) }`. Scheduled in the secondary Economy `add_systems` block (primary at Bevy's 20-element ceiling); `apprentice_progress_system` orders after it.
- **Inspector injury readout (`ui/inspector.rs`).** Identity block shows `Injured: severity N (fresh|stable, D d since onset)` when `Injury` present, coloured yellow→red. `fresh → stable` at `now - last_damage_tick >= TICKS_PER_DAY/2`.
- **`EarnIncomeScorer` (`goal_scorers::EarnIncomeScorer`):** Score = `posting.reward × skill_competence(primary_skill_for(prof)) × disposition.earn_income_multiplier()` over the highest-reward unclaimed paid posting on the agent's faction whose `JobKind` matches `job_kinds_for(prof)`. Reads `OpportunityIndex::PaidJob` first, falls back to `JobBoard`. Declines for `None` / `Apprentice` agents or pure-Subsistence factions (both `economic_policy` and `wage_signal` empty). Tier = `Enterprise`. Travel cost stays in `job_claim_system`'s `U_bid`.
- **Professions (`person.rs::Profession`):** `None | Farmer | Hunter | Bureaucrat | Trader | Crafter | Apprentice | Healer`. Persistent role. Farmer auto-assigned by `faction_profession_system` when food < 100. Hunter via `faction_hunter_assignment_system` (Economy, every `TICKS_PER_DAY/4`): target = `max(1, adults*0.20)` × martial × prey-density, capped at adults/2; ordered after Crafter/Healer passes so scarce `None` candidates go to explicit demand first. Bureaucrat/Trader cover settlement administration and arbitrage (see Pluralist Economy). **Crafter** via `chief_craft_assignment_system` (Economy, after `chief_bureaucrat_appointment_system`, every `CRAFTER_ASSIGNMENT_CADENCE = TICKS_PER_DAY/4`): target = `max(1, members × CRAFTER_MIN_RATIO (0.25))` capped at `members / CRAFTER_MAX_DIVISOR (3)`, gated on max Craft `wage_signal` EMA via `crafter_target_with_hysteresis(craft_ema, current, members)` — `ema ≥ CRAFTER_WAGE_PROMOTE_FLOOR (1.0)` → ratio target; `ema ≤ CRAFTER_WAGE_DEMOTE_CEILING (0.3)` → 0; in between → hold. Crafter affinity into `U_bid`: `jobs::CRAFTER_AFFINITY_BONUS = 3.0`; unpaid `profession_bias` arms `(Crafter, Craft) = 0.5` and `(Crafter, Stockpile) = 0.1`. `capital::tool_profession(tools) → Crafter` and `WorkshopKind::affine_to(Crafter)` (Workbench / Loom) wire capital recognition.
- **Capital recognition (`capital.rs`):** read-only helpers used by EV-driven profession choice. `OwnedBy { faction_id, kind: WorkshopKind, tile }` stamped on every Workbench / Loom / Granary / Shrine / Market / Barracks / Monument at finalize (blueprint + seeded paths in `construction.rs`); add/remove hooks (registered in `SimulationPlugin::build`) maintain a `WorkshopOwnership` resource (`AHashMap<u32, Vec<WorkshopEntry>>`). Three composing factors with `[1.0, ~2.0]` shape: `tool_capital_factor` → 1.5 if any inventory/hand item maps via `capital::tool_profession` (today: `weapon → Hunter`); `workshop_capital_factor` → +1.0 for household-owned profession-affine workshop within `WORKSHOP_AFFINITY_RADIUS = 12` chebyshev, else +0.5 for village-owned; `land_capital_factor` → 1.5 for Farmer when household holds an Agricultural plot under non-StateOwned tenure. `WorkshopKind::affine_to(Profession)`: Market → Bureaucrat, Workbench/Loom → Crafter, Shrine → Healer.
- **Asymmetric demotion buffers:** `HUNTER_DEMOTE_BUFFER = 1` and `BUREAUCRAT_DEMOTE_BUFFER = 1` add a one-slot tolerance on the demote arm of `faction_hunter_assignment_system` and `chief_bureaucrat_appointment_system`. Promotion fires on any shortfall; demotion only fires when `current > want + buffer`. `want == 0` (survival override / treasury-empty quit / tech-loss) bypasses the buffer for full stand-down. Crafter has analogous EMA-band hysteresis via `crafter_target_with_hysteresis`.
- **Survival override:** `FARMER_SURVIVAL_FLOOR = 16.0` per-head food units. `faction_hunter_assignment_system`, `chief_bureaucrat_appointment_system`, `chief_craft_assignment_system`, `chief_healer_assignment_system` each compute `per_head = food_total / member_count` and force `target = 0` when below the floor, regardless of normal gates. Incumbents demote via shared `demote_profession_state`. Two-floor band: <16 stands down all specialised labor; 16–32 → Farmer ramp (`FARMER_PROMOTE_PER_HEAD = 32`); >32 × `FARMER_DEMOTE_RATIO (1.6)` → demote excess Farmers. Apprentices unaffected (override zeros promotion target, not active `ApprenticeOf` links). Tests use `seed_faction_food(fid, qty)` to avoid accidental override.
- **Unified cross-profession switcher (`profession_choice::cross_profession_switch_system`):** daily Economy pass; lets an employed agent switch directly between `{Hunter, Bureaucrat, Crafter}` when `EV(target) > EV(current) × EV_SWITCH_HYSTERESIS (1.20)`. Closes the None ↔ X round-trip gap of the per-profession systems. `EV(target) = expected_wage(faction, target, skills, capital_factor)` minus `switching_cost_skill_regret(faction, current, peaks) = peak[primary] / SKILL_MAX × SKILL_REGRET_FRACTION (0.20) × aggregate_wage_per_day(current)`. Faction caps via `faction_cap_for(faction, target)`: Hunter ≤ adults/2 (chief-aware `HUNTING_SPEAR`), Bureaucrat ≤ `max(1, adults × BUREAUCRAT_MIN_RATIO)` (`state_funds_public_works`), Crafter ≤ `adults / CRAFTER_MAX_DIVISOR`. Survival override locks every non-Farmer cap to `None`. Sub-`APPRENTICE_THRESHOLD` Crafter targets route through `Profession::Apprentice`. Ordered after every other profession-mutation system; `profession_change_log_system` orders after the switcher.
- **Profession-choice helpers (`profession_choice.rs`):** `skill_competence(s) = 0.2 + (s/SKILL_MAX) * 0.8`; `job_kinds_for(Profession) -> &'static [JobKind]` (Farmer → `[Farm, Stockpile]`, Hunter → `[Stockpile]`, Bureaucrat → `[Build]`, Trader → `[Haul]`, Healer → `[Craft]`); `primary_skill_for(Profession) -> Option<SkillKind>`; `aggregate_wage_per_day(faction, prof)` sums `wage_signal[(kind, _)].ema_per_day` across job kinds (target_rid wildcard); `expected_wage = aggregate × skill_competence × capital_factor`. Shared `demote_profession_state(entity, ai, aq, reservations, commands)` consolidates teardown (drop task, release reservation, strip `Carrying`). Hunter / Bureaucrat / Farmer systems sort candidates by `(expected_wage, primary_skill)`; raw skill is the wage-zero tiebreaker. Capital is threaded through `capital::capital_factor` so weapon-bearing Hunters / Market-adjacent Bureaucrats / land-holding Farmers see real EV lifts.
- **Healer scaffolding:** `Profession::Healer` recognized by `profession_choice` (`primary_skill_for = Medicine`, `job_kinds_for = [Craft]` until a dedicated `JobKind::Heal` lands, `faction_cap_for = adults / CRAFTER_MAX_DIVISOR`). `WorkshopKind::Shrine.affine_to(Healer) = true`. `ApprenticeProgress { target_profession }` plumbs Healer-target graduation: writes `SkillKind::Medicine ≥ APPRENTICE_THRESHOLD` and sets `*prof = progress.target_profession`. Auto-promotion via `medicine::chief_healer_assignment_system` (injury-driven, not wage-EMA).
- **Apprenticeship (`apprenticeship.rs`):** novice-Crafter onboarding. When `chief_craft_assignment_system` would promote a `None` candidate with `Skills[Crafting] < APPRENTICE_THRESHOLD (30)`, scans the faction's mentor pool (Crafter + `Skills[Crafting] >= MASTER_THRESHOLD (100)` + no existing `MentorOf`). On match: candidate becomes `Apprentice` + `ApprenticeOf { mentor }` + `ApprenticeProgress { ticks: 0, target_ticks: TICKS_PER_DAY × APPRENTICESHIP_DURATION_DAYS (30) }`; master gets `MentorOf { apprentice }`. No mentor → falls back to direct Crafter promotion. `apprentice_progress_system` (Economy, daily, after `chief_craft_assignment_system`) advances `ticks` by `TICKS_PER_DAY`; at target graduates to `Crafter`, lifts `Skills[Crafting]` to threshold, strips links. Stale mentor links (master despawned / `MentorOf` removed) demote the orphan to `None`, discarding progress. Apprentices count toward `current_crafters` but stay out of Farmer / Hunter / Bureaucrat promotion pools. Payouts in `job_payout_system` split: claimant `share × WAGE_FRACTION_APPRENTICE (0.4)`; mentor `share × WAGE_FRACTION_MENTOR_FEE (0.1)` (skipped if mentor == beneficiary or no `EconomicAgent`); residual `share × 0.5` refunds to beneficiary on despawn. Deliberate-practice 2× XP via `apprenticeship::xp_with_apprentice_bonus(base, Option<&ApprenticeOf>)` threaded into Crafting XP sites (`crafting.rs` per-tick + recipe-completion; `corpse.rs` butchering) at `APPRENTICE_XP_MULT (2)`.
- **Skills (`skills.rs`):** `[u32; 8]` clamped to `SKILL_MAX=255` — Farming, Mining, Building, Trading, Combat, Crafting, Social, Medicine. Default 5; `gain_xp()` clamps at `SKILL_MAX`. Companion components `SkillPeaks`, `SkillUseTicks`, `SkillsLastSeen` (all `[u32;8]`) drive the Phase 1 decay model: `skill_peaks_tracker_system` ratchets peaks up and stamps `use_ticks[i] = now` whenever a slot increases (observer over `Changed<Skills>`); `skill_decay_system` runs daily and shrinks each skill toward a peak-derived floor — `peak ≥ SKILL_MASTERY_LINE (80)` → floor `SKILL_MASTERED_FLOOR (30)`; otherwise floor `max(SKILL_FLOOR_BASE (5), peak * 0.30)`. Decay is exponential with `SKILL_DECAY_HALF_LIFE_DAYS=90`.
- **Bucketing:** agents sliced across 20 fixed `BucketSlot`s.
- **LOD:** `Full / Aggregate / Dormant`. Dormant skip sim. Focus-aware (see Game lifecycle). `cohort.rs` currently provides the non-destructive cohort surface: `CohortRegistry` summarizes Aggregate agents by `{ faction, settlement_or_camp, profession, age_band, wealth_band, lifestyle }`, `cohort_pin_full_sim_system` runs in ParallelA after camera LOD and promotes commanded/chief/drafted/combat agents back to `Full` via `PinnedFullSim`, and `can_demote_to_cohort` encodes the safety gate for the future despawn-backed aggregate pass.
- **Memory & gossip (`memory.rs`):** known locations + agent sightings, `u8` freshness decay. Tech awareness gossips between socializing agents via `knowledge::awareness_gossip_system`. `MemoryKind` is a 3-variant `Copy` enum: `AnyEdible`, `Resource(ResourceId)`, `Prey`.
- **Thirst pipeline (drink + sanitation + sickness, `drink.rs` + `sanitation.rs` + `medicine.rs`):** survival pressure peer-to-hunger; never damages `Health`. `Needs.thirst` decays at `THIRST_RATE = 4.0/s` (~2× hunger), gated by `THIRST_TRIGGER = 180.0` and `THIRST_SEVERE = 230.0`. `Needs::worst` and `avg_distress` (divisor 8) include thirst. `ThirstScorer` (Survival) fires `AgentGoal::Drink`; `htn_drink_dispatch_system` (ParallelB after `htn_eat_dispatch_system`) tries inventory `clean_water` first, then `nearest_fresh_drinkable_tile` (chebyshev rings, skips salt via `world::biome::water_kind_at`; severe thirst widens scan 2×). `drink_task_system` (Sequential after movement) consumes from inventory/hands or sips an adjacent fresh tile; raw (non-river) tile drinks bump sickness severity 60 via `apply_sickness_severity`, `SanitationMap`-contaminated tiles bump 140. Boiling = `CraftRecipe 13` (`raw_water × 2 + wood × 1 → clean_water × 1`, `FIRE_MAKING` gated, Workbench). Animals: `AnimalNeeds.thirst` (`0.06/s`), `AnimalState::Drinking`, `animal_water_seek_system` + `animal_drink_system` route to non-salt water; raw sources bump `AnimalNeeds.sickness`.
- **`SanitationMap`** sparse `AHashMap<(i32,i32), f32>` populated by `sanitation_emit_system` (daily) from `WastePile` via `1/(d²+1)` falloff inside `CONTAMINATION_RADIUS = 6`. `sanitation_decay_system` (daily) applies `2^(-1/4)` per-day multiplier, purges below `CONTAMINATION_FLOOR (0.05)`. `is_water_contaminated(tile)` returns true above `CONTAMINATION_DRINK_THRESHOLD (0.5)`. **Sickness** lives as a parallel `Sickness { severity, applied_tick }` Component (orthogonal to `Injury`, which is derived from `Body` damage). `sickness_decay_system` (daily) subtracts `SICKNESS_DECAY_PER_DAY = 16`. `sickness_work_factor(severity) -> [0.5, 1.0]` multiplies both `work_progress.saturating_add` sites in `movement_system`, scaling every movement-driven executor. `agent_defecation_system` spawns a `WastePile` per Person every `DEFECATION_INTERVAL_TICKS (2 game-days)` (per-agent stagger via `Entity.index() % interval`); a `Latrine` entity within `LATRINE_ROUTING_RADIUS (8)` marks the pile `LatrineContained` (reduced emit factor). `BuildSiteKind::Latrine` (right-click → Build Latrine, `2 wood + 1 stone`, no tech gate, 50 work_ticks) finalizes to a `sanitation::Latrine` marker with `StructureLabel("Latrine")`. `corpse_decay_system` drops `WastePile(intensity=1.5)` on despawn. Deferred follow-ups in `plans/thirst-and-drinking-system-plan.md`: chief water stockpiling + worker `CollectCleanWaterMethod` / `BoilRawWaterMethod`; inspector Sickness readout.
- **Needs:** 7 needs (hunger, thirst, sleep, shelter, safety, social, reproduction), `[0,255]`, decay over time. Maslow tiers (`Physiological → Safety → Belonging → Esteem → SelfActualization`) layer on top via `MaslowTier::next_unmet(needs)`. **Decay rates are anchored on the 180-sec game-day:** `HUNGER_RATE = 2.0/sec` (180 → `EAT_TRIGGER_HUNGER` in ~90 s → eats ~2×/day), `SLEEP_RATE = 1.2/sec` paired with `SLEEP_RECOVER_RATE = 6.0/sec` (12.0/sec on a bed) (sleepy by Dusk/Night; recovers within the ~27-sec Night window). `WILLPOWER_WORK_DRAIN = 1.8/sec`, `WILLPOWER_IDLE_DRAIN = 0.15/sec`, `WILLPOWER_SLEEP_RECOVER = 3.0/sec`. `con_scale` (Constitution modifier) still attenuates hunger+sleep decay with floor 0.25×.
- **Goal-update cadence (`goal_update_system`):** active agents re-evaluate every **200 ticks (10 s)**; idle agents bypass the gate. Target validation is narrow — only despawned `target_entity` invalidates (Gather/Scavenge/Construct/HaulMaterials). Stage/kind drift on a still-alive target is checked at arrival in the executors (`gather::finish_gather`, `items::finish_scavenge`, `construction`), feeding `MethodOutcome::FailedTarget` into `MethodHistory`.

## Method-design rules

- **Method scoring = viability + distance, with time-of-day/fatigue scaling for AcquireFood / AcquireGood{wood,stone}.** Base utility tiers: `UTIL_BASELINE=1.0` / `UTIL_VISIBLE_GROUND=1.5` / `UTIL_CLAIMED_HAUL=2.0` / `UTIL_EXPLORE_FALLBACK=0.3`. Dispatcher subtracts a chebyshev-distance penalty capped at `MAX_DIST_PENALTY=0.30` so inter-tier ranking is preserved. Recent failures bias via `MethodHistory` + `score_method_with_history` (`METHOD_FAILURE_PENALTY=0.4`, `METHOD_HISTORY_LEN=6`, `METHOD_HISTORY_TTL_TICKS=600`). Two stacked failures (-0.8) push a baseline method below the Explore fallback. **Context-aware penalty:** `PlannerCtx.scope` defaults to `ScoringScope::Geometric`. AcquireFood and `GatherWood`/`GatherStone` swap to `ScoringScope::ContextAware { time_phase, dusk_remaining, fatigue }` via `context_aware_scope(&calendar, needs)`. `dist_penalty` / `full_trip_penalty` multiply the geometric value by time-of-day (Day 1.0 / Dawn 1.10 / Dusk 1.0→2.0 ramp / Night 4.0) and fatigue (`1.0 + fatigue`, `fatigue = 0.6·sleep_norm + 0.4·(1−willpower_norm)`); cap rises to `MAX_DIST_PENALTY_NIGHT=1.50` only at Night so 16-tile gathers drop below the explore floor. Phase cuts in `world/seasons.rs` (`PHASE_DAWN_START=0.00` / `PHASE_DAY_START=0.05` / `PHASE_DUSK_START=0.65` / `PHASE_NIGHT_START=0.85`).
- **In-hand fast-path for haul dispatchers.** Both `htn_acquire_good_dispatch_system::Haul branch` and `htn_build_claimed_blueprint_dispatch_system` Path B check the agent's `Carrier + EconomicAgent` inventory before issuing `WithdrawMaterial`; if the agent already carries the targeted resource and the bp's slot still needs it, they dispatch `Task::HaulToBlueprint` directly. Strictly scoped to dispatchers running on a held `JobClaim::Haul` (or personal-bp ownership). **Never count agent inventories in posting creation or chief blueprint-candidate scoring** — those layers must read deposited storage only.
- **Farming is `Farm`-goal only.** Both halves are HTN-driven: `PlantFromStorage` plants; `HarvestPlant` harvests. Seed↔plant mapping centralised in `PlantKind::seed_resource()` + `PlantKind::ALL` (`plants.rs`). Adding a seed/plant pair = one `PlantKind::ALL` entry + arm in `seed_resource()`.
- **Faction storage refresh:** `FactionStorage.totals` rebuilt every Economy tick by `compute_faction_storage_system`. HTN dispatchers read these directly when scoring.
- **`MF_UNINTERRUPTIBLE`:** methods set this when their multi-leg chain must survive across goal-dispatch ticks.

## Player command authority (`player_command.rs`)

UI inserts player input as `PlayerCommandEvent` (event bus). The sim drains in `SimulationSet::Input`, attaches a `Commanded { command, status, issued_tick, command_id }` component per actor, then `dispatch_player_command_system` (ParallelB) routes the command into the typed-task pipeline. **All routing happens sim-side** — UI never mutates `PersonAI` / `ActionQueue` / authority markers directly.

- **Variants:** `Move`, `Gather`, `Mine`, `Build`, `Deconstruct`, `DigDown`, `PickUpItem`, `PickUpCorpse`, `AttackEntity`, `Teach`, `HoldLecture`, `ReadItem`, `EncodeTablet`, `Muster`, `Disband`, `MilitaryMove`, `MilitaryAttack`, `PackCamp`, `PitchCamp { tile, z }`. Pack/Pitch are faction-scoped (chief actor) — dispatcher writes into `nomad::PendingCampOps`, drained by `apply_pack_camp_command_system` / `apply_pitch_camp_command_system` (Sequential, exclusive `&mut World`).
- **Goal forcing replaces filter scatter.** `goal_update_system` reads `Commanded` via `GoalValidationQueries.commanded_q`; non-terminal forces `AgentGoal::FollowingPlayerCommand` and `continue`s. HTN dispatchers don't recognize the goal so they naturally skip. No `Without<PlayerOrder>` filters; legacy `PlayerOrder` / `PlayerOrderKind` are gone.
- **Lifecycle (`player_command_lifecycle_system`, Sequential, after executors):** Move/MilitaryMove → chebyshev arrival; Gather → target despawned OR agent idle; Mine → tile no longer Wall/Stone; Build → blueprint despawned; PickUp* → target gone; Attack/MilitaryAttack → foe dead; Muster/Disband → immediate; Knowledge orders → on idle. `reap_terminal_commands_system` strips `Commanded` once status is `Completed | Failed(_) | Superseded`; UI reads `RemovedComponents<Commanded>` for feedback. **MilitaryMove arrival reads `PersonAI.dest_tile`** (which holds the per-actor slot tile for multi-actor formation moves), not the click anchor — otherwise a single unit reaching the anchor would terminate the whole group's order.
- **Multi-unit `MilitaryMove` formations (`military/formation.rs`).** A `MilitaryMove` event with `actors.len() > 1` is expanded into a compact ring of per-actor slot tiles around the click anchor. `expand_military_move_system` (Input, alongside drain; independent `EventReader`) walks each event, snapshots actor positions, builds an `is_passable` closure over `ChunkMap` + `DoormatReservations` + `SpatialIndex.agent_occupied` (selected actors' own tiles always pass so a tight group can reshuffle without self-blocking), runs `plan_compact_ring(anchor, n, is_passable)` (deterministic cardinal-then-diagonal ring walk, cap `MAX_R = 16`), then `greedy_assign(anchor, &actors, &slots)` matches actor → slot in ascending Chebyshev-from-anchor order. Output lives in `PendingFormationSlots: AHashMap<Entity, Option<FormationAssignment>>`, cleared each tick. `dispatch_player_command_system`'s `MilitaryMove` arm reads the side-table: `Some(a)` → route to `a.slot_tile` + insert `MilitaryFormationSlot { anchor, slot_index, group }`; `Some(None)` → `Failed(Unreachable)`; absent → single-actor move, anchor == slot (byte-equal to pre-formation behaviour). The anchor (not the slot) seeds `HotspotKind::RallyPoint` so the whole group shares one flow field. `MilitaryFormationGroupGen` allocates one `u32` group id per multi-actor dispatch (wrapping). `MilitaryFormationSlot` is removed on supersede (pre-`aq.cancel()` path in the dispatcher) and on terminal status (`reap_terminal_commands_system`); the `Task::WalkTo` arrival semantics on `military_task_system` are unchanged. `MilitaryAttack` is deliberately untouched — attacks converge on a foe, not a ring.
- **Faction-level commands** (currently `EncodeTablet`) carry an empty `actors` list and apply directly in `drain_player_command_events_system` (writes `PlayerCraftRequest`).
- **Schedule:** `Input → ParallelA → ParallelB → Sequential → Economy`. `Input` is exclusive to player-command drain; events deliver before the next FixedUpdate tick.
- **Adding a new order type:** new `PlayerCommand` variant + `dispatch_one` arm + `player_command_lifecycle_system` arm (all `player_command.rs`) + UI button emits the event.
- **UI surface:** right-click menu (`ui/orders.rs`) maps `MenuAction → PlayerCommand` via `menu_action_to_command`. HUD muster/draft, inspector Read/Lecture/Encode, military right-click all emit `PlayerCommandEvent`. No UI code touches `PersonAI` / `ActionQueue` directly.

## ActionQueue and typed Task variants (`typed_task.rs`)

`aq.current: Task` is canonical "task running now"; `Task::Idle` default; every Person spawn site bundles `ActionQueue::idle()`. Behind `current` sits a fixed-capacity `queued: [Task; 4]` prefetch ring (private; access via `enqueue` / `pop_next` / `peek_next` / `queued_len` / `queued_is_empty` / `clear_queued`).

- **Producers** (HTN dispatchers, `ui/orders.rs`, `teaching.rs::ReadItem`) route through `aq.dispatch(task)` — enqueues then promotes head into `current` if `current == Idle`.
- **Consumers** (executor exit paths in `gather.rs`, `dig.rs`, `corpse.rs`, `construction.rs`, `items.rs`, `production.rs`, `teaching.rs`, plus `MilitaryMove` arrival in `military.rs`) call `aq.advance()` instead of writing `current = Idle`.
- **External preempts** (`dispatch_player_command_system`, hunter demote, `goal_dispatch_system` stale reset) call `aq.cancel()`, dropping both `current` and the queue.
- Per-tick "pin" sites (lecture/teach pin writes in `teaching.rs`) stay as direct `aq.current = X` writes — idempotent re-assertions, not fresh dispatches.

**Variants:** `Idle`, `WalkTo { tile, z, why }`, `WithdrawGood { filter }`, `WithdrawMaterial { resource_id, qty }`, `WithdrawFood { tile }`, `Equip { slot, resource_id }`, `Construct { blueprint }`, `Gather { tile }`, `Dig { tile }`, `Scavenge { target }`, `Read/Teach/HoldLecture/AttendLecture { tech }`, `PickUpCorpse { corpse }`, `HuntPartyMuster { hearth }`, `Hunt { prey }`, `HaulCorpse { dest }`, `Butcher`, `TameAnimal { target }`, `Sleep { bed }`, `Eat`, `HaulToBlueprint { blueprint }`, `HaulToCraftOrder { order }`, `WorkOnCraftOrder { order }`, `DepositToFactionStorage { resource_id }`, `WalkAndTakeFromMember { target, resource_id, qty }`, `PlayThrow`, `PlayPlant { tile }`, `Explore { kind }`, `Migrate`, plus `Lead/Defend/Raid/RescueAlly/Socialize/Play` (faction/social).

**Rules:**

- Systems mutating typed task **must** include `&mut ActionQueue` alongside `&mut PersonAI`.
- Every executor exit (success/abort/timeout/precondition fail) must `aq.advance()` (or `aq.cancel()` for chain-drop).
- External teardowns clear both `aq` and the legacy `task_id`.
- `military_task_system`, `withdraw_good_task_system`, `withdraw_material_task_system` fall back to `Idle` if `task_id == X` but `aq.current` is the wrong variant — defence in depth.

## HTN domain (`htn.rs`)

- **`AbstractTask` / `AbstractTaskKind`** — high-level goal a method decomposes (`Sleep`, `Eat`, `AcquireFood`, `AcquireGood { resource_id }`, `StockpileFood`, `Scout`, `EquipHuntingSpear`, `ReturnSurplus`, `TameWildHorse`, `PlantFromStorage { resource_id }`, `HarvestPlant`, `ConstructBlueprint`, `Socialize`, `DeliverMaterialToCraftOrder { resource_id }`, `WorkOnCraftOrder`, `HarvestGrainForCraftOrder`, `Play`, `JoinHuntParty`, `EngagePrey`, `DeliverHuntKill`, plus four single-step combat/faction goals).
- **`Method` trait** — `precondition` / `utility` / `expand(ctx) -> Vec<Task>` / `flags` / `tech_gate` / `profession_gate` / `policy_gate` / `name`. `policy_gate` returns `&'static [(ResourceId, RequiredFlag)]` for methods that require a specific `ResourceControlPolicy` flag.
- **`MethodRegistry`** — `Resource`, `AHashMap<AbstractTaskKind, Vec<Box<dyn Method>>>`. Built once in `SimulationPlugin::build` via `register_builtin_methods`. Read-only after init.
- **`MethodFlags`** — plain `u8` bitmask. `MF_UNINTERRUPTIBLE` survives goal flips.
- **`MethodId`** — `u16` newtype with one `pub const` per registered method. Stable identity for `MethodHistory` keying and `PersonAI.active_method`.
- **`MethodHistory`** — per-agent ring (`METHOD_HISTORY_LEN=6`, TTL 600 ticks) of `(MethodId, MethodOutcome, tick)`. Spawned at every Person spawn site.
- **Failure-biased scoring:** multi-method dispatchers score via `score_method_with_history(method, abstract_task, ctx, history, now) = utility - failures * METHOD_FAILURE_PENALTY` (penalty 0.4). Routing/missing-target failures push `FailedRouting` / `FailedTarget`.
- **Cancel paths record failure.** Every executor `aq.cancel()` path that drops a chain due to invalid target / unreachable trailing leg pushes the agent's `active_method` onto `MethodHistory` before cancelling, via `record_target_failure(...)` (target loss) or `record_routing_failure(...)` (chain trailing-leg). Sites: `gather::finish_gather`, `items::item_pickup_system`, `items::finish_scavenge`, `production::finish_withdraw_material`. `MethodId::UNKNOWN` is a sentinel for cancels where `active_method` was already cleared.
- **Goal-flip Abandoned (`record_abandoned_method_system`).** ParallelA, after `goal_update_system`, filtered on `Changed<AgentGoal>`. Pushes `MethodOutcome::Abandoned` against the agent's `active_method` and clears it — *unless* the method is `MF_UNINTERRUPTIBLE`, in which case `goal_dispatch_system`'s preserve-arm matrix keeps the chain alive across the flip. `MethodRegistry::flags_by_id` looks up the method's flags by stable id.
- **Terminal Explore fallback** (`htn_acquire_food_dispatch_system` + `htn_stockpile_food_dispatch_system`). When every registered method's precondition fails, the dispatcher falls through to a synthetic `Task::Explore { kind: AnyEdible }` rather than returning silently. Stamps `MethodId::TERMINAL_EXPLORE` on `ai.active_method`. Routes via the same `pick_explore_tile` + `assign_task_with_routing` path the regular `Task::Explore` arm uses.
- **Reachability-aware deposit pick** (`StorageTileMap::nearest_for_faction_reachable`). Dispatchers picking a deposit/withdraw storage tile filter out tiles whose exact `(tx, ty, nearest_standable_z)` isn't reachable from the source tile per `ChunkConnectivity::tile_reachable`, falling back to the connectivity-blind result when every tile is rejected. The `source_tile` is the worker's post-pickup tile (gather / scavenge tile), not the agent's pre-walk position — deposit chains gate on the leg the agent actually walks, not the pre-walk state. Wired into `htn_acquire_food_dispatch_system`, `htn_acquire_good_dispatch_system`, and `htn_stockpile_food_dispatch_system`.
- **Reachability-aware vision pickers** (`CurrentVision::nearest_gather_target` / `nearest_scavenge_target`). Both take an `is_reachable: impl Fn((i32,i32)) -> bool` closure with the same two-pass fallback. Each htn.rs dispatcher builds a `reach_from_agent` closure once and threads it into every visible-target pick — closes the symmetric gap where a Manhattan-closest visible plant or ground item sits across an impassable boundary. The `SharedKnowledge::nearest_in_tier_set` cluster path still uses Manhattan distance only.
- **Chronic-failure release** (`goals::chronic_failure_release_system`). Economy schedule, every `TICKS_PER_DAY/4`. For each agent (any goal except `Survive` / `Sleep`) with ≥3 non-success `MethodHistory` entries within the TTL window: stamps the goal into the `GoalCooldown` ring (`(goal_disc, expires_tick)` × 4 slots; oldest evicted on overflow); also writes the legacy `craft_until_tick` for `should_craft`'s fast path. Then *bifurcates*: a `JobClaim`'d agent has the claim removed + claimant slot released so the chief reposts to a better-positioned worker; an autonomous (no-claim) agent is inserted into `ForceGoalReevaluate` so `goal_update_system` bypasses the 200-tick cadence and flips the goal *next* tick. `Survive` / `Sleep` are exempt — terminal Explore (Phase 3) is the relief valve there. Orthogonal to `job_claim_release_system`'s stuck-idle `fail_count` bump (which fires on `task_id == UNEMPLOYED` for STUCK_FAIL_INTERVAL ticks); this one fires on the *quality* of recent attempts.
- **`ForceGoalReevaluate`** (`Resource`, `AHashSet<Entity>`). Drained by `goal_update_system` on each entity it processes — the agent's `task_id` is reset to UNEMPLOYED, target cleared, then the normal goal-selection logic runs. Ensures the new cooldown bites immediately rather than after the next 200-tick cycle.
- **Success recording:** dispatchers stamp `PersonAI.active_method = Some(method.id())` on successful `aq.dispatch(...)` and clear before pushing `Failed*`. `htn_method_completion_system` (Economy) observes `aq.current == Idle && queued_is_empty && active_method.is_some()` and records `Success`.

### Dispatch systems

All run in `ParallelB`, ordered after `goal_dispatch_system`. Each system serialises on `&mut PersonAI` / `&mut ActionQueue`; Bevy schedules them in parallel where component sets allow. The craft-order + Harvest + Play dispatchers share their own `add_systems` call (Bevy's 20-tuple ceiling). Spear-arming runs *before* food dispatchers so an unarmed hunter prefers fetching their spear over eating.

Each dispatcher: gates on goal + Idle + non-Dormant + `task_id == UNEMPLOYED`, builds `PlannerCtx`, runs argmax across registered methods, routes the head via `assign_task_with_routing`, prefetches the tail on `ActionQueue`. SOLO/unsettled agents skip storage-dependent methods.

**`ClaimTarget.kind: ClaimKind`** (`jobs.rs`) — `None | Specific(ResourceId) | AnyEdible`. Mirrors `MemoryKind`. Stockpile/Haul postings yield `Specific(rid)`; chief `Calories` postings yield `AnyEdible`; Build yields `None`. Food dispatchers gate on `ClaimTarget::is_food()` (true for `AnyEdible` or `Specific(rid)` where `rid.is_edible()`), so adding a new edible to `assets/data/resources/*.ron` automatically participates without code changes.

**Subsistence vs coordinated stockpile (`htn_stockpile_food_dispatch_system`).** `AgentGoal::GatherFood` fires in two regimes: (1) subsistence reflex — autonomous fallback when `prioritize_food` (food ratio < 1.0), no claim; (2) coordinated — agent holds `JobClaim::Stockpile{food}` from a chief/household/self-post. Claim is optional. Deposit target is household-first: if the agent has `HouseholdMember` and the household owns a `FactionStorageTile`, the chain ends in the household larder; else the village's. Runs after every specialised ParallelB dispatcher; filters out `Carrying` / `MigrationTarget` / `TraderPlan` carriers and `Profession::{Bureaucrat, Trader}` — subsistence is lowest-priority autonomous work.

### Chain handoffs

Trailing legs of multi-task chains live in exit helpers on the head executor:

- **`production::finish_withdraw_food`** — primes `task_id = Eat`.
- **`production::finish_withdraw_material`** — releases reservation; routes `HaulToBlueprint` / `HaulToCraftOrder` / `Planter`; primes `Equip` / `PlayThrow` / `Play` in-place.
- **`gather::finish_gather`** (5 exit sites) — routes `DepositToFactionStorage` to nearest storage; routes `HaulToBlueprint` / `HaulToCraftOrder`; primes `Eat` for the Forage chain. Routing failure drops the chain (`aq.cancel()`). Takes a `FinishGatherOutcome::{Completed, TargetInvalid}` parameter: `TargetInvalid` (stale plant, wrong tile kind) calls `aq.cancel()` so the queued tail isn't run with empty hands; `Completed` calls `aq.advance()` and routes the tail as before.
- **`items::finish_scavenge`** — mirror of `finish_gather`; primes `Eat` for the AcquireFood path. Same `FinishGatherOutcome` parameter — GroundItem despawned mid-walk fires `TargetInvalid` so the chain doesn't walk to storage empty-handed.
- **`drop_items_at_destination_system`** — executor for `TaskKind::DepositResource`; dumps hands at `dest_tile`, credits any `JobClaim::Stockpile`, `aq.advance()`s.

### Stale-reset / catch-all (`goal_dispatch_system`)

Runs before HTN dispatchers. Covers only:

- **Stale-task reset** — leftover `task_id != UNEMPLOYED` clears to `(Idle, UNEMPLOYED)` (and `aq.cancel()`) unless the goal legitimately keeps it (preserve-arms enumerate the (goal, task_id) pairs each HTN dispatcher's chain needs to survive). Catch-all at bottom resets `Explore` arrivals.

### Adding a method

New `AbstractTask` variant (if needed) + `Method` impl + register call in `register_builtin_methods`. Method tests in `htn::tests` exercise the trait directly without an `App`. New abstract-task *kinds* also need a per-goal dispatch system (or new branch).

## Memory & gathering (`shared_knowledge.rs`)

3-tier `SharedKnowledge` map is the system of record for static-resource queries.

- **`SharedKnowledge` resource** — per-tier `KnowledgeMap`s keyed by `KnowledgeTier::{Household(u32), Settlement(SettlementId), Faction(u32)}`. Resources stored as `ResourceCluster` influence nodes (center + radius + LRU `representative_tiles[4]` + `estimated_count`); `report_sighting` merges within `CLUSTER_MERGE_RADIUS=8`, `report_depleted` decrements and despawns at zero. `nearest_in_tier_set(TierSet, kind, from, owner_filter, claim_penalty)` walks finest-tier-first. `cluster_decay_system` drops clusters older than `CLUSTER_DECAY_TTL_TICKS=25200` daily. `estimated_count` mirrors rep-slot occupancy. `ResourceCluster::nearest_target_tile` returns `None` when every rep slot is empty; `KnowledgeMap::nearest` filters such clusters out.
- **Cluster-saturation skip:** `nearest_with_cluster_filter` and `nearest_in_tier_set_with_cluster_filter` accept a `Fn(&ResourceCluster) -> bool` predicate. `GatherKnowledge::nearest_target_tile` injects `gather_claims.cluster_is_saturated(...)` to skip clusters at `MAX_PARALLEL_GATHERERS_PER_CLUSTER = 3`.
- **Pressure-aware rep selection:** `ResourceCluster::pick_least_pressured_rep(from, penalty)` minimises `chebyshev(rep, from) + penalty(rep)` across LRU rep slots. Plumbed into `KnowledgeMap::nearest` and `nearest_in_tier_set` so workers fan out.
- **`ResourceOwner`** — `Public | Person(Entity) | Household(u32) | Settlement(SettlementId) | Faction(u32)`. `is_accessible_to(viewer, household, settlement, faction)` gates harvest without theft semantics.
- **`LandClaim` on Plants** — stamped at planting (`production.rs` Planter arm): `HouseholdMember` → `Household(id)`; chief Farm posting → `Faction(id)`; else → `Person(self)`. Wild reseed → no `LandClaim` (Public).
- **`GatherClaims` (`gather_claims.rs`)** — mutex-wrapped per-`(tile, kind)` reservation map. `pressure(tile, now, viewer)` feeds `claim_penalty` ×4 weight so agents fan out. Staked at dispatch by all four resource-target dispatchers post-`assign_task_with_routing`, TTL from `suggested_expiry`. Released at every `gather::finish_gather` exit and on `goal_dispatch_system` stale-cancel. `CurrentVision::nearest_gather_target` consumes the same weight. `gather_claim_expiry_system` (Economy, every 150 ticks) sweeps. Cluster-level: `pressure_across_tiles(rep_tiles, now, viewer)`, `cluster_is_saturated(rep_tiles, now, viewer)`.
- **Vision write-through (`memory.rs::vision_system`)** — every active agent reports plant/item/prey/stone sightings to `SharedKnowledge`'s finest tier (Household if member, else Faction). Settlement/Faction tiers materialise via gossip. Vision is additive only — `report_depleted` is never called from vision. Cluster shrinkage happens at gather-arrival via `gather_system::report_depleted` on every `finish_gather` exit. Empty-arrival paths push `MethodOutcome::FailedTarget` into `MethodHistory` and clear `ai.active_method`.
- **Vision-first dispatch (`CurrentVision`)** — `vision_system` populates per-agent `CurrentVision { entries: Vec<VisionEntry> }` (cleared per bucket-active pass). Entries record `{ kind, tile, entity, owner }` for plants / GroundItems / prey / stone seen this pass; food ground items dual-write under `MemoryKind::AnyEdible` + `MemoryKind::Resource(rid)`. The four resource-target dispatchers consult `nearest_gather_target` / `nearest_scavenge_target` before `gk.nearest_target_tile`. Buffer freshness bounded by bucket cadence (~20 ticks); never feeds depletion.
- **Stale-target neighbor-scan retarget (`gather.rs`):** when `gather_system` arrives to find the plant despawned or not `Mature`, `retarget_neighbor` scans chebyshev `P6B_RETARGET_RADIUS = 2` for a same-`MemoryKind` mature plant. On hit: re-route, swap cluster claim, rewrite `aq.current`, reset `work_progress`, stamp `last_retarget_tick`. Throttled to one swap per `P6B_RETARGET_COOLDOWN = 40` ticks; after cooldown, stale arrivals fall back to `FailedTarget`.
- **HTN under-foot fast path (`plants.rs`):** `nearest_mature_plant_under_agent(plant_map, plant_query, kind_filter, from, radius=2)` probes `PlantMap` directly. ## Knowledge & technology

- **Per-person (`knowledge.rs`):** `PersonKnowledge` carries `aware` and `learned` (`u64` bitsets, learned ⊆ aware) plus `learned_at: [u32; 64]`. `complexity(tech)` is era-based (Paleolithic 1 → Bronze Age 5; Cuneiform/Lunar/City-State 6). No cap on Learned techs. Learning speed slows with load: `learning_slowdown(stats, k) = 1 + complexity_used / (intelligence × 2)`; applied to `add_study_progress` (divides amount, floor 1) and to passive teaching's chance roll. Per-action discovery is unaffected. Founders spawn via `paleolithic_seed`; newborns inherit `Aware ∪ Learned` from both parents. `study_progress: AHashMap<TechId, u32>` accumulates scaled ticks; at `study_threshold(tech) = complexity * STUDY_TICKS_PER_COMPLEXITY (3600)` runs `try_learn`.
- **Discovery (per-action):** `discovery_system` consumes `DiscoveryActionEvent`s from `gather_system`, `production_system`, `combat_system`, `social_fill_system`. Eligible = prereqs Learned + not already Aware. Roll `base × (1 + INT_mod × 0.1) × (1 + skill_xp / 1000)`, cap 0.5; success grants Aware and jump-starts `study_progress` by `complexity × INSIGHT_PROGRESS_PER_COMPLEXITY (1200)` capped strictly below threshold. Mastery still requires Read/Lecture/Teach or passive teaching.
- **Awareness gossip:** `awareness_gossip_system` (Economy) — adjacent socialising agents OR each other's `aware` bitsets per tick; also OR-merges `AgentMemory.visited_settlements`.
- **Passive teaching:** `tech_teaching_system` (Economy, after gossip) scans pairs within 3 tiles, finds Learned-by-teacher + Aware-but-not-Learned-by-student techs, rolls `0.004 × INT_scale / learning_slowdown(student)`. Highest-complexity teachable tech transfers; lands directly in Learned.
- **Directed teaching (`teaching.rs`):** three `study_progress` pathways; per-tick credit = base × `INT_scale` ÷ `learning_slowdown`. **Read** `Task::Read{tech}`: `+1`/tick (inspector button on Aware-only rows with matching tablet/book). **Lecture**: inspector button → `LectureRequest`; `apply_lecture_request_system` drafts ≤8 same-faction adults within 6 tiles, inserts `Lecturing { ends_tick=now+600, tech, anchor }` + `Attending`; `lecture_tick_system` (Sequential, after movement) credits `+2`/tick. **1-on-1 Teach**: right-click → `TeachingPair`/`BeingTaught` (ends_tick=now+120); `teach_task_system` `+3`/adjacent-tick. All carry `Drafted`.
- **Faction-tech derivation:** `FactionData.techs` is a chief-Aware cache projected every tick by `sync_faction_techs_from_chief_system` (Economy); used for planning / posting authority (`can_direct_tech`).
- **Community adoption (`technology_adoption.rs`):** `FactionData.tech_adoption: [AdoptionStage; TECH_COUNT]` derived by `derive_tech_adoption_system` (Economy, every `ADOPTION_DERIVE_CADENCE = TICKS_PER_DAY/4 = 900` ticks). Six stages: `Unknown / Rumored / Demonstrated / Practiced / Adopted / Institutionalized`. Per-tech `AdoptionScale` (Personal / Household / Subsistence / Specialist / MilitaryTransport / Institutional) drives thresholds — Specialist needs workshop + recent crafts; Institutional needs chief-learned + prereqs adopted + civic building + population scale; Personal/Subsistence land at Adopted under broad-learning shortcuts. `RecentTechUse` per-(faction, tech) ring (cap 8, TTL 60 days). Gating surface: `can_direct_tech(faction, tech)` = chief-Aware (planning); `community_has_adopted(faction, tech)` (civic/tier/material gates, also `FactionData::community_has`); `worker_can_perform(person, tech)` = Learned. `community_adoption_bitset(faction)` packs Adopted into a `FactionTechs` snapshot so `best_X_for(&FactionTechs)` helpers keep their signatures. `current_era(&community_adoption_bitset(faction))` projects era as a label only; real gating is per-tech. Per-person execution gates use `has_learned` (HTN `tech_gate`, Craft claim, `HUNTING_SPEAR`, `HORSEBACK_RIDING`).
- **Decay:** `derive_tech_adoption_system` is symmetric — conditions falling away walks the stage down. Downgrades throttled to one stage per game-day via `stage_changed_at_tick`; upgrades land immediately. `step_down`: `Institutionalized → Adopted → Practiced → Demonstrated → Rumored → Unknown`. Upgrades to Adopted/Institutionalized emit `ActivityEntryKind::TechAdopted` / `TechInstitutionalized`; downgrades silent.
- **Founder seeding (`PersonKnowledge::seeded_realistic_through_era`):** chief learns Personal+Household+Subsistence+Specialist+Institutional techs of era ≤ E; every ~8th member is Specialist (+MilitaryTransport); rest carry only Personal+Household+Subsistence Learned. Everyone remains Aware of the full era. `seeded_through_era` (everything Learned) retained for tests/fixtures.
- **Tablets and books (`Item.tech_payload: Option<TechId>`):** `Good::ClayTablet` (OneHand, 1500 g) and `Good::Book` (Small, 600 g). Recipes gated on `CUNEIFORM_WRITING`, Workbench. Reading doesn't consume.
- **Tablet posting (`chief_tablet_posting_system`):** every `CHIEF_TABLET_POSTING_INTERVAL=3600` ticks. Picks highest-complexity chief-Learned bit that <50% of adults are Aware of (skipping live tablet postings), posts `JobKind::Craft` with `tech_payload=Some(chosen)`. Player override via `PlayerCraftRequest` (inspector "Encode"): consumed every tick, `JobSource::Player`/priority 180.
- **Tech tree (`technology.rs`):** 44 techs (incl. `PORTABLE_DWELLINGS`), 5 eras, prereq DAG, per-tech `triggers: &[TechTrigger]`. `TechBonus` adds yield/storage/combat bonuses; bonuses aggregate via `faction.techs` (chief-aware).
- **Tech panel UI (`ui/tech_panel.rs`):** Per-tech icon now reflects `AdoptionStage` not chief-Aware: ◆ Institutionalized (bright green) / ✓ Adopted (green) / ● Practiced / ◐ Demonstrated / ◉ Rumored / ◎ Discoverable / ○ Locked. ## Hunting pipeline (`corpse.rs` + the three-phase HTN pipeline)

- Wolf/Deer no longer drop Meat/Skin on death. `combat.rs::death_system` strips AI/needs/species and inserts `Corpse { species, fresh_until_tick }`, indexed in `CorpseMap`. `corpse_decay_system` (Economy) despawns at `CORPSE_FRESHNESS_TICKS=600`.
- **Chief hunt orders (`HuntOrder` on `FactionData`):** `chief_hunt_order_system` posts daily (`TICKS_PER_DAY=3600`, staggered by `fid`). Scans `SpatialIndex` within `HUNT_SCAN_RADIUS=40`, posts `Hunt { species, area_tile, target_party_size = 4(Wolf)/2(Deer), mustered, deployed_tick }` or `Scout`. Mid-day invalidation sweep clears stale orders.
- **HuntFood pipeline:** three HTN abstract tasks (hunter-only, `HUNTING_SPEAR`-gated, `MF_UNINTERRUPTIBLE`): `JoinHuntParty` (Muster + Travel) under chief `HuntOrder::Hunt`; `EngagePrey` (Hunt + PickUpCorpse) at the area; `DeliverHuntKill` (HaulCorpse + Butcher) gated on `Carrying`. Butcher drops `species_yield()` Meat+Skin, despawns corpse.
- **Weapon precondition:** `HuntPreyMethod` / `MusterAtHearthMethod` / `TravelToHuntAreaMethod` all gate on `PlannerCtx.agent_has_weapon` (true when a `weapon` resource sits in `Equipment[MainHand]`, `Carrier` hands, or `EconomicAgent.inventory`). The engage / join-party dispatchers compute the field by reading `&EconomicAgent + &Carrier + Option<&Equipment>`. `PickUpFreshCorpseMethod` deliberately skips the gate (no combat — agent just carries the kill). An unarmed hunter under `HuntOrder::Hunt` therefore can't dispatch `Task::Hunt` / `Task::HuntPartyMuster` and falls through to `htn_equip_hunting_spear_dispatch_system`.
- **Scout:** `htn_scout_dispatch_system` + `ScoutForPreyMethod` emits `Task::Explore { kind: Prey }` while chief holds `HuntOrder::Scout`; chief flips to Hunt the moment `vision_system` writes a prey sighting.
- **EquipHuntingSpear:** **goal-agnostic** — runs for any unarmed Hunter (no goal filter) so a hunter mid-`Lead` / `Defend` / `Socialize` still fetches their spear when `HuntOrder::Hunt` is live. Gates: Hunter profession + Learned `HUNTING_SPEAR` + no weapon in any slot + faction storage has weapon stock. Chain preservation across goal flips keys on `ai.reserved_resource == Some(weapon())` for the `WithdrawMaterial` leg and `Task::Equip { resource_id == weapon() }` for the `Equip` leg, so the chain survives external goal-update cycles without explicit per-goal arms.
- **Live chase (`combat::hunt_chase_system`):** Sequential schedule, after `movement::sync_indexed_after_move_system` and before `combat::combat_system`. Reads `aq.current = Task::Hunt { prey }`, looks up the prey's `Transform + Health`, and acts:
  - Prey despawned → `record_target_failure` + `aq.cancel()` (the next dispatch tick re-evaluates with the recency bias).
  - Prey dead → no-op; `combat_system::death_system` handles `aq.advance()` in the same tick.
  - Prey moved (`prey_tile != ai.dest_tile`) and Chebyshev distance ≤ `HUNT_LEASH_RADIUS (30)` → update `ai.dest_tile` / `ai.target_tile` / `ai.target_z` to the live prey tile; if state was `Working`, drop back to `Seeking` (so `movement_system`'s `Following`-arm replan picks up the goal mismatch); clear `PathFollow.segment_path` + status → `Idle` to force a fresh A* request.
  - Prey beyond leash → cancel with `FailedTarget` so the dispatcher can pick a closer prey or transition to Scout.
- `corpse_follow_system` (Sequential, after `movement_system`) snaps corpse Transform to carrier. `respond_to_distress_system` recruits any same-faction Hunter within `HUNTER_RESPOND_RANGE=50` regardless of LOS, removing `Carrying`.
- **`Carrying(Entity)`** — marker for "carrying corpse E"; inserted at pickup arrival, removed at butcher / decay / rescue / muster / hunter-demote.

## Plant lifecycle (`plants.rs`)

Calendar-driven, season-edge-triggered. Replaces the previous per-frame tick clock — season modulates the *rate* of growth, not whether transitions can fire. `Plant.growth: u16` is a stage-time accumulator; on each season change `plant_lifecycle_system` adds `season_growth(kind, prev_season)` and runs at most one stage transition per plant.

- **Per-season growth rates (`season_growth`):** Winter is dormant for all kinds (zero growth, lethal for Grain). Per kind: Grain Spring/Summer/Autumn = 4/5/2; BerryBush 5/4/2; Tree 4/5/3.
- **Stage thresholds (`stage_threshold`):** Grain Seed/Seedling/Mature = 3/5/2 (annual; no Harvested stage). BerryBush 5/30/4 with Harvested mirroring Seedling at 30 → ~3 calendar years to first fruit. Tree 12/48/18 with Harvested mirroring Seedling → ~4 calendar years from seed to maturity.
- **Sprout chance:** Seed → Seedling rolls `fastrand::f32() < 0.20`; failure despawns the entity. Combined with §scatter, this culls exponential spread.
- **Mature → fruiting:** at the transition, a single dice roll fires (`PlantKind::Grain` 20% / `BerryBush` 10% / `Tree` 5%) for one Seed in chebyshev radius (Grain/Berry r=2, Tree r=3) onto a `Grass | Farmland` tile. Post-roll Grain enters Overripe, BerryBush reverts to Harvested, Tree reverts to Mature.
- **Winter mortality:** at Winter onset, every Grain plant despawns. Mature grain performs the scatter dice first (the "wheat goes to seed in winter" path). Immature grain just dies.
- **System scheduling:** `plant_lifecycle_system` runs in `FixedUpdate` `SimulationSet::Sequential`, ordered after `advance_calendar_system`. Edge-triggered via `Local<Option<Season>>` — fires only on season change (≤4 calls per game year).
- **No fertility multiplier on growth speed.** Removed when migrating off `growth_ticks`; can return as a per-season bonus or yield modifier later.
- **`deer_graze_system`** still resets a grazed bush to `Harvested` with `growth = 0`; nothing else in the file changed externally.

## Animal spawn distribution (`animals.rs::spawn_animals`)

Initial-condition only. Per-species `SocialPattern` picks a `(group_min..=group_max, cluster_radius)`: `HERD` (Deer/Horse/Cow — 8–15, r=5), `PACK` (Wolf/Pig/Rabbit/Fox — 3–6, r=3), `SOLITARY` (Cat — 1, r=0). `cluster_spawn_tiles` pops shuffled centers from the species' biome pool (`forest_tiles` / `grass_tiles`) and lays members on tiles inside the biome `AHashSet` within `cluster_radius` of the center. Total counts (`WOLF_COUNT`, `DEER_COUNT`, …) and per-species component bundles are unchanged. Runtime cohesion is not modeled — `animal_movement_system` wanders independently.

## Nomadic mode (`nomad.rs`, `nomad_pool.rs`, `wild_herd.rs`, `pack_deploy.rs`, `sedentary_collapse.rs`)

See root `CLAUDE.md` for the high-level survey. Detailed system list:

- **`Lifestyle::{Settled (default), Nomadic}`** on `FactionData`. Households inherit parent village's lifestyle via `spawn_household` alongside `land_policy`.
- **`Deployable { packed_form, refund_pct, refund_resource, refund_qty }`** (`pack_deploy.rs`) on every nomadic structure. `Deployable::fully_packable(rid)` for full-carry shelter (Bedroll/Yurt); `Deployable::refund_only(pct, resource, qty)` for sticks-and-leaves teardown (Tent uses `refund_only(0.5, wood, 6)` → 3 wood drops as `GroundItem` on migration teardown). `compute_refund_drop()` returns `floor(qty × pct)`.
- **Build sites:** `BuildSiteKind::{Bedroll, Tent, Yurt}`. Bedroll (1 skin + 2 wood, no gate) finalizes as `(Bed { tier: Crude }, Deployable, StructureLabel("Bedroll"))` — existing sleep dispatch finds it via `BedMap`. Tent (6 wood + 3 skin). Yurt (8 wood + 6 skin, gated on `PORTABLE_DWELLINGS`, Workbench station) packs into `packed_yurt` good (`bulk: two_hand`, `weight_g: 80000`).
- **Camp seeding** (`seed_nomadic_camp`): hearths via `paleolithic_hearth_positions`, then `seed_bedrolls_around_hearth` (radial 2..=5, one per founder), `seed_tents_around_hearth` (outer 5..=7, ~1 per 4 founders), `seed_yurts_around_hearth` (Neolithic+, inner 3..=5, capped 2). `auto_found_default_settlements_system` short-circuits on nomadic; `spawn_population` skips the per-faction `FactionStorageTile` spawn for nomads.
- **Camp lifecycle (`FactionData.camp_state` + `migration_phase` + `nomad_autopilot`):** every nomadic faction carries `CampState::{Pitched, Packed { since_tick }}` and `MigrationPhase::{Idle, Surveying { started_tick, scouts, quadrants }, PendingCommit { target, chosen_tick }, PackingCamp { target, old_home, started_tick, radius }, Traveling { target, old_home, departed_tick, caravan_tile }, PitchingCamp { target, old_home, started_tick, pitch_started_tick, repair_unlocked }}`. AI factions (`nomad_autopilot == true`) run Surveying → PendingCommit → PackingCamp → Traveling → PitchingCamp → Idle. Player factions (`nomad_autopilot == false`, set by `spawn_population` for `Lifestyle::Nomadic`) skip the AI FSM — free-form Pack Camp → Move orders → Pitch Camp Here.
- **AI migration pipeline** (`nomad.rs`):
  - `nomad_migration_system` (Economy, daily, exclusive `&mut World`): for `Idle` factions past cooldown, scores local food clusters within `NOMAD_FORAGE_RADIUS = 24`. If `< members × 3`, picks 2–3 lowest-need non-Drafted/non-chief members, stamps `AgentGoal::Scout` + `ScoutAssignment { quadrant, target_tile, assigned_tick }`, transitions to `Surveying`. Quadrant targets at `SURVEY_SCOUT_RADIUS = 100` chebyshev from home seed faction-tier `SharedKnowledge` far beyond normal vision.
  - `nomad_survey_dispatch_system` (ParallelB) routes Scout agents via `Task::Explore { kind: AnyEdible }`; vision write-through populates clusters.
  - `nomad_survey_completion_system` (Economy, daily, exclusive): at `now - started_tick >= SURVEY_WINDOW_TICKS = TICKS_PER_DAY*4`, runs `pick_migration_target`, writes `pending_migration`, transitions to `PendingCommit`, strips Scout state, queues `ForceGoalReevaluate`.
  - `pick_migration_target`: score = food + herd + water + biome-season + danger + recency + continuous distance penalty (`-d * DIST_WEIGHT (0.4)`). `NOMAD_MIN_TARGET_DIST = 8` rejects on-camp candidates; `NOMAD_MAX_TARGET_DIST = 200` safety cap.
  - `nomad_migration_commit_system` (Sequential, exclusive) is now a lifecycle driver, not an atomic teleport. `PendingCommit` validates a loaded, passable, reachable final target before teardown. `PackingCamp` flips `camp_state = Packed`, stamps `PackingDuty`, and reuses `UnpitchStructure` labor for `Deployable` shelters plus no-cargo `Campfire` teardown. `Traveling` starts once old camp targets are gone, stamps every member with `MigrationTarget { route_tile: None, bounce_count: 0 }`, inserts `FollowingBand` on owned `Tamed`, and keeps `home_tile` at the old camp while `caravan_tile` tracks chief/centroid for UI/debug. `PitchingCamp` begins when any member reaches the final target; only then do `FactionData.home_tile` and `Camp.home_tile` move, recent-camps update, and CampMoved logs fire.
  - `pack_targets_remaining` / `dispatch_unpitch_tasks` radius from `seed_nomadic_camp_extent(members, era) = 8 + members/6 + 4`, clamped 12..=22. Pack targets include all `Deployable` structures and `Campfire`.
  - `MigrationTarget { tile, route_tile, started_tick, last_dispatched_tick, bounce_count }` + `WalkReason::Migration` + `TaskKind::Migrate`. `tile` is always the final camp. `bounce_count` (cap 2) reroutes to a reachable peer/centroid via `route_tile`; reaching a route tile clears only `route_tile`, never the final marker or camp state.
  - `nomad_migration_dispatch_system` (ParallelB, `ParamSet`-split): no `Without<Drafted>` filter (drafted hunters migrate with the band). On dead-end, snapshots `(fid → reachable peer tiles)`, sets `route_tile = median peer`, increments `bounce_count`; releases only at `bounce_count >= 2` with no reachable peer.
  - `nomad_migration_arrival_system` (Sequential, after movement): strips marker on chebyshev final-camp arrival within `MIGRATE_ARRIVAL_RADIUS = 4`, `MIGRATE_TIMEOUT_TICKS = TICKS_PER_DAY*3`, or `MIGRATE_STALL_TICKS = TICKS_PER_DAY/2` idle. Route-waypoint arrival only clears `route_tile`. Phase completion belongs to `nomad_migration_commit_system` once pitch/arrival thresholds pass.
  - `goal_update_system` carries `Option<&MigrationTarget>` + `Option<&ScoutAssignment>` (sub-tuple for 16-query ceiling). `goal_dispatch_system` preserve-arms: `(MigrateToCamp, Migrate)`, `(Scout, Explore)`.
- **Player Pack / Pitch (`PlayerCommand::PackCamp` / `PitchCamp { tile, z }`):** chief-actor faction-scoped, written to `PendingCampOps`. Dispatcher validates mobile faction, correct camp state, target passable + reachable. `apply_pack_camp_command_system` runs observable HTN labor: stamps non-drafted members with `PackingDuty` (preempts task chains, releases reservations, drops `JobClaim`), enumerates every `Deployable` within `seed_nomadic_camp_extent` of `home_tile`, dispatches `Task::UnpitchStructure { structure }` to nearest worker. `unpitch_structure_task_system` (`nomad_pack_labor.rs`, Sequential after movement) accumulates to `UNPITCH_WORK_TICKS = 40`, despawns entity, drops `packed_form` / `packed_bundles` / refund as `GroundItem`s. `camp_state = Packed` flips immediately so the goal gate halts settled-life work. `apply_pitch_camp_command_system` keeps the synchronous `seed_nomadic_camp` call for player-issued pitch.
- **Manual scout dispatch (`PlayerCommand::SendScout { direction, range }`):** chief-actor command queues `PendingManualScout`; `apply_manual_scout_command_system` stamps `ScoutAssignment { kind: ScoutKind::PlayerManual { faction_id }, .. }` on the least-needy non-Drafted non-chief member, `target_tile = home + direction*range`. `manual_scout_completion_system` runs `pick_migration_candidates(intent, k=4)` on arrival, marks `validated = true`, folds into `FactionData.candidate_sites` (ring cap `MAX_CANDIDATE_SITES = 16`). `PlayerCommand::SetMigrationIntent { intent }` writes `faction.migration_intent`; AI `pick_migration_target` consumes `intent.weights()` (six-multiplier vector).

Candidates carry per-tile `reasons: Vec<CandidateReason>` (FreshWater / Pasture / Herd / Wolves / PoorShelter / LongCarry / SnowRisk / EnemyFaction / Sanitation) from component-score thresholds.
- **Migration Panel UI (`ui/migration_panel.rs`):** egui side-panel from HUD "Plan Migration" button (mobile player factions only). Display-only — does not gate movement. Status / Intent picker (6 buttons) / Send Scouts (8 cardinals) / Candidate Sites list. Movement still flows through HUD "Pack Camp" + right-click "Move" / "Pitch Camp Here".

- **Mobile-state goal gate (`goals::mobile_state_goal_gate_system`, ParallelA after `goal_update_system`):** `CampState::Packed` members get settled-life goals demoted and `JobClaim` stripped. Player `Forage` uses `allowed_while_packed` (Survive / Sleep / GatherFood / Socialize / Defend / Rescue / Play / FollowingPlayerCommand / MigrateToCamp / Scout / care / Drink). AI `PackingCamp` / `Traveling` / `PitchingCamp` uses the stricter survival-only `allowed_while_ai_caravan`: Survive, Defend, Rescue, SeekCare, ProvideCare, Drink, FollowingPlayerCommand, MigrateToCamp.
- **Pitch conserves shelter count (`despawn_old_camp_leftovers` + `consume_band_packed_goods_after_pitch`):** `apply_pitch_camp_command_system` sweeps `Deployable` structures within `seed_nomadic_camp_extent` of the old `home_tile` (refund-only Tents drop materials, fully-packable shelters silently discard since they're already packed). After `seed_nomadic_camp` re-seeds, debits one `bedroll` / `packed_yurt` per spawned shelter from member then pack-animal inventories. Missing supply is forgiven (first pitch); subsequent cycles conserve exactly. Also syncs `FactionCenter` Transform.
- **Player-locked packed migration (`PackedMigrationAutonomy::{Hold, Forage}`):** player-driven nomads (`nomad_autopilot == false`) default to `Hold` and reset on every `PackCamp`. While Packed in `Hold`, `mobile_state_goal_gate_system` strict-demotes everything except `FollowingPlayerCommand`, `PlayerManual` `ScoutAssignment`, and sim-owned `MigrateToCamp`. AI nomads ignore the field.

- **Inspector polish:** `ui/debug_panel.rs` faction section shows `Lifestyle:` + Camp/Home tile label + `Last migration: tick N`. Hover surfaces "Packs as: {good}" or "Teardown: drops {qty} {resource} ({pct}% refund)". Inspector shows `Migrating to (x, y)` when `pending_migration` is set.

## Pluralist Economy

Per-resource policy flags + per-settlement markets + sub-faction households + Bureaucrat/Trader professions + P2P currency + escrow + U_bid scoring + tribute + craft contracts. Currency invariant: `EconomicAgent.currency` + `FactionData.treasury` + `Settlement.treasury` + `JobEscrow.amount` is conserved across every operation.

### Settlements (`settlement.rs`)

The **economic** unit (market + treasury + market_tile), distinct from `SettlementPlan` (the layout of zones around a hearth). One faction can own many settlements; a megachunk can host many competing settlements.

- `Settlement` (Component): `id`, `owner_faction`, `market_tile`, `founding_tick`, `name`, `treasury: f32`, `market: SettlementMarket`, `peak_population`.
- `SettlementId(u32)` newtype; `SettlementMap` (Resource) with `by_id` / `by_megachunk` / `by_faction` indices.
- `auto_found_default_settlements_system` (FixedUpdate, before `settlement_planner_system`): spawns one Settlement per non-SOLO faction at its `home_tile` if missing.

### Currency + escrow

- **`pay(world, from, to, amount) -> bool`** (`economy/transactions.rs`): atomic agent-to-agent transfer. Only sanctioned way to move currency between agents.
- **`FactionData.treasury: f32`** — faction-level wealth pool, distinct from per-settlement treasuries.
- **`JobEscrow { amount, beneficiary }`** sidecar entity per funded posting. Producer debits wallet + spawns sidecar. `job_payout_system` (Economy, exclusive `&mut World`, after `job_claim_release_system`) resolves via `JobEscrowIndex(AHashMap<JobId, Entity>)`; on completion splits `escrow.amount` across claimants (direct credit to `EconomicAgent.currency` — no `pay()` since funds already held), zeros, despawns. On `completed = false`, despawns with amount intact and the `on_job_escrow_remove` hook refunds `beneficiary` — so `aq.cancel()` sites stay untouched. `JobCompletedEvent { claimants, completed, target_rid }`. Paid workers get an `Earnings(VecDeque<EarningEntry { job_kind, target_rid, amount, tick }>)` ring (cap 16) + `ActivityEntryKind::WagePaid`.
- **Chief-funded postings:** `chief_post_funding_system` (Economy, exclusive `&mut World`, after `chief_job_posting_system`) scans `JobBoard` for chief-source `reward == 0.0` postings. For Mixed/Market factions (non-empty `economic_policy`), `chief_wage_for(progress)` computes: Stockpile/Haul/Craft → `trade_base_value(rid) * qty * CHIEF_MARGIN (0.5)` (Haul ×0.5 extra); Calories/Build/Planting → flat per-day formulas bounded by `CHIEF_BUILD_WAGE_CAP=30`. Debits `faction.treasury`, spawns `JobEscrow { amount, beneficiary: chief_entity }`, registers in `JobEscrowIndex`. Unaffordable wages leave `reward = 0` (free communal contract). Subsistence factions skip funding entirely.
- **Wage signal:** every `FactionData` carries `wage_signal: AHashMap<(JobKind, Option<ResourceId>), WageEMA { ema_per_day, last_update_tick, samples }>`. `faction_wage_signal_system` (Economy, daily at `tick % TICKS_PER_DAY == 0`, after `job_payout_system`) walks each member's `Earnings` ring, sums last-day payouts per `(village, kind, rid)` triple via `FactionRegistry::root_faction`, folds into the village's EMA at `α = 1 − 0.5^(1/5) ≈ 0.129` (5-day half-life). Pre-existing keys with no fresh sample decay toward zero. Cross-faction perception lives on `PerceivedFactionWages { by_key: AHashMap<(fid, kind, rid), (ema, observed_tick)> }` (cap 32, oldest-first). `wage_gossip_system` (Economy, after `awareness_gossip_system`) piggybacks `AgentGoal::Socialize` within `WAGE_GOSSIP_RADIUS = 3`: snapshots up to `WAGE_GOSSIP_TOP_K = 4` recent observations (own `wage_signal` + cached perceptions), neighbours merge with `exp(-age/TICKS_PER_DAY)` staleness. Same-faction comparisons always read `wage_signal` directly.
- **Hook registration:** `JobsPlugin::build` calls `register_component_hooks::<JobEscrow>().on_remove(on_job_escrow_remove)`.

### Per-resource policy (`economy/policy.rs`)

`ResourceControlPolicy` flags: `chief_allocates_labor`, `private_actors_allowed`, `state_sells_at_market`, `prices_fixed_by_state`, `fixed_price`. `Default` = all-communist; `capitalist()` preset.

`FactionData.economic_policy: AHashMap<ResourceId, ResourceControlPolicy>` — empty by default. `Method::policy_gate` returns `&'static [(ResourceId, RequiredFlag)]`; `method_passes_policy_gate(method, faction)` returns true iff every gate entry is satisfied. SOLO agents reject any non-empty gate.

### Sub-factions / households

A household is a `FactionData` with `parent_faction = Some(village_id)`. Reuses the entire faction primitive — storage, treasury, chief-equivalent (`household_head`), member tracking.

- **`FactionData.parent_faction: Option<u32>`** — `None` = top-level village, `Some(id)` = sub-faction.
- **`FactionData.household_head: Option<Entity>`** — analogue to `chief_entity`.
- **`FactionData.children_factions: Vec<u32>`** — reverse pointer.
- **`FactionRegistry::spawn_household(parent, home_tile, head, &catalog)`** — creates the sub-faction, wires parent/child links, sets `household_head`. Policy inheritance: non-empty parent `economic_policy` → household stamped `ResourceControlPolicy::capitalist()` on every catalog resource; empty parent map → empty household map. Lets households exist structurally under communist tribes without firing private contracts.
- **`FactionRegistry::root_faction(id)`** — walks `parent_faction` to the village.
- **Formation:** `CoSleepTracker.bond_strength: u16` accumulates ticks of cosleep with the *same* partner. At `HOUSEHOLD_BOND_THRESHOLD` (one game-week) `household_formation_system` (Economy) spawns the household. Both members get `HouseholdMember { household_id }`.
- **Spawn-time seeding (Market preset only):** `spawn_population` (`person.rs`) calls `seed_market_households` after the per-faction member loop. Every adult founds a one-person household with its own plot tile, a dedicated `FactionStorageTile`, and `HOUSEHOLD_SEED_TREASURY (15.0)`. Subsistence/Mixed presets skip this.
- **Inheritance:** `pregnancy_system` threads the mother's `Option<&HouseholdMember>` to the newborn.
- **`HouseholdMember` is a marker only — not a faction migration.** Parents keep `FactionMember.faction_id` pointing at the village.

### Bureaucrat profession

Officials physically employed by the settlement treasury — demote when treasury can't fund them.

- **`Profession::Bureaucrat`** + `FactionData.state_funds_public_works: bool` (default false) governance flag.
- **`FactionData.bureaucrat_treasury_empty_streak: u32`** — running tally; **decoupled from bureaucrat count** so a demote-to-zero can't immediately re-promote until funds arrive.
- **`chief_bureaucrat_appointment_system`** (Economy, every `BUREAUCRAT_ASSIGNMENT_CADENCE = TICKS_PER_DAY/4`): mirrors `faction_hunter_assignment_system`. Target = `max(1, member_count * BUREAUCRAT_MIN_RATIO)`; ranks by Social skill. When `bureaucrat_treasury_empty_streak >= BUREAUCRAT_QUIT_DAYS * TICKS_PER_DAY`, target=0 → all demote.
- **`bureaucrat_salary_tick_system`** (Economy, every `TICKS_PER_DAY/24`): debits the faction's first settlement treasury, credits each bureaucrat by `BUREAUCRAT_DAILY_WAGE/24`. Treasury bottoms at 0.
- **`bureaucrat_admin_dispatch_system`** (`htn.rs`, ParallelB after combat dispatcher): for any Bureaucrat with `task_id == UNEMPLOYED + Idle + non-Dormant`, dispatches `Task::Lead { dest = settlement.market_tile }` via `assign_task_with_routing`. Direct dispatch (no HTN method).

### Chief postings gated on policy

`JobPosting` carries `poster_class: PosterClass` (`Chief / Bureaucrat / HouseholdHead / Individual`), `reward: f32`, and `settlement_id: Option<SettlementId>`.

Per-resource gates in `chief_job_posting_system`:

| Branch | Gate |
|---|---|
| Stockpile Calories (food) | `policy_for(Fruit).chief_allocates_labor` |
| Stockpile Wood/Stone | `policy_for(target_rid).chief_allocates_labor` |
| Stockpile (CraftOrder demand) | per `target_rid` |
| Haul (per-blueprint deposit) | `policy_for(slot.resource_id).chief_allocates_labor` |
| Build | `!faction.state_funds_public_works` |
| Craft | `policy_for(recipe.output_resource).chief_allocates_labor` |
| Farm | `policy_for(Grain).chief_allocates_labor` |

Default factions have an empty policy map → all chief postings fire. Capitalist factions opt out selectively.

**Workforce-budget policy gate:** `compute_workforce_budget` (`projects.rs`) consults `chief_will_post_for_slot(faction, slot)` per slot. Slots whose corresponding chief posting is policy-disabled collapse to 0 share, and their proportional cut of `usable = 1 − FREE_FLOOR` rerouted to `free`.

**Household income skim:** `split_market_earnings_with_household(world, agent, earned)` (`economy/transactions.rs`) redirects `caps.income.household_skim_pct` of market earnings to the household treasury when the agent has `HouseholdMember`.

**Household-poster path:** `household_contract_posting_system` (Economy, every `HOUSEHOLD_POSTING_CADENCE = TICKS_PER_DAY`) walks every household with `treasury >= HOUSEHOLD_MIN_TREASURY_FOR_POSTING (10.0)` and posts one paid craft contract via `post_craft_contract_from_treasury`. Recipe picked by `pick_household_recipe(MaslowTier::next_unmet(head_needs), village_techs)`: Belonging-tier head + LOOM_WEAVING-aware village → recipe 4 (Cloth); else → recipe 0 (Tools). `poster_class=HouseholdHead`, `reward = HOUSEHOLD_CONTRACT_REWARD (5.0)`, lands on the **village's** job board.

### Per-settlement markets

Production trade routes through the agent's faction's first settlement market when one exists; SOLO/unsettled fall back to the global `Market`.

- **`SettlementMarket`** (`economy/market.rs`): `calculate_price`, `sell_item`, `try_buy_item`, `clear_flow`, `set_stock` (trader fast path).
- **`market_sell_system` / `market_buy_system`**: take `Res<SettlementMap>` + `Query<&mut Settlement>` + `&FactionMember`. Disjoint borrows.
- **`settlement_price_update_system`** (Economy, alongside global `price_update_system`): walks every Settlement, ticks `update_prices`, clears bid counters. Bid-driven discovery — see `economy/CLAUDE.md` for the price formula. No synthetic demand injection.
- **`ui/economy_panel.rs`** prefers the player faction's first settlement market.

### Maslow needs + esteem/self-actualization triggers

Two additive needs on `Needs`: `esteem: f32` (Tier 4) and `self_actualization: f32` (Tier 5). Both inverted polarity (0 = unfulfilled).

`MaslowTier::next_unmet(needs)` returns the lowest-numbered tier below threshold. Strictly *additive* — does not replace `goal_update_system`'s goal selection.

- **`esteem_driven_posting_system`** (Economy, every `ESTEEM_POSTING_CADENCE = TICKS_PER_DAY`): walks agents whose `next_unmet == Esteem` AND `currency >= ESTEEM_POSTING_MIN_CURRENCY (50.0)`; posts a Torch contract via `post_craft_contract` with `reward = ESTEEM_CONTRACT_REWARD (8.0)`, bumps `esteem` by `ESTEEM_POSTING_GAIN (30.0)`. Hungry/unsafe/lonely agents never reach the gate.
- **`self_actualization_teaching_system`** (`teaching.rs`, Economy, every `SELF_ACTUALIZATION_CADENCE = TICKS_PER_DAY`): walks agents whose `next_unmet == SelfActualization` with at least one Learned tech; picks the highest-complexity Learned tech and writes `LectureRequest`. Bumps `self_actualization` by `SELF_ACTUALIZATION_LECTURE_GAIN (30.0)`.

### U_bid scoring at job-claim layer

`job_claim_system` (`jobs.rs`) branches on `posting.reward`:

- **Paid (`reward > 0.0`):** `U_bid = E(R) + priority_bonus + affinity_bonus - C_action - C_opportunity` where `E(R) = posting.reward * wealth_modifier(agent.currency) * disposition.earn_income_multiplier()`; `disposition multiplier = 1.0 + entrepreneurial/255` (falls back to 1.5 median when absent); `wealth_modifier(c) = 1.0 + 0.5 / (c + 50)` bounded at 1.0; `priority_bonus = posting.priority * 0.01` (chief 200, household 180, individual 100 — the +2.0 cushion prevents `CHIEF_MARGIN = 0.5` chief postings from being outcompeted by full-value private contracts at equal reward); `C_action = euclidean(agent_tile, work_tile) * BID_DIST_DISCOUNT`; `C_opportunity = 0.0` (stub).
- **Unpaid (`reward == 0.0`, chief / legacy):** legacy `priority + skill + bias - distance`.

### Trader profession (autonomous arbitrage)

`Profession::Trader`. `trader_buy_at_settlement` / `trader_sell_at_settlement` are atomic primitives in `economy/transactions.rs`. `SettlementMarket::set_stock(id, qty)` is the fast-path setter so the helpers don't double-mutate currency.

- **`TraderPlan { phase, buy_settlement, sell_settlement, resource_id, qty }`** component. `TraderPhase::TravelingToBuy | TravelingToSell`.
- **`trader_market_step_system`** (Economy, exclusive `&mut World`, after `self_actualization_teaching_system`, **`run_if(tick % 20 == 0)`**): on arrival at the phase's market tile, calls buy/sell helpers and advances or clears the plan. Seeds new plans on idle traders by scanning `AgentMemory.visited_settlements` pairs for the best Cloth gap exceeding `TRADER_MIN_GAP=0.25`. Currency floor `TRADER_MIN_CAPITAL=30.0`. `pick_arbitrage` snapshots `(price, stock, treasury)` per visited settlement once.
- **`trader_route_dispatch_system`** (ParallelB, after `bureaucrat_admin_dispatch_system`): for plan-bearing traders not at their phase target, dispatches `Task::Lead { dest }`.
- V1 scope: Cloth-only arbitrage + `TRADER_TRADE_QTY=5`.

### Tribute

Treasury-to-treasury transfer between factions — agent currency untouched.

- **`FactionData.dominance_over: Vec<u32>`** + **`subordinate_to: Option<u32>`**.
- **`FactionRegistry::set_dominance(dominant, subordinate)`** — idempotent.
- **`tribute_payment_system`** (Economy, after `bureaucrat_salary_tick_system`, every `TRIBUTE_CADENCE = TICKS_PER_DAY`): walks every faction with `subordinate_to.is_some()`; transfers `min(TRIBUTE_PER_DAY, subordinate.treasury)` to overlord. Destitute subordinates pay 0 — no debt.

### P2P craft contracts

- **`post_craft_contract(world, poster, faction_id, recipe, qty, reward, deadline)`** (`jobs.rs`): atomic — debits poster's currency, pushes a `JobPosting` with `poster_class=Individual` + contract terms, spawns a `JobEscrow` sidecar. Refuses on insufficient funds / invalid recipe / qty=0 / non-positive reward.
- Lifecycle: completion = caller zeros `escrow.amount` then despawns. Cancellation/expiry = caller despawns with amount intact; `on_job_escrow_remove` refunds the poster.
- **U_bid integration:** `reward > 0` fires the U_bid branch — wealthy poster's contract outscores an equidistant chief Craft posting.

## Land ownership (`land.rs`)

Plot-based ownership layer over the compatibility `SettlementPlan` projection generated from `SettlementBrain` parcels/districts. Plots start `Tenure::StateOwned` held by the settlement's owning faction; in Mixed/Market presets the listing system publishes them, households acquire what they can afford, the rent-collection system bills monthly with eviction after two misses, and sharecropping plots split harvest yield between tenant and landlord at gather time. Settlement-expansion plot diff (Phase 8) deferred.

- **`Plot`** (Component): `id, settlement_id, faction_id, rect, z, zone_kind, tenure, holder, base_value, last_valued_tick, missed_payments, frontage_edge, access_tile, parent_plot`. `Tenure` ∈ `{StateOwned, Leased, Sharecropping, Freehold}`. `TenureHolder` ∈ `{State, Household}`.
- **`PlotIndex`** (Resource): `by_id`, `by_settlement: u32 → Vec<PlotId>`, `by_tile: (i32,i32) → PlotId` (surface-only), `by_faction_hash: u32 → u64` for idempotency, `next_id`. `plot_at(x, y)` is the hot lookup.
- **`LandListings`** (Resource): `for_sale` / `for_lease` Vec stubs.
- **`carve_plots_system`** (FixedUpdate, Economy, after `settlement_planner_system`, before `chief_directive_system`): per faction, if `plan.culture_hash != by_faction_hash[fid]` — tear down existing plots and re-carve, valuing each new plot via `compute_plot_value`. Multi-settlement factions currently bind every plot to `SettlementMap::first_for_faction(fid)`. Plot sizes: Residential 6×6, Crafting/Storage 4×4, Agricultural 10×10; Civic/Sacred/Market/Defense kept whole-zone. `subdivide()` folds trailing slivers (`< stride/2`) into the previous slice.
- **`compute_plot_value`** (`PLOT_BASE_VALUE = 50.0`): `BASE * zone_mul * (centre_factor + home_factor) * terrain_factor`. Distance factors anchor on `Settlement.market_tile` + `FactionData.home_tile`; zone multipliers 0.6–1.5; terrain samples fertility at centre + four corners (Agricultural reweights harder). Falls back to neutral 0.5 terrain factor when the plot lies in unloaded chunks.
- **`LandPolicy`** (`../economy/policy.rs`): faction-level governance flags. Set by `land_policy_for(EconomyPreset)` in `spawn_population` and inherited by households via `spawn_household`. Subsistence = all-false; Mixed = rent + sharecrop; Market = adds sale + freehold. `default_lease_period_days = 30`, `rent_yield_pct = 0.04` (≈ 50 %/year), `default_share_to_landlord = 0.30` for Mixed/Market.
- **`tile_buildable_by`**: rejects placements when the tile sits on a plot held by another faction or another household. Wired into `construction.rs::chief_directive_system` after candidate selection — civic blueprints pass `requesting_household=None`. Wild tiles always pass. Pure-logic core in `holder_permits_build` for testing.
- **`land_listing_system`** (Economy, every `TICKS_PER_DAY/4`): publishes Sale + Lease listings for `StateOwned` plots when the faction's `LandPolicy` permits. Civic-class zones (Civic / Sacred / Defense / Market) are skipped. Cap `TARGET_LISTINGS_PER_FACTION = 8`. Sale asking = `base_value`; lease asking = `base_value × rent_yield_pct` per month, floored at `MIN_MONTHLY_RENT`. Skips `parent_plot.is_some()` plots.
- **`household_land_acquisition_system`** (Economy, every `TICKS_PER_DAY`): plot-less household with `treasury ≥ HOUSEHOLD_MIN_TREASURY_FOR_LEASE` browses parent-village listings. Affordability: lease ≤ 40%, buy ≤ 70% of treasury. Preference order: cheapest Sale → Lease → Sharecrop. Direct faction-to-faction treasury transfer. Plot tenure flips, holder becomes `Household { faction_id: household_id }`. Also claims nearest unowned same-village Agricultural plot within 12 chebyshev as a child (mirrored tenure, parent-funded rent).
- **`rent_collection_system`** (Economy, every 30 game-days): each Leased plot with expired `paid_through_tick` attempts household→landlord transfer. Success advances cycle, resets `missed_payments`; failure bumps `missed_payments` (cycle still advances). At `missed_payments >= EVICTION_MISS_THRESHOLD (2)`: tenure → `StateOwned`, emits `PlotEvictedEvent { plot_entity, plot_id, plot_rect, plot_z, landlord_faction, policy }` with the landlord's `caps.land.eviction_policy`. Skips `parent_plot.is_some()` plots.
- **`evicted_plot_cleanup_system`** (Economy, after `rent_collection_system`): drains queue per policy. `LeaveStructures` (default) / `RevertToState` no-op; `Demolish` walks `StructureIndex.0` over `plot_rect`, refunds via `Deployable::compute_refund_drop` as `GroundItem`, `despawn_recursive`s. Non-`Deployable` buildings despawn without drops.
- **Sharecropping**: `state_sharecrops` flag adds `ListingKind::Sharecrop` for `Agricultural` plots. Zero upfront; tenure = `Sharecropping { share_to_landlord, paid_through_tick }`. `gather.rs` harvest hook (`lookup_sharecrop_split`) routes landlord's share via `split_sharecrop_yield(qty, share_to_landlord)` (floor in tenant favour) as a `GroundItem` at the landlord's nearest `FactionStorageTile`; tenant remainder flows through `route_yield`. `SharecropResources` SystemParam keeps `gather_system` under the 16-param ceiling.
- Inspector hover (`../ui/hover.rs`) surfaces plot info — id, zone, rect, tenure, holder, value, parent-plot link.

## Game lifecycle and regions

- **`GameState`:** `SpawnSelect` (default) | `Playing`. Globe gen at `WorldPlugin::build`. Spawn-select UI in `SpawnSelect`; spawn systems on `OnEnter(Playing)`. Update-stage systems gated `in_state(Playing)`; FixedUpdate sim systems are not gated.
- **`PendingSpawn(Option<(i32, i32)>)`:** player's chosen mega-chunk; `None` = globe centre (sandbox).
- **Sandbox bypass:** `SandboxPlugin` flips state to `Playing` at Startup.
- **`SettledRegions` (`region.rs`):** `AHashMap<RegionId, SettledRegion>` + `by_megachunk` reverse index. `SettledRegion { megachunk, founding_tick, name, camera_bookmark, player_owned }`. `settle()` idempotent.
- **`MegaChunkCoord`:** `from_chunk/from_tile/center_tile/chunk_range`. `MEGACHUNK_SIZE_CHUNKS=16`, independent of climate cells.
- **Multi-focus chunk streaming (`SimulationFocus`):** `Vec<FocusPoint>` rebuilt each tick from camera + every settled region centre. Chunk DATA loads for any focus disc (camera `LOAD_RADIUS=12`, region `REGION_LOAD_RADIUS=6`); SPRITES + plants + loose rocks only inside camera focus.
- **Focus-aware LOD (`update_lod_levels_system`):** camera distance produces base LOD; entities within 8 chunks of any non-camera focus are promoted from Dormant to Aggregate.
- **Edge-walk expansion (`region::detect_edge_crossing_system`, Economy):** each tick checks every player-faction Person's mega-chunk; if unsettled, calls `settle()`, names "Outpost N", bookmarks position, emits `ActivityLogEvent::RegionSettled`.
