# Simulation (`src/simulation/`)

Agent AI, factions, knowledge, hunting, typed-task pipeline, pluralist economy. See root `CLAUDE.md` for cross-cutting `SimulationSet` ordering and tile/Z conventions.

## Game speed (`speed.rs`)

`GameSpeed { current, last_unpaused }` is player-facing; `SpeedPreset::{Paused, Normal, Fast (2×), VeryFast (5×)}`. `sync_game_speed_to_virtual_time` (PreUpdate, change-detected) mirrors onto `Time<Virtual>`. Higher presets fire FixedUpdate more often per real second — per-tick decay, cooldown, calendar, and `tick % CADENCE == 0` Economy systems scale naturally. `SimClock.scale_factor()` carries bucket compensation only. `handle_speed_keybinds_system` maps `Space` → pause toggle, `1/2/3` → presets (egui-focus-gated). Per-preset CPU budgets via `SpeedPreset::budget_ms_per_tick()`.

## Background work budgets (`perf.rs`, `world_sim.rs`)

`PerfWorkBudget` centralises main-thread apply caps: chunk data loads 24/tick, camera sprite loads 8/tick, chunk unloads 32/tick, graph classify 16/task, tile refreshes 512/tick, faction plan applies 4/tick, settlement plan applies 2/tick. `BackgroundWorkDiagnostics` is the shared debug surface.

`world_sim_system` does not scan the full `512 × 256` globe on `tick % 60 == 0`. `WorldSimCursor` walks ~1/60 of the globe per FixedUpdate, snapshots unloaded claimed cells + the loaded-cell mask, computes food/population/raid intents on `AsyncComputeTaskPool`. `WorldSimTaskState` queues returned deltas + raids; main thread applies a bounded batch, validating faction id before writing live `Globe` cells. Stale task results drop if `WorldSimCursor.generation` has advanced.

## Agent AI (Goals → HTN → Tasks)

- **Goals (`goals.rs`):** `Survive`, `GatherWood`, `GatherStone`, `GatherFood`, `Haul`, `Defend`, `Farm`, `Build`, `Socialize`, `Sleep`, `MigrateToCamp`, …
- **HTN registry (`htn.rs`):** per-goal decomposition. Methods carry `precondition` / `utility` / `expand(ctx) -> Vec<Task>`; dispatcher argmaxes and routes head, prefetching tail on `ActionQueue`.
- **Tasks (`tasks.rs`):** current action (`TaskKind` enum). `Idle` between tasks.
- **`Disposition { entrepreneurial, gregariousness, curiosity, martial }`** (`u8`, scattered at spawn) drives per-agent preferences.
- **`GoalClass`** (`Survival > Subsistence > Safety > Belonging > Esteem > Enterprise > Discretionary`, `Ord`-derived) tiers goals; registry argmax picks `Survival → Sleep` over `Enterprise → Craft` regardless of raw score.
- **`GoalScorer` trait + `GoalScorerRegistry` resource**: pluggable scorers (`SurvivalHunger`, `Sleep`, `ReturnSurplus`, `Social`, `Play`, `TameAnimal`, `PersonalBuild`, `CraftDemand`, `Stockpile`, `HealNeed`, `ProvideCare`, `EarnIncome`, …). `register_default_scorers` at `SimulationPlugin::build`. Continuous utility curves (`utility_curves.rs`) anchored on legacy `HUNGER_*` / `SLEEP_*` cliffs. Per-scorer disposition lifts in one tunables block atop `goal_scorers.rs`. **Invariant**: a survival-maintenance scorer must not emit a goal below its dispatcher's executable gate, or it pins the agent on an indecomposable work-preempting goal. `SurvivalHungerScorer` / `fallback_pick` gate personal `Survive` on `EAT_TRIGGER_HUNGER` (180) — same threshold every food HTN precondition uses. `ThirstScorer` likewise gates on `THIRST_TRIGGER`. Sub-180 hunger prep is faction/economy scope (`GatherFood` / `Stockpile`), not personal `Survive`. `HUNGER_*` constants in `goals.rs` derive from `HUNGER_ACT = EAT_TRIGGER_HUNGER as f32` so scoring and HTN gates cannot drift.
- **`goal_update_system` is the sole goal-selection path.** Builds `GoalScoringContext` from precomputed gates + `Calendar::time_phase()` and calls `registry.best_with_incumbent(...)`. Hysteresis: `GOAL_CHALLENGER_MARGIN = 0.10`. `interrupt_policy_allows` gates replacement. Legacy imperative cascade survives as `fallback_pick` for SOLO / fixtures.
- **Goal/dispatcher contract (`goal_contract.rs`).** A scorer may only emit a goal whose HTN dispatcher can decompose it now. Broad gates that select an indecomposable goal cause permanent idle loops. `goal_contract.rs` centralises scan-radius constants (`TAME_SEARCH_RADIUS`, `SOCIAL_PARTNER_RADIUS`, `PLAY_PARTNER_RADIUS`, `PLAY_ITEM_RADIUS`, re-exported `HEAL_SCAN_RADIUS` / `DRINK_*`) so scorer gate and dispatcher scan don't drift. `has_tameable_animal` is candidate-within-`TAME_SEARCH_RADIUS`, not global. Per-scorer executable gates in `GoalScoringContext`: `has_local_care_patient` (radius-filtered `CareNeed` + injured-agent positions within `HEAL_SCAN_RADIUS`); `has_social_partner` (box sweep over a fresh per-tile `Person`-count grid built same-tick from live `Transform`s — `SpatialIndex` cold would flip a freshly-spawned `Socialize` agent off goal on tick 1). `ProvideCareScorer` / `SocialScorer` decline when gate false. `Play` has **no** scorer gate (cheap gate false-negative-prone for solo fallbacks); instead every `htn_play_dispatch_system` no-task exit records `goal_contract::blocked`. `Craft` gated by `should_craft` (recipe-input availability); `Build` deliberately ungated.
- **No-task backstop (`goal_contract.rs`).** When a dispatcher selects a goal but produces no task, records a throttled synthetic `MethodHistory` failure via `record_no_task_backstop` (≥`BACKSTOP_THROTTLE_TICKS=20` apart). After 3 entries within history TTL, `chronic_failure_release_system` cooldowns the goal + forces re-evaluation. `MethodId::UNKNOWN` entries filter out of `recently_failed_count` but DO count toward `chronic_failure_release`'s non-success tally. Single-dispatcher goals (`TameAnimal`/`ProvideCare`/`Socialize`/`Play`) call `goal_contract::blocked(..)` inline at every dispatcher no-task `continue`. Multi-dispatcher goals (`Craft`/`Build`/`Farm`) covered by `goal_contract_backstop_system` (head of `Sequential`, after every ParallelB dispatcher).
- **Cooperative maintenance for durable work:** `JobClaim` and `ScoutAssignment` keep active episodes intact; when idle, `goal_update_system` runs a maintenance-only scorer pass (`Survive` / `Drink` / `Sleep` / `SeekCare`). Maintenance preserves claim/assignment; when it stops scoring, claimed job or scout goal is restored. `Socialize` / `Play` use the separate opportunistic cancel-and-replan path.
- **`ScorerInputs` SystemParam** bundles registry, `JobBoard`, `OpportunityIndex`, `DecisionMetrics`, per-agent queries to stay under Bevy's 16-param ceiling.
- **Opportunity index (`opportunity.rs`)** rebuilt every 20 ticks in Economy after `compute_faction_storage_system`. Producers emit `PaidJob` / `MaterialDeficit` / `FoodSource` / `CareNeed`. `EarnIncomeScorer` and `ProvideCareScorer` read it first with direct fallback.
- **Opportunistic en-route interrupts (`opportunistic.rs`):** scorers opt in via `GoalScorer::opportunistic()`. `opportunistic_interrupt_system` (ParallelA, 20-tick cadence) flips the goal when an eligible scorer scores above `OPPORTUNISTIC_INTERRUPT_THRESHOLD (0.50)` and the prior goal isn't on `GoalCooldown`. Skips `MF_UNINTERRUPTIBLE` chains and `JobClaim`-locked agents.
- **HTN disposition lift:** `Method::disposition_lift(...)` multiplies utility (cap 1.3 below tier breakpoints). Overridden by Socialize / Hunt / Play methods. `dispatch_for_goal(...)` is the shared argmax helper.
- **Goal-update cadence:** active agents re-evaluate every 200 ticks; idle bypass. Target validation is narrow — only despawned `target_entity` invalidates. Stage/kind drift caught at arrival.

### Healing pipeline (`medicine.rs`)

- **`Injury { severity, applied_tick, last_damage_tick }`** derived reactively from `Health.current < Health.max` by `injury_tracking_system` (observer on `Changed<Health>`); clears at full restore.
- **Goals:** `AgentGoal::{SeekCare, ProvideCare}`. `HealNeedScorer` is **Survival**; `ProvideCareScorer` (Subsistence 0.65) fires for Healer/Apprentice when any same-faction injured exists.
- **Provider chain:** `htn_provide_care_dispatch_system` (ParallelB) routes nearest patient within `HEAL_SCAN_RADIUS = 12` → `Task::Heal { patient }`. `heal_task_system` (Sequential, after combat) requires chebyshev ≤ 1, ticks `Injury.severity -= 1`, grants Medicine XP, removes `Injury` at zero.
- **Patient chain:** `htn_seek_care_dispatch_system` routes to nearest faction-owned `Shrine` via `WorkshopOwnership`; short-circuits within `SEEK_CARE_AT_SITE_RADIUS = 6`. A patient already inside is parked (legitimate no-task state — maintenance goal, exempt from `chronic_failure_release`, not covered by backstop).
- `htn_provide_care_dispatch_system` no-patient `continue` carries `goal_contract::blocked` (`BlockedReason::NoCarePatient`).
- **Chief auto-promotion:** `chief_healer_assignment_system` (Economy, daily quarter) mirrors `chief_craft_assignment_system`; injury-driven (`HEALER_PER_INJURY_DIVISOR=4`, capped by `HEALER_MAX_DIVISOR=3`). Sub-threshold Medicine routes through `ApprenticeProgress { target_profession: Healer }`.

### Thirst pipeline (`drink.rs` + `sanitation.rs` + `medicine.rs`)

`Needs.thirst` decays at ~2× hunger rate, gated by `THIRST_TRIGGER` / `THIRST_SEVERE`. Never damages `Health`.

- **`ThirstScorer`** (Survival) fires `AgentGoal::Drink`; `htn_drink_dispatch_system` tries inventory `clean_water`, then ranked routed candidates: nearest local water-bearing `WellMap`, nearest local fresh tile, nearest home/camp wet well, nearest home/camp fresh tile. Home/camp fallback scans `DRINK_HOME_SCAN_RADIUS=32` around `FactionData.home_tile` only for settled or nomads with `CampState::Pitched`. `nearest_well_tile` skips dry via `drink::well_has_water`; `nearest_fresh_drinkable_tile` skips Salt/Brackish via `world::biome::water_kind_at(...).is_drinkable()`; severe widens scans. **Salinity:** `WaterKind::{Fresh,Brackish,Salt}` from `Globe::salinity_at`; `is_drinkable()` = Fresh-only. **Physical wells (`well.rs`):** `WELL_REACH_Z`/`well_reaches` gone — a well's water is the live `RuntimeWater` column at its central shaft. `well_water_column(runtime_water, shaft)` reads depth; `well_has_water` = column ≥ `well::WELL_MIN_DRINKABLE_DEPTH (0.3)`. `perform_drink` draws column down `WELL_SIP_DRAWDOWN_Z (0.02)` per sip (cell never removed — its `AQUIFER_SEEP_RATE` source recharges). Column below drinkable floor reads `DrinkOutcome::WellDry` and refills.
- **`drink_task_system`** consumes from hands or sips adjacent fresh tile / well; raw drinks bump `Sickness.severity` 60, contaminated tiles 140 (well tiles read `SanitationMap::is_contaminated` like river tiles, so wells without nearby `Latrine` get tainted by `WastePile`). Multi-sip loop: drinks until `thirst ≤ DRINK_SATIETY_FLOOR (40)`, source exhausted, or `MAX_SIPS_PER_ACTION (4)`. `ThirstScorer` uses `utility_curves::thirst_utility` (two-stage smoothstep, weights 0.60/0.40) so urgency at `THIRST_TRIGGER` ≈ 0.60.
- **Physically-excavated wells (`well.rs`)** — `BuildSiteKind::Well` (right-click → Build Well, `WELL_DIGGING` Neolithic). Staged structure. `convert_well_blueprint_system` (Sequential, before `construction_system`) turns a placed Well blueprint into `WellSite`: resolves depth from aquifer via `well_spec_at` (`aquifer_z_at` shared frame; shaft dug to one Z below the table, capped `MAX_HAND_DUG_WELL_DEPTH_Z=8`). Geometry: 5×5 stepwell — centre water-shaft column, 8-tile descending helix of carved `Dirt` floors (`|Δz|≤1` steps, no `Ramp` needed), 16-tile outer rim of lining `Wall` blueprints (one gateway gap). `well_site_progression_system` drives `WellPhase::{Excavating → Lining → Capping}`: excavation via `TerraformSite`/`terraform_system`; on completion charges `RuntimeWater` (`WELL_INITIAL_CHARGE_Z`) and spawns lining walls; finalize spawns `Well { faction_id, shaft_tile, bottom_z, surf_z }` in `WellMap`. `DrinkSource::Well { tile }` (`tile` = shaft/`WellMap` key). `organic_settlement::SettlementPressureKind::WaterAccess` queues wells when local fresh water far or population > 40 / 90. **Footprint validation:** `convert_well_blueprint_system` rejects (despawns) Well blueprint whose 5×5 `well_footprint` overlaps any `StructureIndex` / `WellMap` / `WellSiteMap` / other `BlueprintMap`. **Persistence:** carved Air/floor tiles are chunk deltas not re-applied on regen; `restamp_wells_on_chunk_load` (FixedUpdate, after `chunk_streaming_system`, before `restamp_runtime_water_on_chunk_load`) re-carves any `Well`/completed-`WellSite` geometry on freshly-loaded chunks via shared `carve_well_geometry`. Seed wells (`spawn_seeded_structure_at_tile`) skip workers — stamp `Well` + charge `RuntimeWater` directly; OnEnter `carve_seeded_wells_system` carves visible shaft + helix (depth stays `surf - 3`).
- **Boiling**: `2 raw_water + 1 wood → 1 clean_water` (CraftRecipe 13, `FIRE_MAKING`, Workbench).
- **`SanitationMap`**: sparse `(tile → f32)` populated daily from `WastePile` via `1/(d²+1)` falloff in `CONTAMINATION_RADIUS = 6`; decays `2^(-1/4)`/day; `is_water_contaminated` above `0.5`. `WastePile` from `agent_defecation_system` (one per `DEFECATION_INTERVAL_TICKS = 2 days`, staggered). `Latrine` within `LATRINE_ROUTING_RADIUS = 8` marks pile `LatrineContained`. `BuildSiteKind::Latrine` (right-click, 2 wood + 1 stone).
- **`Sickness`** (orthogonal to `Injury`): daily decay `-16`; `sickness_work_factor(severity) → [0.5, 1.0]` multiplies movement-driven work progress.
- **Animals:** `AnimalNeeds.thirst`, `AnimalState::Drinking`, `animal_water_seek_system` + `animal_drink_system`.

### Needs

7 needs (hunger, thirst, sleep, shelter, safety, social, reproduction), `[0,255]`, decay over time. Maslow tiers (`Physiological → Safety → Belonging → Esteem → SelfActualization`) via `MaslowTier::next_unmet(needs)`. Decay rates anchored on 180-sec game-day: `HUNGER_RATE = 2.0/s`, `SLEEP_RATE = 1.2/s` paired with `SLEEP_RECOVER_RATE = 6.0/s` (12.0 on bed). `WILLPOWER_WORK_DRAIN = 1.8/s`, `WILLPOWER_IDLE_DRAIN = 0.15/s`. `con_scale` attenuates hunger+sleep decay (floor 0.25×). `tick_needs_system` only accrues sleep need (when not `Sleeping`); `sleep::sleep_task_system` (Sequential) owns sleep recovery + retirement keyed on typed `Task::Sleep` (not `ai.state`).

### Energy (`energy.rs`)

`Energy { current, max, exhausted }` — physiological exertion/fatigue pool, **separate from `Needs`** (must NOT feed `worst()`/`avg_distress()`). 0–255 float; `default()` full. Spawned on every Person. Hysteresis: `ENERGY_EXHAUSTED=40` (flips stateful flag), `ENERGY_TIRED=90`, `ENERGY_RECOVERED=140` (only crossing this clears `exhausted`). `energy_factor()` ramps 1.0→`ENERGY_FACTOR_FLOOR=0.45` below `ENERGY_TIRED`.

- **Drain:** movement per tile (scaled by terrain effort + mounted ×0.15), labor (`energy_tick_system`, ParallelA after `tick_needs_system`, while `AiState::Working`), combat per swing (tired attacker cooldown ×1.3).
- **Recovery:** sleep (`sleep_task_system`, same bed ×2 multiplier), idle (`energy_tick_system`, `AiState::Idle`).
- **Effects:** movement speed and work-progress accrual both ×`energy_factor()` (alongside sickness factor). **Exhaustion goal gate:** `GoalScoringContext.is_exhausted` (set from `energy_q` in `ScorerInputs`); `GoalScorerRegistry::best_with_incumbent` drops every non-`Survival`/`Safety` candidate while it holds. Eat/drink/sleep/heal/flee/defend stay ungated.

### Swimming (`swimming.rs`)

Amphibious pathfinding (see `pathfinding/CLAUDE.md` → Amphibious traversal) lets a human plan across water; `swimming.rs` owns what happens **while** wet. `SwimmingState { wet_ticks, exhausted_ticks, last_xp_tick, last_safe_tile }` attached on water entry, removed on bank arrival by `swimming_system` (Sequential, after `movement_system`). While wet: heavy `Energy` drain (`SWIM_ENERGY_DRAIN`), periodic `SkillKind::Swimming` XP. **Fatigue-first risk:** drowning (`Body` torso damage) begins only once exhausted swimmer has been in deep water (`depth ≥ DEEP_WATER_DEPTH`) past `DROWN_GRACE_TICKS`. `SkillKind::Swimming = 9` innate. **Currents:** `swimming_system` reads `world::water_current::WaterCurrentField` (`Option<Res>` — absent in headless fixture); swimming into a river current scales energy drain up to +50%, resisting current grants XP. Emergency bank-retreat retarget for exhausted swimmers deferred (`last_safe_tile` tracked but not yet consumed).

### Ambient work-social multitasking (`social_contact.rs`)

Workers doing compatible primary work near a same-root-faction coworker gain secondary social interaction without leaving the job. `SecondarySocial { partner: Option<Entity>, mode: Ambient, expires_tick }` spawned on every Person (`inactive()` default), only mutated — Person archetype never churns. `ambient_social_pairing_system` (ParallelA, after `tick_needs_system`, before `goal_update_system`/`opportunistic_interrupt_system`): per compatible agent, on per-entity-staggered `PAIRING_RESCAN_CADENCE` picks deterministic nearest (`(chebyshev, entity-bits)` min) compatible coworker within `SOCIAL_RADIUS` and stamps `expires_tick = now + PAIRING_WINDOW (400)`; off-cadence clears invalid pairings. `is_ambient_social_compatible` rejects SOLO/Dormant/Drafted/CombatTarget/Raid/Rescue/Migrate/Pack/maintenance/dedicated-Socialize; accepts durable-work goal set (Gather*/Build/Craft/Farm/Haul/Stockpile), `AiState` not gated.

Shared gate **`is_social_contact`** (dedicated `Socialize` OR live `SecondarySocial`, non-Dormant) is the "any social channel" predicate.

| Consumer | Ambient behaviour |
|---|---|
| `faction::social_fill_system` (social relief) | yes — same-root-faction `Person` neighbors only |
| `knowledge::awareness_gossip_system` | yes — full strength |
| `knowledge::cluster_tier_promotion_system` | yes — full strength |
| `jobs::wage_gossip_system` | yes — full strength |
| `memory::conversation_memory_system` | yes — two-tier: dedicated pair +5/tick uncapped; purely-ambient +1/tick capped at `AMBIENT_AFFINITY_CAP=40` |
| `knowledge::tech_teaching_system` | **no — deliberate-only** |

`AMBIENT_AFFINITY_CAP < PARTNER_AFFINITY_THRESHOLD (60)` keeps ambient bonds at "acquaintance"; cohabitation requires deliberate `Socialize`. `opportunistic_interrupt_system` skips `Socialize` challenger while `SecondarySocial` live; `social_fill` ambient drain keeps `needs.social` low so `SocialScorer` doesn't fire.

## Method-design rules

- **Scoring = viability + distance + time-of-day/fatigue for AcquireFood / AcquireGood{wood,stone}.** Base tiers: `UTIL_BASELINE=1.0` / `UTIL_VISIBLE_GROUND=1.5` / `UTIL_CLAIMED_HAUL=2.0` / `UTIL_EXPLORE_FALLBACK=0.3`. Chebyshev distance penalty capped at `MAX_DIST_PENALTY=0.30`. Recent failures bias via `MethodHistory` + `score_method_with_history` (`METHOD_FAILURE_PENALTY=0.4`, ring len 6, TTL 600 ticks). Two stacked failures push baseline below Explore. **Context-aware penalty:** `ScoringScope::ContextAware { time_phase, dusk_remaining, fatigue }` multiplies geometric value by time-of-day (Day 1.0 / Dawn 1.10 / Dusk ramp / Night 4.0) and fatigue (`1.0 + fatigue`). Cap rises to `MAX_DIST_PENALTY_NIGHT=1.50` only at Night.
- **AcquireFood concrete-first partition (`plans/food-pickup.md`).** `htn_acquire_food_dispatch_system` argmaxes over concrete food methods first (Withdraw / Scavenge / Forage / Fish), only falls through to `ExploreForFoodMethod` when no concrete passes. Failure-bias still reorders among concretes. All four sit at `UTIL_BASELINE = 1.0 − dist_penalty` — Scavenge's legacy `UTIL_VISIBLE_GROUND` premium dropped once preconditions read live world state. Tier baselines encode intent, not certainty.
- **In-hand fast-path for haul dispatchers.** `htn_acquire_good_dispatch_system::Haul` and `htn_build_claimed_blueprint_dispatch_system` Path B check `Carrier + EconomicAgent` inventory before `WithdrawMaterial`; if already carrying, dispatch `HaulToBlueprint` directly. Never count agent inventories in posting creation or chief blueprint-candidate scoring.
- **Farming is `Farm`-goal only.** Both halves HTN-driven (`PlantFromStorage`, `HarvestPlant`). Seed↔plant mapping centralised in `PlantKind::seed_resource()` + `PlantKind::ALL`.
- **`MF_UNINTERRUPTIBLE`** marks methods whose multi-leg chain must survive across goal-dispatch ticks.

## Player command authority (`player_command.rs`)

UI emits `PlayerCommandEvent`. Sim drains in `SimulationSet::Input` (exclusive), attaches `Commanded { command, status, issued_tick, command_id }`, then `dispatch_player_command_system` (ParallelB) routes into typed-task pipeline. **All routing is sim-side** — UI never mutates `PersonAI` / `ActionQueue` / authority markers directly.

- **Variants:** `Move`, `Gather`, `Mine`, `Build`, `Deconstruct`, `DigDown`, `PickUpItem`, `PickUpCorpse`, `AttackEntity`, `Teach`, `HoldLecture`, `ReadItem`, `EncodeTablet`, `Muster`, `Disband`, `MilitaryMove`, `MilitaryAttack`, `PackCamp`, `PitchCamp { tile, z }`. Pack/Pitch are faction-scoped (chief actor); dispatcher writes into `nomad::PendingCampOps`.
- **Goal forcing replaces filter scatter.** `goal_update_system` reads `Commanded` via `GoalValidationQueries.commanded_q`; non-terminal forces `AgentGoal::FollowingPlayerCommand`. HTN dispatchers don't recognize the goal so they naturally skip.
- **Lifecycle (`player_command_lifecycle_system`):** Move/MilitaryMove → chebyshev arrival; Gather → target despawned OR agent idle; Mine → tile no longer Wall/Stone; Build → blueprint despawned; PickUp* → target gone; Attack/MilitaryAttack → foe dead. `reap_terminal_commands_system` strips `Commanded` once terminal; UI reads `RemovedComponents<Commanded>`. **MilitaryMove arrival reads `PersonAI.dest_tile`** (per-actor slot), not click anchor.
- **Multi-unit `MilitaryMove` formations (`military/formation.rs`).** Events with `actors.len() > 1` expand into compact ring of per-actor slot tiles. `expand_military_move_system` runs `plan_compact_ring(anchor, n, is_passable)` + `greedy_assign`; output in `PendingFormationSlots`. `dispatch_player_command_system` routes each actor + inserts `MilitaryFormationSlot { anchor, slot_index, group }`. Anchor seeds `HotspotKind::RallyPoint`. `MilitaryFormationGroupGen` allocates one `u32` group id per dispatch. `MilitaryAttack` untouched.
- **Faction-level commands** (e.g. `EncodeTablet`) carry empty `actors`, apply directly in `drain_player_command_events_system`.
- **Adding new order:** new `PlayerCommand` variant + `dispatch_one` arm + `player_command_lifecycle_system` arm + UI button.

## ActionQueue and typed Task variants (`typed_task.rs`)

`aq.current: Task` is canonical "task running now"; `Task::Idle` default. Behind it sits fixed `queued: [Task; 4]` prefetch ring (private; access via `enqueue` / `pop_next` / `peek_next` / `queued_len` / `clear_queued`).

- **Producers** (HTN dispatchers, `ui/orders.rs`, `teaching.rs::ReadItem`, legacy-only `building_upgrade_system` / `terraform_dispatch_system` / `military_task_system` reroute) route through `aq.dispatch(task)` — enqueues then promotes head into `current` if `current == Idle`. Returns `DispatchOutcome::{Promoted, Queued, Rejected}`. `Rejected` (queue full) fires `debug_assert!` inside `dispatch`.
- **Autonomous direct dispatch** that bypasses a registry `Method` (job-driven single-leg tasks like field preparation) must use `tasks::dispatch_autonomous_task_with_routing`. Wraps `assign_task_with_routing` + `aq.dispatch(...)` and stamps `AutonomousTaskLifecycle { owner_goal, task_kind, job_id, preserve_across_goal_dispatch }`. `goal_dispatch_system` consumes that before legacy preserve-arm list, so tasks survive stale-reset ticks only while owning goal/task matches.
- **Consumers** (executor exits in `gather.rs`, `dig.rs`, `corpse.rs`, `construction.rs`, `items.rs`, `production.rs`, `teaching.rs`, `sleep.rs`, plus `MilitaryMove` arrival) call `aq.advance()` instead of `current = Idle`. `sleep::sleep_task_system` owns `Task::Sleep` end-to-end: arrival `Working`→`Sleeping`, recovery, `finish_task` retirement, **orphan self-heal** (`cancel_chain` when `current == Sleep` but state is `Idle`/`Attacking`). Canonical exit helpers: `aq.finish_task(&mut ai)` (success — `state = Idle` + `work_progress = 0` + advance), `aq.cancel_chain(&mut ai)` (abort — same + cancel). Long non-cargo work uses `goals::yield_for_maintenance_boundary` at safe boundaries; clears `active_method` before dropping current chain so HTN history records neither success nor failure.
- **External preempts** (`dispatch_player_command_system`, hunter demote, stale reset, `movement::release_to_idle`, `combat::combat_retaliation_cleanup_system` draining `CombatRetaliationStartedEvent`) call `aq.cancel()` / `aq.cancel_chain(&mut ai)`. Retaliation cleanup runs in `Sequential` right after `combat_system` because `combat_system`'s `attacker_query` holds `Option<&mut ActionQueue>` mutably across iteration. **A *Sleeping* victim emits `CombatRetaliationStartedEvent` on every damaging hit** so multi-attacker swarms can't skip the deferred cancel. `goal_update_system` overrides (Lead/Defend/Raid/Rescue/Scout/Migrate chief / commanded force-flip / earnincome override) must also call `aq.cancel()`. **Debug-time enforcement:** `aq.dispatch` fires `debug_assert!` when incoming task shares `TaskKind` with `current` and queue is empty.
- Per-tick pin sites (lecture/teach pin writes) stay as direct `aq.current = X` writes — idempotent re-assertions.

**Variants:** `Idle`, `WalkTo { tile, z, why }`, `WithdrawGood { filter }`, `WithdrawMaterial { resource_id, qty }`, `WithdrawFood { tile }`, `Equip { slot, resource_id }`, `Construct { blueprint }`, `ConstructBed { blueprint }`, `Deconstruct { tile }`, `Terraform { tile }`, `MilitaryAttack { foe }`, `Gather`, `Dig`, `Scavenge`, `Read/Teach/HoldLecture/AttendLecture`, `PickUpCorpse`, `HuntPartyMuster`, `Hunt`, `HaulCorpse`, `Butcher`, `TameAnimal`, `Sleep`, `Eat`, `HaulToBlueprint`, `HaulToCraftOrder`, `WorkOnCraftOrder`, `DepositToFactionStorage`, `WalkAndTakeFromMember`, `PlayThrow`, `PlayPlant`, `Explore`, `Migrate`, `UnpitchStructure`, `UnloadCampCargo`, `PitchStructureAt`, plus `Lead`/`Defend`/`Raid`/`RescueAlly`/`Socialize`/`Play`.

**Rules:**

- Systems mutating typed task **must** include `&mut ActionQueue` alongside `&mut PersonAI`.
- Every executor exit must `aq.advance()` (success) or `aq.cancel()` (chain-drop). Prefer `aq.finish_task(&mut ai)` / `aq.cancel_chain(&mut ai)`.
- `military_task_system`, `withdraw_good_task_system`, `withdraw_material_task_system`, `drink_task_system`, `sleep_task_system` fall back to `Idle` (`cancel_chain`) if `aq.current_task_kind() == X` but `aq.current` is wrong variant. `sleep_task_system` extends to state-orphan case.
- **Reading the discriminant.** All consumers read `aq.current_task_kind()` (derives `u16` via `task_kind_for`). Legacy `PersonAI.task_id` mirror gone; `UNEMPLOYED_TASK_KIND` (`u16::MAX`) is "no task" sentinel for `Task::Idle`.

## HTN domain (`htn.rs`)

- **`AbstractTask` / `AbstractTaskKind`** — high-level goal (`Sleep`, `Eat`, `AcquireFood`, `AcquireGood { resource_id }`, `StockpileFood`, `Scout`, `EquipHuntingSpear`, `ReturnSurplus`, `TameWildAnimal`, `PlantFromStorage`, `HarvestPlant`, `ConstructBlueprint`, `Socialize`, `DeliverMaterialToCraftOrder`, `WorkOnCraftOrder`, `HarvestGrainForCraftOrder`, `Play`, `JoinHuntParty`, `EngagePrey`, `DeliverHuntKill`, plus single-step combat/faction goals).
- **`Method` trait** — `precondition` / `utility` / `expand(ctx) -> Vec<Task>` / `flags` / `tech_gate` / `profession_gate` / `policy_gate` / `name`. `policy_gate` returns `&'static [(ResourceId, RequiredFlag)]`.
- **`MethodRegistry`** built once in `SimulationPlugin::build` via `register_builtin_methods`. Read-only after init.
- **`MethodFlags`** = `u8` bitmask. **`MethodId`** = `u16` newtype with one `pub const` per registered method.
- **`MethodHistory`** — per-agent ring (len 6, TTL 600). Spawned at every Person spawn.
- **Failure-biased scoring:** `score_method_with_history(method, abstract_task, ctx, history, now) = utility - failures * 0.4`. Routing/missing-target push `FailedRouting` / `FailedTarget`.
- **Cancel paths record failure.** Executor `aq.cancel()` paths that drop chain push `active_method` onto `MethodHistory` via `record_target_failure(...)` / `record_routing_failure(...)`. `MethodId::UNKNOWN` for cancels with cleared `active_method`. Sites: `gather::finish_gather`, `items::item_pickup_system`, `items::finish_scavenge`, `production::finish_withdraw_material`.
- **Goal-flip Abandoned (`record_abandoned_method_system`)** ParallelA, after `goal_update_system`, filtered on `Changed<AgentGoal>`. Pushes `Abandoned` on agent's `active_method` and clears it — unless `MF_UNINTERRUPTIBLE`, in which case `goal_dispatch_system`'s preserve-arms keep chain alive.
- **Terminal Explore fallback** (`htn_acquire_food_dispatch_system` + `htn_stockpile_food_dispatch_system`): when every precondition fails, falls through to synthetic `Task::Explore { kind: AnyEdible }`.
- **Reachability-aware deposit pick** (`StorageTileMap::nearest_for_faction_reachable`) and **vision pickers** (`CurrentVision::nearest_gather_target` / `nearest_scavenge_target`) take `is_reachable` closure with two-pass fallback. `source_tile` for deposits is agent's post-pickup tile.
- **Detour-aware (river-aware) target/deposit selection.** All gather/scavenge/cluster/storage/bid selection ranks candidates by `pathfinding::detour::DetourEstimator` (chebyshev-equivalent tiles, `max(chebyshev, chunk-hop cost)`) instead of straight-line. Sites: `memory.rs` vision/scavenge pickers, `shared_knowledge.rs` cluster picker, `nearest_for_faction_reachable`, `jobs.rs` U_bid `C_action`. Plain `nearest_for_faction` stays Manhattan (cheap accessor). `dist_penalty`/`full_trip_penalty` stay chebyshev (arbitrate between methods for already-detour-selected target). Drink/heal/hunt short-range scans keep chebyshev.
- **Chronic-failure release** (`goals::chronic_failure_release_system`, Economy daily quarter): agents with ≥3 non-success entries within TTL get the goal stamped on `GoalCooldown` (ring of 4 with `(goal_disc, expires_tick)`). Bifurcates: claim-holders release claim; autonomous land in `ForceGoalReevaluate` (drained next tick to bypass 200-tick cadence). Maintenance goals exempt.
- **Success recording:** dispatchers stamp `ai.active_method = Some(method.id())` on success; `htn_method_completion_system` observes `current == Idle && queued_is_empty && active_method.is_some()` and records `Success`.

### Dispatch + chain handoffs

## Memory & gathering (`shared_knowledge.rs`)

3-tier `SharedKnowledge` map is system of record for static-resource queries.

- **`SharedKnowledge`** keyed by `KnowledgeTier::{Household(u32), Settlement(SettlementId), Faction(u32)}`. - **Tier gossip promotion (`knowledge.rs`)**: `HouseholdMember` who is a social contact (`is_social_contact`) within 3 tiles of same-root-faction `Bureaucrat`/`Chief` promotes household clusters to Settlement tier; two officials → Settlement bubbles to Faction.

### Construction (`construction.rs`)

### Bridges & Dams

`BuildSiteKind::Bridge` adds tile-replacing finalize (alongside `Wall`): writes `TileKind::Bridge` over prior `River` cell, spawns `Bridge { restore_tile, faction_id, tile }`. Deconstruct refunds `2 wood + 1 stone` on nearest passable bank.

**Dams mirror this pipeline.** `BuildSiteKind::Dam` → `TileKind::Dam=25` over `River`/`Water` + `Dam { faction_id, tile, restore_tile, crest_z }` in `DamMap`. Entity is durable truth; tile kind is cache projection, restamped from `DamMap` on chunk reload by `world::water_runtime::restamp_runtime_water_on_chunk_load` (shared `stamp` closure with Bridge). Recipe `6 stone + 4 wood`, 180 work, `DAM_BUILDING`-gated (Bronze, prereqs `BRIDGE_BUILDING + MONUMENTAL_BUILDING`). Cell is **not** water-like/fresh/drinkable. Finalize calls `RuntimeWater::register_dam(tile, crest_z=bp.target_z)`. Deconstruct shares `water_anchored_refund_tile` with Bridge: restores prior tile, `RuntimeWater::clear_dam`, refunds `3 stone + 2 wood`. `FurnitureMaps` bundles `dam_map` + `runtime_water`. Built right-click Build Dam on River/Water **and** AI-planned by `organic_settlement::dam_intent_emitter_system`.

- **Bridge recipe**: "Timber Bridge" — 4 wood + 2 stone, 120 work_ticks, `tech_gate = Some(BRIDGE_BUILDING)`.
- **Tech**: `BRIDGE_BUILDING` (Chalcolithic, `AdoptionScale::Institutional`, prereqs `PERM_SETTLEMENT + DUGOUT_CANOE + COPPER_TOOLS`). **Civic gate**: `CivicKind::Bridge` at `(Chalcolithic, 20)` peak pop.
- **Worker adjacency**: `Blueprint.work_stand: Option<(i32, i32)>` caches chebyshev-1 bank tile. `BuildSiteKind::is_water_anchored()` returns true for **Bridge and Dam**. `work_stand_for_bridge(...)` populates at spawn.
- **AI bridges**: `organic_settlement::bridge_intent_emitter_system` scans each faction's `SettlementBrain.road_segments` for river runs `1..=MAX_BRIDGE_SPAN = 4` bounded by passable banks; one bridge per faction per cadence.
- **AI dams** (`dam_intent_emitter_system`, scheduled beside bridge emitter): gated `DAM_BUILDING` + `CivicKind::Dam` (Bronze, 30 pop) + settled non-SOLO. `score_dam_site` composes **Irrigation** (nearest watercourse upstream of Ag-belt, gravity-feed: water_z ≥ belt_z), **Reservoir** (nearest watercourse to home × dryness × population × seasonal-low lift), **RoadCrossing** (short crossing detected via `detect_bridge_runs_in_trace`). `DAM_SEARCH_RADIUS=40`, `DAM_MIN_SPACING=24`, `DAM_SCORE_THRESHOLD=1.0`; author-less. `season_low` from `Calendar::discharge_multiplier`.
- **River-aware seeding**: `paleolithic_hearth_positions_river_aware` projects offsets through `river_context::project_to_safe_bank` (bounded BFS); rejects `river_distance_at <= 1` flood band. Wired into Paleo/Meso seed + `seed_nomadic_camp`.
- **River-aware organic planner** (`build_road_network`): pre-`BRIDGE_BUILDING`, drops river-crossing segments via `trace_crosses_river` and filters anchors through `same_bank_bfs`. Primary spoke order `RiverAxis`-aware. Post-tech, filters lift and `bridge_intent_emitter_system` back-fills crossings. **Anchor-driven phase additions:** Village emits perpendicular cross only when `member_count ≥ 16` OR a WaterAccess/Field/Market anchor projects > 6 tiles off spine. Chiefdom drops symmetric ±18 parallels, emits secondary spurs toward top-3 unmet (perp-projection > 2) anchors, weight-sorted, capped at 3. Secondary/Alley endpoints ±1 tile jitter keyed on `layout_hash`; Primary spine endpoints stay verbatim.

- Wolf/Deer don't drop Meat/Skin on death. `combat.rs::death_system` strips AI/needs/species, inserts `Corpse { species, fresh_until_tick }`. `corpse_decay_system` despawns at `CORPSE_FRESHNESS_TICKS=600`.

- **`EquipHuntingSpear`** is **goal-agnostic** — runs for any unarmed Hunter so mid-`Lead`/`Defend`/`Socialize` hunter fetches spear when `HuntOrder::Hunt` live. Chain preservation keys on `ai.reserved_resource == Some(weapon())` for `WithdrawMaterial` and `Task::Equip { resource_id == weapon() }`.
- **Tool-acquisition dispatcher (`tools::htn_acquire_tool_dispatch_system`)** second goal-agnostic pre-dispatch in ParallelB (after spear, before work dispatchers). Maps worker's `AgentGoal` to `ToolForm` (GatherWood→Axe, GatherStone→Pick, Farm→Sickle, Craft→first `tool_requirement` via `JobBoard::get`); when `ToolKit` lacks form and faction storage has satisfying `Item`, dispatches `Task::WithdrawTool { req }` + `Task::StowToolKit { req }`. `WithdrawTool` shares `TaskKind::WithdrawMaterial` (production executor's `as_withdraw_tool()` branch). `goal_dispatch_system` preserve-arms key on `aq.current.as_withdraw_tool().is_some()` and `TaskKind::StowToolKit`. `GatherFood` intentionally **not** mapped.
- **Live chase (`combat::hunt_chase_system`):** Sequential, after `sync_indexed_after_move_system`, before combat. Reads `aq.current = Task::Hunt { prey }`; on despawn → `record_target_failure + aq.cancel()`; on prey-moved within `HUNT_LEASH_RADIUS (30)` → update tiles and replan; beyond leash → cancel `FailedTarget`.
- **`Carrying(Entity)`** marker for "carrying corpse E"; inserted at pickup, removed at butcher/decay/rescue/muster/hunter-demote. `respond_to_distress_system` recruits any same-faction Hunter within `HUNTER_RESPOND_RANGE=50` regardless of LOS.

## Raids (`raid.rs`)

Faction-level state machine — `FactionData.raid_phase: RaidPhase` is source of truth; `raid_target` / `under_raid` are derived projections.

- **`RaidPhase`** `{ Idle, Preparing { since_tick, target }, Marching { .. }, Engaged { .. }, Cooldown { until_tick } }`. `is_active()` = `Preparing|Marching|Engaged`.
- **Sustained-pressure trigger (`faction_decision_system`, Economy):** raid only when BOTH `food_deficit` (`food_total < members`) AND `hunger_crisis` (`avg_hunger >= RAID_MIN_AVG_HUNGER=140`) held `RAID_TRIGGER_DAYS=2` days. Per-faction streak counters track last false tick. No emergency override; no single-tick flip. `RAID_COOLDOWN_DAYS=10` after every raid (half on aborted prep).
- **Target filter** (`pick_raid_target`): excludes SOLO/self/same-root kin/tributary pairs/food-poor (`food <= members * RAID_RIVAL_RESERVE_PER_MEMBER=5`)/beyond `RAID_MAX_TRAVEL_TILES=500`. Nearest.
- **Capability drafting** (`raid_capability` + `select_raid_party`): ranks by `0.5*hp_frac + 0.3*str_norm + 0.2*armed`; rejects dead/destroyed-torso/<50% HP. Party cap = `min(RAID_MAX_PARTY_ABS=8, ceil(members * 0.35))`. Chief and `Drafted` excluded; chief topped up only if party would be `< RAID_MIN_PARTY=2`. Stored in `FactionData.raid_party` — **only** set `goal_update_system` may put on `AgentGoal::Raid` (via `FactionRegistry::is_raid_party_member`).
- **Preparing phase:** `raid_prep_dispatch_system` (ParallelB, before combat dispatcher) routes unarmed party to withdraw+equip `weapon`. `Preparing → Marching` when all armed, or half-armed past `RAID_PREP_TIMEOUT_TICKS/2`. Aborts on recovery/target loss/`RAID_PREP_TIMEOUT_TICKS=3 days`.
- **March / engage:** `htn_combat_faction_dispatch_system`'s `Raid` arm only marches `Marching|Engaged`. `raid_detection_system` flips `Marching → Engaged` when raider reaches enemy home (chebyshev ≤ 1).
- **`raid_execution_system`** (Economy): only `Engaged` raiders within chebyshev ≤ 1 of enemy home steal food. Per-raider steal cooldown via `PersonAI.last_raid_steal_tick`; reserve gate keeps target above `members * 5`; successful steal bumps `raid_stolen_food`.
- **End (`Engaged → Cooldown`):** food recovered, target drained, `RAID_ENGAGED_TIMEOUT_TICKS=2 days`, or living party `< RAID_MIN_PARTY`.

## Plant lifecycle (`plants.rs`)

Calendar-driven, season-edge-triggered. `Plant.growth: u16` is stage-time accumulator; on each season change `plant_lifecycle_system` adds `season_growth(kind, prev_season)` and runs at most one stage transition.

- **Per-season growth:** Winter dormant (lethal for Grain). Grain Spring/Summer/Autumn = 4/5/2; BerryBush 5/4/2; Tree 4/5/3.
- **Stage thresholds:** Grain Seed/Seedling/Mature = 3/5/2 (annual). BerryBush 5/30/4 with Harvested mirroring Seedling. Tree 12/48/18.
- **Sprout chance:** Seed → Seedling rolls `f32() < 0.20`; failure despawns. Plants with `Cultivated` marker (sown via `Task::Planter`) **skip the roll**; wild scatter / `PlayPlant` keep 20%.
- **Harvest seed cofactor:** Grain harvest returns `+2` `grain_seed` (`harvest_extra_yields`).
- **Mature → fruiting:** single roll at transition (Grain 20% / BerryBush 10% / Tree 5%) for one Seed in chebyshev radius (Grain/Berry r=2, Tree r=3) onto `Grass | soil_like` (incl. `Cropland`, via `seed_target_tile_ok`). Post-roll Grain → Overripe, BerryBush → Harvested, Tree → Mature.
- **Winter mortality:** every Grain plant despawns at Winter onset.
- `plant_lifecycle_system` runs FixedUpdate `Sequential` after `advance_calendar_system`. Edge-triggered via `Local<Option<Season>>`.

## Fishing (`fishing.rs`)

Terrain-based renewable food, unlocked by Mesolithic `FISHING`. Fish are local renewable **stocks**, never per-tile entities.

- **`FishStock`** Resource — sparse `AHashMap<(i32,i32), FishStockCell { habitat, biomass, capacity }>`. Lazily initialized per water tile on first fishing touch, deterministic from `WorldSeed` (capacity `0.75..1.25 ×` habitat base, +15% near rivers). Untouched read implicit-full. Lives across chunk streaming.
- **`FishHabitat { River, Lake, Marsh, Coast }`** — derived by `habitat_at(chunk_map, globe, tile)` from `TileKind` + salinity. `Bridge`/`Dam` not spots.
- **`FishingMethod`** — v1 uses `Handline` (90 work ticks); `Trap`/`Weir`/`Net`/`BoatLine` reserved.
- **`fish_regen_system`** (Economy, daily) — logistic regrowth with flat `REGEN_SEED` floor; seasonal multiplier.
- **HTN methods** — `FishForImmediateFoodMethod` (`AcquireFood` → `[Fish, Eat]`) and `FishForStorageMethod` (`StockpileFood` → `[Fish, DepositToFactionStorage]`), `UTIL_BASELINE`. Food/stockpile dispatchers populate `PlannerCtx.fish_spot_tile` via `nearest_fishable_water` (`ChunkMap`-only within `FISHING_SEARCH_RADIUS`) — only when faction knows `FISHING`, so `fish_spot_tile.is_some()` doubles as tech gate. Each dispatcher's head-match has `Task::Fish` arm; stakes `GatherClaims` keyed `(spot, Resource(fish))`. Depleted-spot avoidance is `MethodHistory` failure-bias.
- **`fish_task_system`** (Sequential, before `gather_system`) — on `Working` adjacency, accumulates `FishingMethod::work_ticks`, harvests `FishStock` (yield × habitat × season × `SkillKind::Fishing` × `food_yield_multiplier`), deposits to free hand (overflow spills `GroundItem`), logs `ActivityKind::Fishing`, grants Fishing XP. `finish_fish` releases claim, routes trailing `Eat` / `DepositToFactionStorage`. `Task::Fish { spot_tile, method, output_resource }` / `TaskKind::Fishing = 55`; `goal_dispatch_system` preserves `(GatherFood|Survive, Fishing)`.
- **Resources:** `fish` (edible, perishable) + `preserved_fish` (`preserved` tag → two-pass ration); auto-counted by `food_total()`. `Preserved Fish` recipe 19: `2 fish + 1 wood → 3 preserved_fish`, `FOOD_SMOKING`-gated. `SkillKind::Fishing = 8` / `ActivityKind::Fishing = 13`.

## Realistic tools (`tools.rs`)

Separate functional tools — `ToolForm` decides *what* (Knife/Axe/Pick/Hammer/Sickle/Awl/FishingKit); `Item.material+quality` → `ToolTier` (Bone<Stone<FineStone<Copper<Bronze) decides *how well*. `ToolKit` (`Vec<Item>`, era-scaled `capacity_for_era` 1/3) separate from `Carrier` hands; spawned on every Person.

- **Hard gates** (graceful — no `ToolKit` reads satisfied; empty kit blocks/degrades): `gather_system` — Axe to fell trees (no-Axe → deadwood, tree NOT despawned), Pick to mine Stone/Wall/Ore (no-Pick → 1-stone trickle), Sickle to reap mature Grain (no-Sickle → FailedTarget); `dig_system` — Pick only when carved floor `is_stone_like()`; `fish_task_system` — Fishing Kit; `craft_order_system` — recipe `tool_requirements`. Better `ToolTier` shortens work via `work_speed_mult`.
- **Seeding** — `seed_starting_tools_system` (OnEnter after `seed_starting_farms_system`): per faction derives era from `current_era(techs)`, builds `starting_tool_loadout`, pre-stows core tool into each kit + spills rest to storage as `GroundItem`. Nomadic round-robin. Sandbox skipped.
- **Tool acquisition** — `Task::WithdrawTool { req }` (shares `WithdrawMaterial` executor/`TaskKind`) selects best-tier matching tool `Item` from storage into hands; trailing `Task::StowToolKit { req }` (`TaskKind::StowToolKit = 57`, `tools::stow_toolkit_task_system`) moves into kit, evicting lowest-tier resident to ground if full.
- **Faction demand** — `compute_faction_tool_deficits` nets era loadout against member-kit + storage counts; feeds `compute_craft_demand` so chief posts missing recipes (`craft_priority` 3). Generic `tools` survives only as Ard-Plow ingredient.

## Animal husbandry v2 — draftwork (`draftwork.rs`)

Plow + cart both shipped via standard `JobBoard` + `JobClaim` pipeline.

1. **Animal training (`animal_training_progress_system`, Sequential, `TICKS_PER_DAY/4`).** Per housed `DomesticAnimal`, bumps `training += TRAINING_INCREMENT_PER_PASS`. Cattle/Horse cross `TRAINING_THRESHOLD_DRAFT = 80` in ~80 days.
2. **Chief plow posting (`chief_job_posting_system` Plow branch, Economy, daily Spring).** Per settled non-SOLO with `ARD_PLOW` tech AND ≥1 `ard_plow` in storage, walks state-owned Ag plots, emits one `JobKind::Plow` per plot with `plowed_year != Some(current_year)`. `JobProgress::Plow { plot_id, area, plowed_tiles: 0, target_tiles: plot.rect.area(), assigned_worker: FarmPlotAssignments.assigned_farmer(plot), animal: None }`. `posting_target_workers(Plow) = 1`. Reachability gated by `rect_reachable_from_home`.
3. **Claim (`jobs::job_claim_system`).** `JobKind::Plow → AgentGoal::Farm`. `assigned_worker` gate restricts to that farmer.
4. **Plow dispatcher (`htn_plow_dispatch_system`, ParallelB).** Per `JobClaim::Plow` holder on `Farm` with idle queue, picks next un-plowed tile via row-major scan keyed on `posting.plowed_tiles`, then picks animal via `pick_idle_draft_animal`: prefers un-claimed trained Cattle/Horse owned by faction; falls back to `None` (human-drawn). Routes via `assign_task_with_routing(TaskKind::Plow)`, dispatches `Task::Plow { plot_entity, animal: Option<Entity> }`. When committing to ox, stamps `posting.animal = Some(ox)` + inserts `AnimalWorkClaim { use_kind: Plow, expires_tick: now + PLOW_CLAIM_TTL_TICKS }`. Human-drawn leaves `posting.animal = None`.
5. **Plow executor (`plow_task_system`, Sequential, before `gather_system`).** Per-tile work. Cost: `plow_work_ticks(animal) = 6` ox / `12` human. Accumulates to threshold, credits `posting.plowed_tiles += 1`, `aq.finish_task`. On final tile: stamps `Plot.plowed_year = Some(calendar.year as u16)`, calls `record_progress_filtered(JobKind::Plow, None, 0)` → `JobCompletedEvent { completed: true }` + despawn posting, releases `AnimalWorkClaim`, removes `JobClaim`, 6 completion XP. Per-tile XP +2. Both modes stamp same `Tilled` marker (1.4× harvest bonus).
6. **Tilled marker stamp (`production_system` Planter branch).** When Grain plant spawned inside plot with `plowed_year == Some(calendar.year)`, plant gets `Tilled`.
7. **Yield bonus (`gather_system` Grain branch).** Plant with `Tilled` → `apply_plow_yield_bonus(base) = base * 7 / 5` (1.4×). Stacks with `food_mul`.

**Plot tillage.** `Plot.plowed_year: Option<u16>` durable per-plot stamp. Re-plowing overwrites; harvest reads `Tilled` on Plant, not plot field.

**Ard Plow implement** (`ard_plow` resource + recipe 14). Wood×3 + Tools×1 at Workbench, `ARD_PLOW`-gated. Tool-class, `two_hand` bulk, sprite reuses `item_hammer`. Posting gates on `faction.storage.stock_of(ard_plow) > 0`. Not consumed at use.

**Preserve-arm.** `goal_dispatch_system` preserves `(AgentGoal::Farm, TaskKind::Plow)`.

**Chief funding.** Plow postings go through `chief_post_funding_system` — Mixed/Market debit treasury, pay on completion. Subsistence unpaid.

## Vehicle system (`vehicle.rs`)

Freeform customizable vehicles. Designs live on bounded **3D cell grid** (`GRID_MAX` 10 wide × 8 deep × 6 tall) — height is load-bearing for mass/stability/clearance/combat.

- **Data model.** `VehiclePartKind` (Frame/Deck/Wall/Axle/Wheel/Hitch/Yoke/CargoBay/CrewSeat/WeaponMount/Engine/Track/ArmorPlate/Turret), `VehicleCell { kind, material: ResourceId, durability, variant: Option<VehiclePartVariantId>, module_id: Option<VehicleModuleId> }`, `VehicleGrid { cells: Vec<(IVec3, VehicleCell)>, modules: Vec<VehicleModuleInstance> }`, `VehicleDesign`, `VehiclePurpose`, `VehicleStats`, `VehicleFootprint` (XY pre-rotated for 4 headings), `VehicleInventory`, plus `Vehicle`/`VehicleCrew`/`VehicleDraft`/`VehicleState`.
- **Variants.** `PartVariantDef` (`core.ron` `variants:`) gives part kind behavioural identity — `mass/support/traction/turn_radius/cargo_volume/durability/engine_power` multipliers. `VehicleCell.variant = None` = plain. `variant_of` resolves; `cell_mass_g` / `support_limit_g` / `derive_stats` / `cell_durability` fold multipliers (turn radius is product). Some variants carry `tech_gates` (`iron_rim_wheel`/`metal_track` → `bronze_casting`).
- **Weapon modules.** `VehicleModuleDef` (`core.ron` `modules:`) is multi-cell weapon firing once per module — `footprint`, `firing_arc` (`None`/`Front90`/`Full360`), `range`/`damage`/`siege_damage`, `crew_required`/`gunner_required`, `cooldown_ticks`, `required_support`, `extra_bill`, `tech_gates`. `VehicleModuleInstance { id, def, cells, facing }` on `VehicleGrid.modules`; cells back-reference via `module_id`. `validate_grid` adds `ModuleOutOfBounds`/`ModuleOverlap`/`UnsupportedModule`/`BadModuleFacing` (Front90 must touch body front edge). `collect_design_tech_gates` unions every gate; custom designs register with derived gate set; `vehicle_assembly_system` enforces faction knows them all.
- **Catalog.** `assets/data/vehicles/core.ron` (materials keyed by catalog `ResourceId`; part defs; stock templates Handcart / Ox Cart / Four-Wheel Wagon / Light Chariot / War Chariot). `load_vehicle_assets()` builds `VehicleData` + `VehicleDesignRegistry`.
- **`validate_design`** — deterministic 3D checks (connectivity, floating cells, wheel↔axle, control cell, draft capacity, axle overload, cargo reachability, chariot rule). **`derive_stats`** — mass / 3D center-of-mass / `stability = track_width/com.z` / payload / draft power / speed caps.
- **VehicleYard** (`BuildSiteKind::VehicleYard`, 12 wood + 6 stone, `ANIMAL_HUSBANDRY`) — assembly + parking anchor; `VehicleYardMap` on `PenMap` hook pattern. `HitchingPost.parked_cart` renamed `parked_vehicle`.
- **Assembly.** `PlayerCommand::QueueVehicle { design_id }` (faction-level) → `VehicleAssemblyQueue`. `vehicle_assembly_system` (Economy, 60-tick) drains queue: `design_bill` checked against storage, consumed, parked `Vehicle` spawned at yard.
- **Pathfinding.** `pathfinding::vehicle_path::footprint_astar` — heading-aware A* over `VehicleNode { x, y, z, heading }`. Successors: forward / forward-diagonal (z ±1) + turn-in-place (cost from `turn_radius`); each candidate rotates footprint to heading and tests via generic `cell_ok(x,y,z)` closure. Live caller `vehicle::plan_vehicle_route` folds `ChunkMap::passable_at` + `vertical_clearance_at` + `VehicleOccupancyIndex`.
- **Occupancy.** `VehicleOccupancyIndex` (tile → `Vehicle`, 2D) — full rebuild per tick by `vehicle_occupancy_sync_system` (Sequential, after `sync_indexed_after_move_system`). `footprint_tiles(design, anchor, heading)` shared enumerator.
- **Rollover.** Pure: `step_tip_torque(stats, RolloverContext)` scales turn/slope/rough/overload disturbance by COM height; `vehicle_rolls_over(stats, torque)` compares accumulated torque to `stability`. Road-bound loaded ox cart never tips; tall narrow overloaded design on slope does.
- **Cargo hauling — vehicle-leads movement.** Vehicle is **authoritative mover**: owns `VehiclePathFollow { path: Vec<VehicleNode>, cursor, tip_torque }` planned by `footprint_astar`; driver + draft animals ride it.
  - **`htn_vehicle_haul_dispatch_system`** (ParallelB, before hand-haul) — for idle `JobClaim::Haul` holder whose posting still needs `>= VEHICLE_HAUL_MIN_REMAINING = 12` units, claims idle owner-faction cargo vehicle (`hauler == None`, `design_is_cargo_capable`, not `Overturned`) + `design.required_animals` trained Cattle/Horse onto `VehicleDraft.hitched`, routes worker on foot to vehicle tile to board.
  - **`vehicle_cargo_haul_task_system`** (Sequential) — boards worker (`BoardedVehicle`) on arrival, drives two-phase state machine: plans vehicle route via `plan_vehicle_route` (tries target tile + surrounding rings as goal anchors); on arrival (`footprint_reaches`) transfers `min(blueprint need, capacity_units(payload))` into `VehicleInventory` (load) / deposits + credits via `record_progress_filtered` (deliver). On completion releases draft, re-parks, unboards, drops `JobClaim`.
  - **`vehicle_movement_system`** (Sequential, after `movement_system`) steps along `VehiclePathFollow` at terrain speed cap, updates `anchor_tile`/`heading`/`z`, accumulates `tip_torque`. Holds when driver Dormant.
  - **`vehicle_rollover_system`** — overturns when `tip_torque > stability`: `VehicleState::Overturned`, ejects driver (fall damage `height_z × 8`), spills `VehicleInventory` as `GroundItem`s, releases draft.
  - **`vehicle_crew_sync_system`** snaps boarded driver + hitched animals each tick. `movement_system` / `recover_stranded_agents_system` skip `Without<BoardedVehicle>`.
  - **`vehicle_haul_recovery_system`** (Economy, 60-tick) re-pools vehicle whose driver lost mid-haul.
  - `Task::VehicleCargoHaul { vehicle, blueprint, resource_id }` / `TaskKind::VehicleCargoHaul = 54`; `goal_dispatch_system` preserves `(AgentGoal::Haul, TaskKind::VehicleCargoHaul)`. `compute_faction_storage_system` folds `VehicleInventory` into `storage.totals`. `entity_sprites::spawn_vehicle_sprites` attaches sprite. Sub-12-unit hauls still hand-carry.
- **AI provisioning.** `vehicle_yard_intent_emitter_system` (Economy, daily) drops `VehicleYard` blueprint for settled `ANIMAL_HUSBANDRY` faction with `≥ YARD_MIN_MEMBERS = 10` and no yard. `vehicle_ai_queue_system` (Economy, daily) enqueues conservative stock (Handcart) for faction that owns yard but no vehicle.

## Nomadic mode

`Lifestyle::{Settled (default), Nomadic}` on `FactionData`. Nomads skip Settlement spawning, run camp pipeline, migrate seasonally. AI commits atomically (Surveying → PendingCommit → PackingCamp → Traveling → PitchingCamp); player nomads via `PlayerCommand::PackCamp` / `PitchCamp`.

- **`Deployable { packed_form, refund_pct, refund_resource, refund_qty }`** (`pack_deploy.rs`) on every nomadic structure. Bedroll (1 skin + 2 wood) and Yurt (8 wood + 6 skin, `PORTABLE_DWELLINGS`-gated) `fully_packable`; Tent (6 wood + 3 skin) `refund_only(0.5, wood, 6)`.
- **Build sites:** `BuildSiteKind::{Bedroll, Tent, Yurt}`. Bedroll finalizes as `(Bed { tier: Crude }, Deployable, StructureLabel("Bedroll"))`. Yurt packs into `packed_yurt`.
- **Camp seeding (`seed_nomadic_camp`):** hearth → bedrolls (radial 2..=5, one per founder), tents (outer 5..=7, ~1 per 4), yurts (Neolithic+, inner 3..=5, cap 2).

### Lifecycle

`FactionData.camp_state: CampState::{Pitched, Packed { since_tick }}` + `migration_phase: MigrationPhase::{Idle, Surveying, PendingCommit, PackingCamp, Traveling, PitchingCamp}` + `nomad_autopilot: bool`. AI runs FSM; player nomads (`autopilot == false`) skip for free-form Pack → Move → Pitch.

**AI migration (`nomad.rs`):**

- `nomad_migration_system` (Economy, daily, exclusive): **strict-AND trigger** — `Surveying` only when stored food `< members × NOMAD_TRIGGER_FOOD_DAYS (3.0)` AND known local food clusters score `< members × NOMAD_KNOWN_FOOD_TARGET_PER_MEMBER (1.0)`. No hunger override. Cooldown via `migration_cooldown_ticks(caps.home)`. Picks 2–3 lowest-need, stamps `AgentGoal::Scout` + `ScoutAssignment`.
- `nomad_survey_dispatch_system` (ParallelB) routes Scouts via `Task::Explore { kind: AnyEdible }` through `SCOUT_CHECKPOINT_STEP = 24` chebyshev checkpoints.
- `nomad_survey_completion_system` (Economy daily, exclusive): at `SURVEY_WINDOW_TICKS = TICKS_PER_DAY*4`, runs `pick_migration_candidates`. Best `>= NOMAD_MIN_ACCEPTABLE_SITE_SCORE (15.0)` → `pending_migration`, `PendingCommit`. Otherwise `Idle`, retry `NOMAD_NO_CANDIDATE_RETRY_DAYS (2)`. `fallback_direction` retained `#[allow(dead_code)]` for manual/debug.
- `pick_migration_candidates` score = food + herd + water + biome-season + danger + recency − `d * DIST_WEIGHT (0.4)`. `NOMAD_MIN_TARGET_DIST = 8`, max `200`. **Herd visibility knowledge-gated** — from `MemoryKind::HerdSighting` clusters in faction-tier `SharedKnowledge`, not omniscient. Water/River/Wall/Ore centre tiles early-rejected.
- `score_danger` penalises `MemoryKind::HostileFactionSighting` only. Must **not** read `MemoryKind::Prey`.
- `nomad_migration_commit_system` (Sequential, exclusive): `PendingCommit` validates → `PackingCamp` flips `camp_state = Packed`, dispatches `UnpitchStructure` for Deployables → `Traveling` stamps every member with `MigrationTarget`, inserts `FollowingBand` on owned `Tamed`, keeps `home_tile` at old camp → `PitchingCamp` begins on arrival; only then `home_tile` moves, `CampMoved` log fires.
- `MigrationTarget { tile, route_tile, started_tick, last_dispatched_tick, bounce_count }` + `WalkReason::Migration` + `TaskKind::Migrate`. `bounce_count` cap 2 reroutes.
- `nomad_migration_dispatch_system` (ParallelB, `ParamSet`): no `Without<Drafted>` filter — drafted hunters migrate.
- `nomad_migration_arrival_system` (Sequential, after movement): strips on `MIGRATE_ARRIVAL_RADIUS = 4`, `MIGRATE_TIMEOUT_TICKS = TICKS_PER_DAY*3`, or `MIGRATE_STALL_TICKS = TICKS_PER_DAY/2`.
- `goal_dispatch_system` preserve-arms: `(MigrateToCamp, Migrate)`, `(Scout, Explore)`.

**Player Pack/Pitch:** chief-actor faction-scoped commands write to `PendingCampOps`. `apply_pack_camp_command_system` runs observable HTN labor — stamps non-drafted with `PackingDuty` (preempts chains, releases reservations, drops `JobClaim`), enumerates `Deployable`s within `seed_nomadic_camp_extent` of `home_tile`, dispatches `Task::UnpitchStructure` to nearest worker. `unpitch_structure_task_system` (`nomad_pack_labor.rs`) accumulates to `UNPITCH_WORK_TICKS = 40`, despawns, drops packed_form/packed_bundles/refund as `GroundItem`. `camp_state = Packed` flips immediately. `apply_pitch_camp_command_system` keeps synchronous `seed_nomadic_camp`.

**Manual scout (`PlayerCommand::SendScout { direction, range }`):** chief-actor queues `PendingManualScout`; stamps `ScoutAssignment { kind: ScoutKind::PlayerManual { faction_id }, … }`. `manual_scout_completion_system` runs `pick_migration_candidates(intent, k=4)` on arrival; folds into `FactionData.candidate_sites` (cap 16). `PlayerCommand::SetMigrationIntent` writes `faction.migration_intent`.

**Intent-weighted scoring** — six intents: `FreeRoute`, `FollowWater` (water×2), `FollowHerds` (herd×2), `SeekWinterShelter` (biome×2.5, dist×0.8), `SeekSummerPasture` (food×1.5, biome×2), `AvoidDanger` (danger×3). Candidates carry per-tile `reasons: Vec<CandidateReason>`.

**Cargo manifest / pitch repair:** `FactionData.cargo_manifest: CampCargoManifest { required, loaded, abandoned, deployed, pitching_started_tick, repair_unlocked }`. AI final-destination pitch unloads carried `bedroll`/`packed_yurt`/`wood`/`skin`, consumes via `PitchStructureAt`. Once minimal camp exists, migration completes after 80% arrival or `TICKS_PER_DAY/2` since pitch start; `nomad_chief_directive_system` repairs missing shelter. Temporary waypoints never unlock repair.

**Mobile-state goal gate (`mobile_state_goal_gate_system`, ParallelA after `goal_update_system`):** `CampState::Packed` demotes settled-life goals + strips `JobClaim`. Player `Forage` uses `allowed_while_packed` (Survive/Sleep/GatherFood/Socialize/Defend/Rescue/Play/FollowingPlayerCommand/MigrateToCamp/Scout/care/Drink). AI caravan uses stricter `allowed_while_ai_caravan`.

**Player-locked packed migration:** `PackedMigrationAutonomy::{Hold, Forage}`. Player default `Hold`, reset on every `PackCamp`. `Hold` strict-demotes everything except `FollowingPlayerCommand`, `PlayerManual` Scout, sim-owned `MigrateToCamp`.

**Pitch conserves shelter count:** `apply_pitch_camp_command_system` sweeps Deployables within `seed_nomadic_camp_extent` of old home, then `seed_nomadic_camp` re-seeds and debits one `bedroll`/`packed_yurt` per spawned shelter from member then pack-animal inventories. First pitch forgiven.

**Slim chief directives (`nomad_chief_directive_system`):** gates on `caps.home.is_mobile() && pending_migration.is_none()`. Targets via `nomad_shelter_targets(members)`. Spawns one Single-tile blueprint per kind per tick, cap `NOMAD_DIRECTIVE_BP_PER_TICK = 2`. No job postings.

**Band redistribution (`nomad_pool.rs`):** `nomad_band_pool_balance_system` shrinks max-min spread for `bedroll`/`packed_yurt`/`preserved_meat` to ≤1 unit across band within `POOL_BAND_RADIUS = 12`.

**Pack-animal logistics:** `PackAnimalInventory { items: [(ResourceId, u32); 6], capacity_g }` on `Tamed`. Capacities: Horse 60kg / Cow 80kg / Pig 30kg / Dog 15kg. `attach_pack_inventory_system` auto-inserts on `Added<Tamed>`. `compute_faction_storage_system` folds into nomad `storage.totals`. `combat::death_system` drops contents as `GroundItem`.

**Preserved meat ration:** `2 meat + 1 wood → 3 preserved_meat` (CraftRecipe 12, `FOOD_SMOKING`-gated). `eat_task_system` two-pass: fresh first, preserved only if nothing else.

**Wild herds (`wild_herd.rs`):** `WildHerdRegistry` per-herd `(id, species, aggregate_count, leader_tile, range_center, bloomed, members, flee_until_tick, last_birthed_tick)`. `seed_wild_herds_system` (Startup) places `WILD_HERD_COUNT = 3` at grasslands, `WILD_HERD_AGGREGATE = 120` each. `wild_herd_migration_system` (daily): predator flee × 3 / water seek / camp avoidance × 4 / non-Winter birth +12 capped at 200. Bloom/collapse at camera distance 32/48 spawns up to 60.

**Sedentarization (`nomad_sedentarize_system`):** emits `SwitchArchetype` when `member_count ≥ 12`, no `pending_migration`, stable ≥ `TICKS_PER_SEASON * 4`. **Reverse collapse (`sedentary_collapse.rs`):** **combined-failure** trigger — settled "failing" only when sustained food deficit (`food_total < members × COLLAPSE_FOOD_DAYS=3.0`) coincides with population crash (`< SEDENTARY_COLLAPSE_MIN_MEMBERS=6`) OR shelter loss (faction-owned beds `< ceil(members × COLLAPSE_MIN_BED_COVERAGE=0.5)`). Single weak signal never collapses. `collapse_failing(food_deficit, pop_crash, shelter_loss)` pure predicate. `collapse_streak` must reach `COLLAPSE_TRIGGER_TICKS = TICKS_PER_SEASON × COLLAPSE_TRIGGER_SEASONS=2` → `SwitchArchetype`. Bed counting (`usable_beds`) scans `COLLAPSE_BED_FALLBACK_RADIUS=32` around home and excludes other-faction beds. On settled→nomadic, `handle_switch_archetype` resets band to `Pitched`/`Idle`, clears `pending_migration`, stamps `last_migration_tick = now`.

## Pluralist Economy

Per-resource policy flags + per-settlement markets + sub-faction households + Bureaucrat/Trader/Healer professions + P2P currency + escrow + U_bid scoring + tribute + craft contracts. Currency invariant: `EconomicAgent.currency + FactionData.treasury + Settlement.treasury + JobEscrow.held()` conserved (`held() = amount + purchase_pool`).

### Settlements (`settlement.rs`)

**Economic** unit (market + treasury + market_tile), distinct from `SettlementPlan` (layout). One faction can own many.

- `Settlement` (Component): `id`, `owner_faction`, `market_tile`, `founding_tick`, `name`, `treasury: f32`, `market: SettlementMarket`, `peak_population`.
- `SettlementId(u32)`; `SettlementMap` with `by_id` / `by_megachunk` / `by_faction`.
- `auto_found_default_settlements_system` (FixedUpdate Economy + OnEnter, idempotent) spawns one per non-SOLO at `home_tile`.

### Currency + escrow

- **`pay(world, from, to, amount) -> bool`** (`transactions.rs`) — atomic agent-to-agent. Only sanctioned way.
- **`FactionData.treasury: f32`** — faction wealth.
- **`JobEscrow { amount, beneficiary, purchase_pool }`** sidecar per funded posting. Producer debits wallet + spawns sidecar. `job_payout_system` (Economy, exclusive, after `job_claim_release_system`) splits `escrow.amount` across claimants on completion, sets `amount = residual`, despawns — `on_remove` hook refunds `held()` to beneficiary. On failure, full `held()` refunds. `chief_post_funding_system` sizes `purchase_pool = max_unit_price·target` for `HaulSource::Market` (0 otherwise), debits `wage + purchase_pool` from treasury. Workers get `Earnings(VecDeque<EarningEntry>)` ring (cap 16) + `ActivityEntryKind::WagePaid`.
- **Chief-funded:** `chief_post_funding_system` (Economy, exclusive, after `chief_job_posting_system`) computes `chief_wage_for(progress)` (Stockpile/Haul/Craft → `trade_base_value × qty × CHIEF_MARGIN (0.5)`, Haul ×0.5; Calories/Build/Planting → flat per-day, cap `CHIEF_BUILD_WAGE_CAP=30`). Subsistence skips.
- **Wage signal:** `FactionData.wage_signal: AHashMap<(JobKind, Option<ResourceId>), WageEMA>`. `faction_wage_signal_system` (Economy, daily, after `job_payout_system`) folds last-day Earnings into EMA at `α ≈ 0.129` (5-day half-life). Cross-faction via `PerceivedFactionWages` (cap 32). `wage_gossip_system` piggybacks `Socialize` within `WAGE_GOSSIP_RADIUS = 3` with `exp(-age / TICKS_PER_DAY)` staleness.
- **Hook:** `JobsPlugin::build` registers `on_remove(on_job_escrow_remove)`.

### Per-resource policy (`economy/policy.rs`)

`ResourceControlPolicy` flags: `chief_allocates_labor`, `private_actors_allowed`, `state_sells_at_market`, `prices_fixed_by_state`, `fixed_price`. `Default` = all-communist; `capitalist()` preset. `FactionData.economic_policy` empty by default. `Method::policy_gate` returns required flags; `method_passes_policy_gate(...)` is the check. SOLO rejects any non-empty gate.

### Sub-factions / households

Household is `FactionData` with `parent_faction = Some(village_id)`.

- `FactionData.{parent_faction, household_head, children_factions}`.
- `FactionRegistry::spawn_household(parent, home_tile, head, &catalog)`. Non-empty parent `economic_policy` → household `capitalist()`; empty → empty.
- `FactionRegistry::root_faction(id)` walks `parent_faction`.
- **Formation:** `CoSleepTracker.bond_strength` accumulates cosleep ticks with same partner; at `HOUSEHOLD_BOND_THRESHOLD` (one game-week) `household_formation_system` spawns household.
- **Market preset spawn seeding:** `spawn_population` calls `seed_market_households` after per-faction loop. Every adult founds one-person household with own plot, `FactionStorageTile`, `HOUSEHOLD_SEED_TREASURY (15.0)`.
- **Inheritance:** `pregnancy_system` threads mother's `Option<&HouseholdMember>` to newborn.
- **`HouseholdMember` is marker only** — parents keep `FactionMember.faction_id` pointing at village.

### Wage-aware labor market

End-to-end: chief-funded postings → escrow → worker payout → faction wage signal → EV-driven profession choice.

- **Capital recognition (`capital.rs`):** `capital_factor(...)` averages three axes in `[1.0, 2.0]`: `tool_capital_factor` (+0.5 when held item maps via `tool_profession`: weapon→Hunter, tools→Crafter); `workshop_capital_factor` (+1.0 household-owned affine workshop within `WORKSHOP_AFFINITY_RADIUS = 12`, else +0.5 village-owned); `land_capital_factor` (Farmer: +0.5 for household-held Agricultural plot under non-StateOwned). `OwnedBy { faction_id, kind, tile }` stamped at workshop finalize; add/remove hooks maintain `WorkshopOwnership`. `WorkshopKind::affine_to`: Market→Bureaucrat, Workbench/Loom→Crafter, Shrine→Healer.
- **Skill decay (`skills.rs`):** `Skills` clamped at `SKILL_MAX=255`; `SkillPeaks` + `SkillUseTicks` ratchet via `skill_peaks_tracker_system`. `skill_decay_system` (daily, half-life 90 days): mastered (peak ≥ `SKILL_MASTERY_LINE=80`) decay toward `SKILL_MASTERED_FLOOR=30`; below mastery toward `max(SKILL_FLOOR_BASE=5, peak × 0.30)`.
- **Profession choice (`profession_choice.rs`):** `expected_wage = aggregate_wage_per_day × skill_competence(primary_skill) × capital_factor`; `skill_competence(s) = 0.2 + (s/SKILL_MAX) × 0.8`. Target-driven assignment systems sort by `(expected_wage, primary_skill)`. Shared `demote_profession_state` releases reservations, cancels tasks, strips `Carrying`.
- **Survival override:** below `FARMER_SURVIVAL_FLOOR = 16.0` per-head food, every non-Farmer assignment zeros target. Bands: <16 stand-down; 16–32 Farmer ramp; >32 × `FARMER_DEMOTE_RATIO (1.6)` shed excess. Apprentices stay bound.
- **Asymmetric demotion hysteresis:** `HUNTER_DEMOTE_BUFFER = 1`, `BUREAUCRAT_DEMOTE_BUFFER = 1` add one-slot tolerance. Promotion eager; demotion only on `current > target + buffer`. `want == 0` bypasses. Crafter uses EMA-band hysteresis.
- **Cross-profession switcher (`cross_profession_switch_system`):** daily Economy; switches Hunter↔Bureaucrat↔Crafter when `EV(target) > EV(current) × EV_SWITCH_HYSTERESIS (1.20)`. `switching_cost_skill_regret = peak[primary]/SKILL_MAX × 0.20 × aggregate_wage(current)`. Faction caps via `faction_cap_for`. Sub-`APPRENTICE_THRESHOLD` Crafter routes through `Apprentice`.
- **Apprenticeship (`apprenticeship.rs`):** novice-Crafter onboarding. `None → Crafter` with `Skills[Crafting] < APPRENTICE_THRESHOLD (30)` scans mentor pool (Crafter + `Skills[Crafting] >= MASTER_THRESHOLD (100)` + no `MentorOf`). Pairs `ApprenticeOf { mentor }` / `MentorOf { apprentice }` + `ApprenticeProgress { ticks, target_ticks: TICKS_PER_DAY × APPRENTICESHIP_DURATION_DAYS (30), target_profession }`. No mentor → direct Crafter. `apprentice_progress_system` (Economy, daily) ticks + graduates. Payouts split: apprentice `share × WAGE_FRACTION_APPRENTICE (0.4)`; mentor `share × WAGE_FRACTION_MENTOR_FEE (0.1)`; residual `0.5·share` refunds. Deliberate-practice 2× XP via `apprenticeship::xp_with_apprentice_bonus`.

### U_bid scoring at job-claim layer

`job_claim_system` branches on `posting.reward`:

- **Paid (`reward > 0.0`):** `U_bid = E(R) + priority_bonus + affinity_bonus − C_action − C_opportunity` where `E(R) = posting.reward × wealth_modifier(currency) × disposition.earn_income_multiplier()`; `disposition multiplier = 1.0 + entrepreneurial/255`; `wealth_modifier(c) = 1.0 + 0.5 / (c + 50)`; `priority_bonus = posting.priority * 0.01` (chief 200, household 180, individual 100); `C_action = euclidean × BID_DIST_DISCOUNT`; `C_opportunity = 0.0` (stub).
- **Unpaid (`reward == 0.0`, chief/legacy):** legacy `priority + skill + bias − distance`.
- **Affinity:** `CRAFTER_AFFINITY_BONUS = 3.0` lifts Crafter paid-Craft; unpaid `profession_bias` arms `(Crafter, Craft) = 0.5`, `(Crafter, Stockpile) = 0.1`.

## Land ownership (`land.rs`)

Plot-based ownership over compat `SettlementPlan` projection from `SettlementBrain` parcels. Plots start `Tenure::StateOwned`; in Mixed/Market `land_listing_system` publishes, households acquire what they can afford, rent collects monthly with eviction after two misses, sharecrop splits harvest at gather.

- **`Plot` (Component):** `id, settlement_id, faction_id, rect, z, zone_kind, tenure, holder, base_value, last_valued_tick, missed_payments, frontage_edge, access_tile, parent_plot`. `Tenure ∈ {StateOwned, Leased, Sharecropping, Freehold}`. `TenureHolder ∈ {State, Household}`.
- **`PlotIndex`:** `by_id`, `by_settlement`, `by_tile` (surface-only), `ag_tiles`, `next_id`. `plot_at(x, y)` hot lookup.
- **`carve_plots_system`** (FixedUpdate Economy, after `settlement_planner_system`, before `chief_directive_system`; also runs once at `OnEnter(Playing)`): **reconciles per-plot every tick** against current `SettlementPlan.zones`. Each zone subdivides into one or more `(ZoneKind, TileRect)` canonical keys; matching `Plot` kept (preserving entity, `plowed_year`, `tenure`, `base_value`); orphaned torn down via `tear_down_plot`. Fresh carved for canonical entries with no match. No `culture_hash` early-return — `layout_hash` is position-blind. Plot sizes: Residential 6×6, Crafting/Storage 4×4, Agricultural **16×16** (matches organic parcel size); Civic/Sacred/Market/Defense whole. Agricultural tiles land in `PlotIndex.ag_tiles` (road protection covers all 256 tiles) and get deterministic role via `field_tile_role(culture_hash, faction_id, tile, rect)` → `{Cropland (60%), CroplandLow (15%), SoilFallow (20%), GrassEdge (5%)}`; perimeter biases GrassEdge +0.30. **Orphaned-Cropland revert pass:** any tile dropped from `ag_tiles` that no current Agricultural plot covers gets `TileKind::Cropland` rewritten back to natural (`biome_bands(classify_at_tile).pick(v)`, `TileChangedEvent` emitted).
- **`compute_plot_value`** (`PLOT_BASE_VALUE = 50.0`): `BASE × zone_mul × (centre_factor + home_factor) × terrain_factor`. Anchors on `market_tile` + `home_tile`; terrain samples fertility at centre + 4 corners.
- **`LandPolicy`** (`economy/policy.rs`): `land_policy_for(EconomyPreset)`. Subsistence all-false; Mixed = rent + sharecrop; Market = adds sale + freehold. `default_lease_period_days = 30`, `rent_yield_pct = 0.04`, `default_share_to_landlord = 0.30`.
- **`tile_buildable_by`**: rejects placements on plots held by another faction/household. Wired into `chief_directive_system`.
- **`land_listing_system`** (Economy, every `TICKS_PER_DAY/4`): publishes Sale + Lease for `StateOwned` when `LandPolicy` permits. Civic skipped. Per-(faction, zone-kind) caps (`RESIDENTIAL_LISTINGS_CAP=6`, `AGRICULTURAL_LISTINGS_CAP=6`, `CRAFTING_LISTINGS_CAP=4`). Sale asking = `base_value`; lease asking = `base_value × rent_yield_pct`/month, floored `MIN_MONTHLY_RENT`.
- **`household_land_acquisition_system`** (Economy, daily): per-zone-kind track. Households browse parent listings for Residential and Agricultural separately; can hold one of each. Affordability: lease ≤ 40%, buy ≤ 70% of treasury. Preference: Sale → Lease → Sharecrop. Agricultural prefers `Profession::Farmer`-headed. Acquiring Ag plot spawns household-private `FactionStorageTile`. Also claims nearest unowned same-village Ag plot within 12 chebyshev as child.
- **`rent_collection_system`** (Economy, every 30 game-days): expired Leased attempt household→landlord transfer. Failure bumps `missed_payments`; at `EVICTION_MISS_THRESHOLD = 2` → `StateOwned`, emits `PlotEvictedEvent`.
- **`evicted_plot_cleanup_system`** drains queue per `caps.land.eviction_policy`. `Demolish` walks `StructureIndex` over `plot_rect`, refunds via `Deployable::compute_refund_drop`, despawns.
- **Sharecropping:** `state_sharecrops` flag adds `ListingKind::Sharecrop` for Agricultural. Zero upfront; tenure `Sharecropping { share_to_landlord, paid_through_tick }`. `gather.rs` harvest hook routes landlord share via `split_sharecrop_yield(qty, share_to_landlord)` as `GroundItem` at landlord's nearest `FactionStorageTile`.

## Farming (`farm.rs`, plot-scoped, seasonal)

- **Seasonal priority.** In-season field work is food-security, not low-share communal job:
  - **Season-aware `farm_pressure(faction, &Calendar)`** — `0` Winter, full annual-deficit ratio (vs `farm::annual_grain_target` = `members×48×5/4`) Spring/Autumn, scaled by `SUMMER_FARM_PRESSURE_SCALE=0.35` Summer. One signal feeds both `compute_priority` and `compute_workforce_budget`. Annual constants (`GRAIN_PER_PERSON_PER_YEAR=48`, `GRAIN_YIELD_PER_TILE_PLANNING=4`, `SUPPLY_SAFETY_NUMER/DENOM=5/4`) live in `farm.rs`, re-exported from `organic_settlement.rs`.
  - **Seasonal hard-priority** — `compute_priority` lifts open seasonal `FieldWork` (`assigned_farmer==None`) to `SEASONAL_FARM_PRIORITY=170` (outranks chief Haul/Build/Craft, `< PRIORITY_PLAYER`). Suppressed when `food_pressure ≥ CRITICAL_FOOD_TRIGGER (80)`. Summer caretaker Prepare and Plow not boosted.
  - **Phase-weighted claim floor.** `cap_bucket` keys `FieldWork` claims by `FarmWorkPhase` so Prepare and Plant get independent caps. `job_claim_system` raises open seasonal `FieldWork` per-phase cap via `farm::seasonal_field_work_floor_share`: Spring splits legacy 0.65 envelope into `SPRING_FARM_PLANT_SHARE=0.30` + `SPRING_FARM_PREPARE_SHARE=0.35`; when only one Spring phase has open postings, the other carries over. Autumn keeps `SEASONAL_FARM_CLAIM_SHARE_AUTUMN=0.45`. All gated `food_pressure < CRITICAL_FOOD_TRIGGER`. Spring `Plant` postings carry `SPRING_PLANT_PRIORITY_BIAS=+5` over `SEASONAL_FARM_PRIORITY`.
  - **Budget farm-backlog gate** — `workforce_budget_system` passes `farm_backlog` (any non-complete `FieldWork`/`Plow`). `compute_workforce_budget`: `farm_backlog==false` ⇒ `raw_farm=0` AND Farm slot policy-dormant.
- **Phase-correct dispatch + progress.** Each farm dispatcher honours claimed posting's `FarmWorkPhase` via `htn::claimed_fieldwork_phase(claim, board)`. `jobs::record_fieldwork_progress(board, job_id, phase, delta)` credits `FieldWork` posting **only when phase matches** — `prepare_field_task_system`→`Prepare`, planting completion→`Plant`, `gather_system` Grain branch→`Harvest`.
- **Seasonal cycle.** Pure-function `farm_season_phase(&Calendar) → FarmSeasonPhase::{SpringPrepPlant, SummerMaintenance, AutumnHarvest, WinterDormant}`. Chief Farm branch dispatches per phase: Spring posts `FieldWork{phase:Prepare, target=unprepared}` (open) + `FieldWork{phase:Plant, target=plantable.min(grain_seed_stock)}` (open); Summer caretaker Prepare (`assigned_farmer:Some`); Autumn one Harvest per plot with `target=mature_grain_count`; Winter nothing. `posting_target_workers(JobKind::Farm)` = `(target/8).clamp(3, 12)`.
- **Field state (`FieldTileIndex`).** Resource keyed `(tile → FieldTileState { plot_id, nutrients: u8, last_crop: Option<PlantKind>, last_worked_year: u16 })`. World-gen `TileData.fertility` is immutable per-tile recovery ceiling; `nutrients` is live pool, mutated by harvest/prepare/fallow only. Seeded by `carve_plots_system` for every Agricultural tile (`nutrients = TileData.fertility`); `seed_starting_farms_system` no longer touches it. `backfill_field_tile_index_system` (OnEnter) is safety net for save-game tiles.
- **Prep gate.** `Task::PrepareField { tile }` (`TaskKind::PrepareField=52`) executor `farm::prepare_field_task_system` (Sequential, after movement, before gather): accumulates `FIELD_PREP_WORK_TICKS=80`, stamps `TileKind::Cropland`, emits `TileChangedEvent`, credits Farm posting through `jobs::record_progress` so target completion removes posting and releases `JobClaim`, grants `SKILL_XP_PER_PREP_TILE=4` Farming XP, bumps `nutrients` to `max(current, EXHAUSTED_FLOOR=30)`. Dispatcher `htn_prepare_field_dispatch_system` (ParallelB) routes Farm-claimed Prepare-phase worker to nearest `find_nearest_unprepared_in_rect(...)` via `dispatch_autonomous_task_with_routing`.
- **Plant gate.** `htn_plant_from_storage_dispatch_system` calls `find_nearest_plantable_in_rect(...)` which requires `tile.kind == Cropland` AND `FieldTileIndex[tile].nutrients >= MIN_PLANTABLE_NUTRIENTS=80`. Bootstrap (no plot rect) keeps legacy `find_nearest_unplanted_farmland` radius search. Both also skip tiles held by in-flight `PlantingReservations`.
- **Generic crop pipeline.** Posting classifier, Plant-target cap (`grain_seed_stock = faction.storage.seed_total()` summed over `PlantKind::ALL.filter_map(|k| k.seed_resource())`), Autumn Harvest target (`mature_crop` over any `kind.is_farm_plantable() && stage == Mature`), and `gather.rs` `FieldWork::Harvest` credit all read `PlantKind::is_farm_plantable()`. Adding a plantable seed = one `PlantKind` entry + one arm in `seed_resource()`.
- **Typed seed on `Task::Planter` / `Task::PlayPlant`.** Both carry `seed_resource: ResourceId` populated at dispatch. Executor reads via `aq.current.as_planter_full()` / `as_play_plant_full()` and consumes exactly that resource via `PlantKind::from_seed_resource(seed_id)`. `production::finish_withdraw_material` Planter/PlayPlant arms include no-seed precheck.
- **Planting tile reservation (`plants::PlantingReservations`).** Per-tile `(tile → {worker, seed_resource, reserved_tick})` for in-flight chains. Both dispatchers reserve at dispatch; the three search helpers skip reserved. Released at every executor exit and chain-cancel. Daily `planting_reservation_gc_system` (Economy, `RESERVATION_MAX_AGE_TICKS=3600`) drops entries whose worker died, no longer running for reserved tile, or aged.
- **`WithdrawMaterial.source_faction_id: Option<u32>`** overrides executor's ownership gate at storage tile. `None` defaults to worker's `FactionMember.faction_id`. `Some(fid)` needed for `FarmScope::Private` (household member drawing from household-private storage).
- **Harvest yield + nutrient debit.** `gather.rs` Grain branch reads `FieldTileIndex[tile].nutrients` and scales qty via `grain_yield_for_nutrients`: `≥180→5, ≥120→4, ≥80→3, else 1`. Then debits `nutrients -= HARVEST_NUTRIENT_DEBIT=30 (saturating)`, sets `last_crop = Some(Grain)`, `last_worked_year = calendar.year`. `FieldTileIndex` + `Calendar` bundled into `GatherRoutingResources`.
- **Fallow recovery (`fallow_recovery_system`, Economy, season-edge).** `Local<Option<Season>>` fires once per season change; walks `FieldTileIndex.by_tile`, for tiles with no live plant AND `last_worked_year < calendar.year` bumps `nutrients += FALLOW_NUTRIENTS_PER_SEASON=15`, capped at fertility ceiling. Exhausted (< 30) tile must rest ~4 seasons.
- **Demand-driven plot sizing (`organic_settlement::parcel_targets`).** `target_plots = (target_active / 96).max(1).min(12)` where `target_active = min(food_tiles, labor_tiles, seed_tiles)`: `food_tiles ≈ members*15`, `labor_tiles = (members*60/100)*24`, `seed_tiles = grain_seed_stock.max(current_ag_tiles).max(32)`. Constants `pub` in `organic_settlement.rs`. 20-member tribe targets ~300 tiles ⇒ ~2 plots.
- **FarmWorkScorer season gate (`goal_scorers.rs`).** Adds `ctx.farm_season` and `ctx.private_plot_has_seasonal_work`. Returns `None` Winter or when household plot has no Prepare/Plant (Spring), Prepare (Summer), or Harvest (Autumn) work in season. `goal_update_system` builds `households_with_seasonal_work` snapshot per cycle.
- **Plot size:** Agricultural plots are 16×16 tiles (~576 m² at 1.5 m/tile) carved 1:1 from organic Agricultural parcels — one plot per parcel.
- **`FarmPlotAssignments`** (Resource): one-to-one match `farmer Entity ↔ PlotId`. Maintained daily by `chief_farm_plot_assignment_system`. Greedy match by chebyshev; releases stale on despawn/demote/plot leaves state ownership/plot vanishes.
- **Seasonal chief Farm postings** emit `JobProgress::FieldWork { phase, completed, target, area, plot_id, assigned_farmer }` per plot per phase. `resolve_farm_scope` reads same variant. Shared `farm::state_owned_ag_plots_for_faction(...)` enumerates plot set. **No Agricultural plot ⇒ NO Farm posting**.
- **Claim restriction:** `job_claim_system` rejects posting whose `assigned_farmer` doesn't match candidate.
- **Plant ownership:** `production.rs` cascade is `JobKind::Farm + FactionMember → Faction-owned`, then `HouseholdMember → Household`, else `Person`.
- **`FarmScope` resolver** (`htn.rs::resolve_farm_scope`): three branches mirror `production.rs` cascade — **Communal** (`JobClaim::Farm` posting with `plot_id` + matching `assigned_farmer` → village storage, plot rect from `PlotIndex`), **Private** (ANY `HouseholdMember` whose household holds `ZoneKind::Agricultural` plot → household storage as `source_faction_id`, `fallback_source_faction_id = village` for §5 seed-fallback, deposit override `Some(household_id)`), **Bootstrap** (else → village storage, no plot rect). Farmer-only gate on Private gone: a Mason in kitchen-garden household can tend their plot.
- **Plot-restricted planting search:** `htn_plant_from_storage_dispatch_system` walks `FarmScope::seed_source_candidates()`. Uses `scope.plot_rect()` to drive `find_nearest_unplanted_in_rect(..)`. Bootstrap falls to `find_nearest_unplanted_farmland(.., radius=15)`. §5 fallback avoids first-season deadlock for freshly-housed kitchen-garden household.
- **Plot-restricted harvest search:** `htn_harvest_plant_dispatch_system` reuses resolver. Communal/Private walk plot's rect via `PlantMap`. Private threads `deposit_target_faction_override = Some(household_id)`; `gather_deposit_tile` routes through household storage. Bootstrap keeps legacy `GatherKnowledge::nearest_target_tile`.
- **`FarmWorkScorer`** (Subsistence-tier, score 0.90 + 0.05 Farmer lift): household-availability gate — `ctx.private_farm_available` true when agent is `HouseholdMember` of household holding `ZoneKind::Agricultural` plot. Any household adult nominates `AgentGoal::Farm`. Grain policy `private_actors_allowed=true` still gates (Mixed/Market).
- **`Task::DepositToFactionStorage` carries `target_faction_id: Option<u32>`:** `Some(household_id)` overrides actor-faction routing.
- **Mixed preset enables capitalist grain:** every catalog resource gets `mixed()` (chief allocates AND private actors allowed).
- **Game-start seeding (`seed_starting_farms_system`):** OnEnter(Playing), after in-chain survey → project → carve. Seeder **does NOT create Plot entities** — `carve_plots_system` owns that. Finds carved Agricultural plot nearest home (walks `PlotIndex.by_settlement[sid]` → filters `Agricultural && faction_id == fid`, deterministic tie-break) and stamps starter patch. If no Agricultural plot exists, seeder stamps no Cropland but still seeds grain + food buffer. Per founding faction:
  - **Seed grain** = `demand_tiles.max(32)` (`demand_tiles` = `parcel_targets` `food_tiles`).
  - **Year-1 food buffer** — edible `grain` `GroundItem` on faction storage, `members * GRAIN_PER_PERSON_PER_YEAR * 3/5`.
  - **Starter Cropland patch** — bounded contiguous fraction of carved plot (`min(demand_tiles/2, plot_area/2)`, row-major) pre-tilled to `Cropland` (`set_tile` + `TileChangedEvent`). Fertility is NOT overridden — `FieldTileIndex` nutrients already seeded by `carve_plots_system` from natural `TileData.fertility`.
  Sandbox + nomadic skipped.
- **Belt placement near-edge bias (`build_ag_belt`):** each candidate pays chebyshev distance penalty `0.5 * ((cheb(c, home) - fp_extent).max(0) / (scan - fp_extent).max(bw))` so "good and adjacent" beats "very slightly better and far".
- **Kitchen-garden parcels (`append_dwelling_gardens`):** after `build_ag_belt`, keyed on actual built dwellings — `detect_dwellings` reconstructs Hut (3×3) and Longhouse (5×3 / 3×5) from survey wall snapshot. For every dwelling with no belt parcel within `KITCHEN_PROXIMITY = 12` chebyshev, emits house-sized Agricultural: **3×3** Hut garden on any clear edge; **3×4** Longhouse garden on **3-tile short wall only**. Overlap keep-out excludes Residential parcels + Residential district disc. `land_listing_system` 12-tile child-claim path (`CHILD_FARM_MAX_DIST`) binds as household's child plot. Routes through `FarmScope::Private`.
- **Storage spawn on child-Ag claim (`land_listing_system::ensure_household_storage`):** Residential→child-Ag claim path now spawns `FactionStorageTile`. Hoisted helper called from both direct-Agricultural and child-claim arms.
- **Plot-demand Farmer promotion (`faction_profession_system`):** target_farmers is `max(food_target, plot_target)` where `plot_target = min(state_owned_ag_plots_count, members/3)`. A village at moderate `per_head` with three unworked plots still recruits ≥ 3 Farmers. Plot-demand forms emergency floor (`food_target_when_low.max(plot_target)`) below promote threshold.

## Game lifecycle and regions

- **`GameState`:** `SpawnSelect` (default) | `Playing`. Spawn UI in `SpawnSelect`; spawn systems `OnEnter(Playing)`. Update-stage gated `in_state(Playing)`; FixedUpdate not gated.
- **Sandbox bypass:** `SandboxPlugin` flips state to `Playing` at Startup.
- **`SettledRegions` (`region.rs`):** `AHashMap<RegionId, SettledRegion>` + `by_megachunk`. `SettledRegion { megachunk, founding_tick, name, camera_bookmark, player_owned }`. `settle()` idempotent.
- **`MegaChunkCoord`:** `MEGACHUNK_SIZE_CHUNKS=16`. Independent of climate cells. Helpers: `from_tile`/`center_tile`/`chunk_range`/`tile_bounds`/`contains_tile`.
- **Player home is mega-chunk-constrained.** `spawn_population` special-cases `group_idx == 0`: calls `region::pick_player_home_in_megachunk(chunk_map, mx, my, world_seed)` so home tile is always inside cell player picked on spawn-select. Picker deterministic (`fastrand::Rng` seeded via `home_pick_seed`, splitmix64 mix of `(world_seed, mx, my)`) with centre-out spiral fallback and forced-centre last resort. **`home_pick_seed` must never use `ahash::AHasher::default()`** — ahash's default key is per-process. AI factions (`group_idx > 0`) keep 32×32-chunk best-of-200 search.
- **Near factions vs world-map factions.** `spawn_population` spawns only player + `NEARBY_RIVAL_COUNT = 3` rivals as full entity groups (`near_factions = NEARBY_RIVAL_COUNT + 1`, capped). AI homes scored by `faction_spacing_score` — continuous farthest-point reward: min straight-line distance to every already-placed home (incl. player), saturating at `NEAR_FACTION_TARGET_SPACING = 280` tiles, `+0..+100` — blended with river-proximity curve (`−80..+60`).
- **Abstract world-map factions (`abstract_faction.rs`).** Remaining slots seeded by `seed_abstract_factions_system` (`OnEnter(Playing)`, after `spawn_population`) as **abstract factions**: `FactionData` (`materialized = false`, Settled archetype + game economy preset, no chief/members) plus `faction_id` stamped on one habitable Globe `WorldCell` with population/food_stock. Homes farthest-point-spread (`cell_spacing_score`) across Globe outside player's pre-gen keep-out. `world_sim_system` ticks population/food/raids abstractly. `AbstractFactions` (`by_id` + `by_megachunk`).
  - **`FactionData.materialized`** (default `true` in `create_faction`) gates entity-driven systems off abstract factions: `auto_found_default_settlements_system` / `auto_found_default_camps_system` skip them; `faction_decision_system` + `pick_raid_target` skip (entity raid FSM never runs for/targets abstract — `world_sim` owns their raids).
  - **Materialization.** `materialize_abstract_faction_system` (FixedUpdate, after `chunk_streaming_system`): when `ChunkLoadedEvent` arrives for chunk containing abstract faction's home tile, faction spawns as full entities. `person::spawn_faction_band` — band-spawn body extracted from `spawn_population` (home `FactionStorageTile` + members from `spawn_tiles_from` reachable flood + chief; advances `clock.population`/`bucket_size` per member) — shared by both call sites. System re-anchors `FactionData.home_tile` onto real ground (`nearest_passable_tile`), spawns up to `MAX_MATERIALIZED_MEMBERS = 40`, flips `materialized = true`, clears Globe cell's faction fields, drops from `AbstractFactions`. Dematerialise back is not built — materialised faction stays (entities persist, Dormant when far).
- **Multi-focus chunk streaming (`SimulationFocus`):** `Vec<FocusPoint>` rebuilt each tick from camera + every settled region centre. Chunk DATA loads for any focus disc (camera `LOAD_RADIUS=12`, region `REGION_LOAD_RADIUS=6`); SPRITES + plants + loose rocks only inside camera focus.
- **Focus-aware LOD:** camera distance produces base LOD; entities within 8 chunks of any non-camera focus promote from Dormant to Aggregate. `cohort.rs` (`CohortRegistry`) summarises Aggregate agents by `{faction, settlement_or_camp, profession, age_band, wealth_band, lifestyle}`; `cohort_pin_full_sim_system` (ParallelA after camera LOD) promotes commanded/chief/drafted/combat back to `Full` via `PinnedFullSim`.
- **Edge-walk expansion (`region::detect_edge_crossing_system`):** each tick, unsettled mega-chunks the player faction enters get `settle()`, naming "Outpost N", bookmarking, emitting `ActivityLogEvent::RegionSettled`.
