# Rendering (`src/rendering/`)

16×16 pixel-art tiles, camera, chunk-streaming visuals, entity sprites, projection. See root `CLAUDE.md` for the `TileChangedEvent` → `refresh_changed_tiles_system` refresh contract and Z conventions.

## Materials & sprites

- `TileMaterials` / `FogTileMaterials` keyed by `(TileKind, OreKind, z_bucket)`. Ore tiles fan out via `RENDERABLE_ORES`; colors in `color_map.rs::ore_tile_color`.
- **`sprite_library.rs`** — procedural pixel art from a 32-color palette via `ascii_to_image`. Reuse the palette/helpers; don't introduce a new color system.
- **PNG textures** in `assets/textures/` toggled by `entity_sprites::toggle_art_mode`.
- **`AnimalTextures`** — 8-direction PNGs for Wolf/Deer/Horse/Cow/Cat from `textures/<species>/rotations/`; ascii fallback (`creature_*` keys) otherwise.
- **GroundItem sprites** — `entity_sprites::spawn_ground_item_sprites` reactively attaches a child sprite via `ResourceDef.sprite_key`. Add by registering a key in `sprite_library.rs` and pointing the catalog entry at it.

## Plant sprites (`plant_sprites.rs`)

`PlantSpriteSet` resource probes `assets/textures/plants/<folder>/<stage>_<variant>.png` at startup via `Path::exists()` — Bevy's `AssetServer::load` is bypassed for missing files so the resolver can route to a higher fallback tier. `PlantDef.sprite_keys { folder, variants }` (RON) overrides the species `key` as the folder slug and the default 3-variant layout. `PlantSpriteVariant(u8)` stamped at spawn from `splitmix64(species, tile) % 3` — persistent across stage transitions. `entity_sprites::resolve_plant_sprite` is a three-tier fallback: species PNG → form PNG (`_form_<form>/`) → legacy ASCII (`get_plant_texture_legacy`). Stage layout: Seed/Harvested single-variant; Seedling/Mature/Overripe 3-variant. Per-form canvas sizes (form-fallback PNGs ship at these; per-species PNGs follow the per-form size, trees tiered): Grass/Forb/Vine/Tuber 16×16, Shrub 24×24, Cactus 16×32, Aquatic 16×24, Tree 48×80 (Medium baseline; per-species art may upgrade to Small 32×48, Large 64×112, Palm 32×96). `Anchor::BottomCenter` on every plant sprite so trunks plant at the tile and canopies overhang. Form-fallback art is procedurally generated by the `gen_form_fallback_pngs` ignored test. Authoring is incremental — `spawn_plant_sprites`/`update_plant_sprites` take `Option<&PlantSpecies>`/`Option<&PlantSpriteVariant>` so unauthored species and legacy raw-`Plant` test fixtures route through the same resolver.

## Vehicle part sprites (`vehicle_part_sprites.rs`)

Hand-drawn ASCII art per `VehiclePartKind` + variant + multi-cell weapon-module composite. Three views (`VehicleSpriteView::{Side, Front, Back}`); `view_for_heading` maps 0=Back / 1=Side(flip) / 2=Front / 3=Side. Asymmetric parts ship distinct `_BACK` art; symmetric parts re-register `_FRONT` under `_back`. `entity_sprites::vehicle_sprite_plan_with_data(design, heading, &VehicleData)` populates a per-cell sprite plan; multi-cell weapon modules collapse to one composite. Connector overlays bridge cells (`push_connector_overlays` walks `NEIGHBORS_6`, emits seam/attach/axle-wheel sprites).

## Edge structures (thin housing walls)

`EdgeWallVisual`/`EdgeDoorVisual` durable entities spawn oriented thin-strip sprites at the edge midpoint via `entity_sprites::spawn_edge_{wall,door}_sprites` (vertical edge → N-S bar, horizontal → E-W bar; open door dimmed); projection y-sort via `auto_attach_dynamic`, art-mode-toggle aware. (Edge-structure data model lives in `../world/CLAUDE.md`.)

## Overlays & animation

- **Water-current streaks** (`water_current_render.rs`) — animated flow-streak sprites on flowing-`River` tiles; scrolls along `world::water_current::WaterCurrentField`. z=0.35, `ProjectedAnchor::Static`.
- **Day-night overlay** (`day_night.rs`) — full-screen sprite at z=90 tinted by `Calendar::day_fraction()`; per-entity fog tinting multiplies below.
- **Worker interaction nudge** (`animations.rs::interaction_nudge_offset`) — `update_animations` composes a `NUDGE_PX = 5.0` lean on every `Person`'s `VisualChild` toward `PersonAI.dest_tile` while `ai.state() == AiState::Working` AND `aq.current_task_kind()` satisfies both `task_interacts_from_adjacent` and `task_is_labor`. Pure-fn helper; additive with combat lunge/hit offset; never writes the parent `Transform` so sim/replication/fog/projection are oblivious.

## Tilted-view projection (`projection.rs`)

`MapViewMode::{TopDown, Tilted}` (toggle `V` / HUD). Symmetric pre/post: `revert_view_projection_system` (PreUpdate) strips so sim sees logical Transforms; `apply_view_projection_system` (PostUpdate) re-projects. `ProjectedAnchor::{Static{z}, Dynamic}` auto-attached per marker by `auto_attach_dynamic::<T>`. Helpers bundled in the `ViewProjection` SystemParam. `CursorParams::pick_cliff_aware` walks `[Z_MIN, Z_MAX]` matching `surface_z_at`. Bookmarks store **logical** coords. `ElevationSkirt` renders south-facing cliffs.
